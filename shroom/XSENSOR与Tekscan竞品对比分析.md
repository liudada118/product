# XSENSOR 与 Tekscan 竞品对比分析

文档日期：2026-05-09  
文档版本：V1.0  
对标对象：JQ Tools 压力感知软件 V2.0  
当前 PRD 基线：`压力感知软件产品需求文档-飞书表格版-含数据处理模块.md` V1.2  
文档用途：用于判断 JQ Tools 现有功能是否能对标 XSENSOR/Tekscan，以及后续 PRD 补强优先级。

## 1. 本次结论

| 结论项 | 判断 |
|---|---|
| 产品方向是否正确 | 方向正确。JQ Tools 做“国产可替代 XSENSOR/Tekscan 的本地离线压力感知系统”是成立的，尤其适合展厅演示、研发测试、科研采集、本地交付场景。 |
| 当前最能打的点 | 离线可用、设备 MAC/类型校验、本地采集回放、框选独立分析、方向翻转/旋转持久化、CSV 导入导出、纤维压力传感器差异化。 |
| 当前最弱的点 | 当前没有 2D 热力图/压力云图；标定、单位、精度、压力范围表达不够产品化；报告能力弱；竞品级 A/B 对比和科研输出还不完整。 |
| V2.0 重点 | 不要追求一次性做全。V2.0 应优先打牢连接稳定性、数据口径、实时展示、框选分析、采集回放、CSV 可信导出、基础 A/B 对比。 |
| 对外表达建议 | 不建议直接说“全面优于 XSENSOR/Tekscan”。建议说“在本地离线交付、国产传感器适配、区域分析和定制场景上具备替代价值”。 |

## 2. 市场范围定义

| 项目 | 内容 |
|---|---|
| 市场 | 压力分布测量、座椅/床垫/人体支撑面压力映射、研发测试和展厅演示软件 |
| 主要客户 | 汽车座椅研发、床垫/坐垫研发、工学椅研发、科研客户、展厅/门店演示客户 |
| 核心任务 | 实时看到压力分布，发现高压点、左右/前后不均、支撑不足、舒适性差异，并能采集、回放、导出数据 |
| 竞品关键能力 | 2D/3D 实时压力图、录制回放、区域分析、测距、指标统计、对比分析、报告、校准和多传感器配置 |
| JQ Tools 机会 | 用更低门槛的本地化软件和自有纤维传感器，覆盖展厅演示、研发验证和国内交付需求 |

## 3. 竞品概览

| 竞品 | 产品/系统 | 主要定位 | 对 JQ Tools 的威胁 | 可借鉴点 |
|---|---|---|---|---|
| XSENSOR | X3 Pro Seating System、ForeSite SS、软件平台 | 高质量压力图像、座椅/轮椅/人体压力应用、传感器和软件一体化 | 品牌成熟、图像质量强、报告/视频/对比和医疗/研发信任度高 | 高分辨率压力图像、座椅/靠背同步、快速记录和对比、多场景产品包装 |
| Tekscan | BPMS、CONFORMat、I-Scan 等 | 压力映射和力测量老牌系统，覆盖座椅、床垫、支撑面、研发测试 | 软件分析能力完整，2D/3D、帧回放、区域隔离、测距、A/B 对比、ASCII/AVI 导出成熟 | 指标体系、区域分析、帧级回放、前后对比、文档/报告化输出 |

## 4. XSENSOR 对标分析

### 4.1 公开能力摘要

| 能力 | XSENSOR 公开资料表现 | 对 JQ Tools 的启发 |
|---|---|---|
| 压力图像 | 强调高质量压力图像、细节可识别、实时测量和可视化 | JQ Tools 当前无 2D 热力图，视觉冲击力弱于 XSENSOR |
| 座椅/靠背 | X3 Pro 面向座椅和靠背表面压力测量，可用于汽车座椅设计 | JQ Tools 的整体/靠背/坐垫切换方向正确，需要把座椅靠背独立方向和数据口径做扎实 |
| 记录和对比 | 公开资料提到可快速记录和比较多个座椅设计，数据和图片易转移 | JQ Tools 的历史 A/B 对比是正确方向，V2.0 至少应完成基础 A/B/差值/指标差值 |
| 统计和辅助分析 | ForeSite SS 提到统计、解剖标记、对称性、3D 视图、报告 | JQ Tools 可以把“压力重心、左右/前后均衡、框选模板”作为后续增强 |
| 传感器性能 | 强调精度、重复性、无需重新校准、高分辨率和多压力范围 | JQ Tools 需要补齐压力单位、压力范围、校准规则、精度说明，否则科研客户信任度不足 |
| 软件平台 | 有桌面、移动、云、AI/ML 分析方向 | JQ Tools V2.0 不必追云和 AI，先把本地离线交付做成优势 |

### 4.2 XSENSOR 强项

| 强项 | 说明 | JQ Tools 当前状态 |
|---|---|---|
| 高质量压力成像 | 强调清晰压力图像和高分辨率数据 | 偏弱，当前无 2D 热力图/压力云图 |
| 座椅研发场景包装成熟 | X3 Pro 直接对准座椅设计、舒适性、进出座椅、耐久测试 | 方向一致，但 JQ Tools 需要把汽车座椅研发指标说清楚 |
| 对比和记录 | 适合快速记录、比较多个设计 | JQ Tools 已规划 A/B 对比，但导出和曲线对比仍在后续 |
| 报告和分享 | ForeSite SS 可保存图片/视频并生成报告 | JQ Tools 当前报告能力弱 |
| 品牌信任 | 国外成熟品牌，传感器性能表达完整 | JQ Tools 需要用本地化交付和硬件差异化建立信任 |

### 4.3 XSENSOR 弱点和 JQ Tools 机会

| XSENSOR 可能弱点 | JQ Tools 机会 |
|---|---|
| 国外品牌，本地定制和国内交付成本较高 | 强化中文软件、本地部署、离线可用、快速适配客户传感器和设备类型 |
| 成熟系统可能绑定自有硬件和流程 | JQ Tools 可以突出纤维压力传感器和设备类型灵活配置 |
| 云/移动/医疗方向较重，不一定适合所有国内展厅和门店 | JQ Tools 可做轻量现场演示版，保证一键连接和稳定展示 |
| 价格和采购链路通常需要询价和方案配置 | JQ Tools 可主打国产替代、交付响应和总成本优势 |

## 5. Tekscan 对标分析

### 5.1 公开能力摘要

| 能力 | Tekscan 公开资料表现 | 对 JQ Tools 的启发 |
|---|---|---|
| 2D/3D 实时和记录数据 | BPMS/CONFORMat 软件支持 2D 和 3D 实时及记录数据 | JQ Tools 有 2D 数字、3D 模型、3D 数字，但缺少 2D 压力图 |
| 指标统计 | 支持接触面积、平均压力、峰值压力、压力中心/力中心轨迹等 | JQ Tools 指标方向一致，需要补齐公式、单位、阈值、是否排除 0 |
| 帧级回放 | 支持逐帧查看数据 | JQ Tools 已有回放和进度拖动，应补齐帧号、时间戳和异常帧处理 |
| 前后对比 | 支持治疗前后/测试前后并排比较 | JQ Tools 数据对比方向正确，V2.0 需要把可比性校验和差值口径落地 |
| 区域隔离分析 | 支持隔离并分析特定区域 | JQ Tools 框选独立模块非常关键，可作为对标亮点 |
| 测距 | 支持两点之间测距 | JQ Tools 量尺已规划，需保证传感点距、缩放/翻转不影响真实距离 |
| 导出 | 支持 ASCII 或 AVI 等导出 | JQ Tools CSV 导出可满足科研基础需求，但报告和视频导出弱 |
| 校准 | Tekscan 强调系统可用已知力校准以提高精度 | JQ Tools 需要尽快补“标定/校准/单位换算”规则 |

### 5.2 Tekscan 强项

| 强项 | 说明 | JQ Tools 当前状态 |
|---|---|---|
| 分析链路完整 | 实时、录制、回放、逐帧、区域、测距、对比、导出覆盖完整 | JQ Tools 功能骨架接近，但细节和验收口径还需补齐 |
| 2D/3D 压力映射成熟 | 对客户来说直观、易演示、易解释 | JQ Tools 缺 2D 热力图，是直观展示短板 |
| 科研/临床可信度 | 明确指标、系统配置、扫描速度、传感器数量、压力范围 | JQ Tools 需要更明确设备参数和数据字典 |
| 多传感器组合 | BPMS 可按支撑面大小配置多个传感器 | JQ Tools 有多设备类型，但多传感器同步和配置表达还弱 |
| 导出和文档 | 支持结果沟通和文档化 | JQ Tools CSV 可用，但报告能力应进入 V2.2 或更早评估 |

### 5.3 Tekscan 弱点和 JQ Tools 机会

| Tekscan 可能弱点 | JQ Tools 机会 |
|---|---|
| 系统专业度高，国内客户部署、培训、售后成本可能较高 | 做中文化、本地交付、低学习成本的一键连接和演示流程 |
| 通用压力测量系统不一定贴合 JQ 自有硬件 | JQ Tools 可深度适配自有纤维传感器、座椅靠背/坐垫结构 |
| 价格和采购通常为方案型询价 | 用国产替代、可定制和交付速度形成商业优势 |
| 对客户自定义业务流程的适配可能慢 | JQ Tools 可快速做客户模板、框选模板、车型/座椅模板 |

## 6. JQ Tools 功能逐项对标

| 模块 | JQ Tools 当前/规划 | 对标 XSENSOR | 对标 Tekscan | 判断 | 建议 |
|---|---|---|---|---|---|
| 打开软件与连接 | MAC 配置、一键连接、重连、串口扫描、设备状态 | XSENSOR 更偏软硬件一体易用 | Tekscan 为成熟硬件软件系统 | 能打 | 连接稳定性是 P0，必须防卡死、超时、资源释放、错误可解释 |
| 离线本地使用 | 本地配置、无联网使用、本地采集 | XSENSOR 有云/移动/桌面平台 | Tekscan 多为 PC 软件和硬件系统 | 能打 | 把“离线可用、数据本地化、国内交付”作为卖点 |
| 数据处理 | 原始帧、矩阵、显示数据、统计数据、入库数据口径正在补 | 竞品强调高质量处理和可靠数据 | 竞品强调校准、指标和分析 | 偏弱 | P0 补齐数据字典、单位、阈值、坏帧、断帧、采样率、矩阵尺寸 |
| 2D 数字视图 | 已有 2D 数字矩阵和放大镜 | XSENSOR 以压力图像为核心 | Tekscan 支持 2D 压力图 | 部分能打 | 2D 数字适合工程排查，但不适合展厅直观演示 |
| 2D 热力图/压力云图 | 当前没有 | XSENSOR 强项 | Tekscan 强项 | 明显弱 | 不要在 V2.0 宣称已对标热力图；建议列入 P1 或 V2.1 |
| 3D 模型/3D 数字 | 已有 3D 模型、3D 数字、视角控制 | XSENSOR 有 3D 视图 | Tekscan 有 3D 实时和记录数据 | 能打但需打磨 | 3D 模型不跟随翻转的规则要在 UI 上解释清楚，避免用户误解 |
| 坐垫/靠背分区 | 座椅类支持整体、靠背、坐垫切换 | XSENSOR 支持座椅和靠背压力测量 | Tekscan CONFORMat 面向座椅/靠背分析 | 能打 | 座椅/靠背独立方向、独立框选和独立统计要做成亮点 |
| 画布翻转/90 度旋转 | 支持左右/上下翻转，已补 90 度旋转和持久化 | 竞品公开资料不突出此项 | 竞品更多强调分析结果 | 可差异化 | 国内客户现场装配方向不确定，方向修正非常实用 |
| 置零 | 支持预压力置零 | 竞品更强调准确压力数据 | 竞品强调校准和准确压力 | 需要补强 | 置零不是校准，必须区分“预压力清零”和“传感器标定” |
| 框选/区域分析 | 支持最多 4 个框选，规划模板，实时和回放解耦 | XSENSOR 有对称性/标记/比较工具 | Tekscan 支持区域隔离分析 | 能打 | 框选应作为 JQ Tools 核心亮点，继续强化模板、命名、颜色、统计和回放复用 |
| 量尺 | 2D 模式按传感点距测量，最多 8 条 | 竞品不作为核心卖点 | Tekscan 支持两点测距 | 基本对标 | 补齐点距来源、单位换算、翻转/旋转后测距不变 |
| 统计指标 | 平均、最大、最小、总和、面积、点数、重心等 | XSENSOR 有统计和对称性 | Tekscan 有接触面积、平均/峰值、中心轨迹等 | 基本对标 | 补左右/前后均衡指数，明确公式和单位 |
| 趋势图表 | 压力曲线、面积曲线、重心轨迹、正态分布图 | XSENSOR 强调时序数据和分析 | Tekscan 有多图表 | 部分对标 | 正态分布 V2.0 不做时要从验收中移除，避免范围混乱 |
| 采集 | 开始/停止、命名、备注、频率、全量保存 | XSENSOR 支持记录 | Tekscan 支持记录和逐帧查看 | 能打 | 强化异常中断恢复、采集中禁止方向变化、全量矩阵不可丢 |
| 回放 | 播放、暂停、倍速、进度条、框选回放、图表同步 | XSENSOR 支持图片/视频记录 | Tekscan 支持逐帧查看 | 基本对标 | 增加帧号、时间戳、坏帧跳过、回放时框选模板复用 |
| 数据导入/导出 | CSV 导入校验、CSV 下载、路径选择、打开文件 | XSENSOR 强调可转移数据和图片 | Tekscan 支持 ASCII/AVI | 部分对标 | CSV 足够 P0，但需要版本号、字段映射、单位和导入兼容 |
| 数据对比 | 规划历史 A/B、差值图、指标差值 | XSENSOR 可比较多个座椅设计 | Tekscan 支持并排比较多个结果 | 方向正确但未成熟 | V2.0 先做历史 A/B 对比；V2.1 再做对比导出和曲线对比 |
| 报告 | 规划 V2.2 PDF/Word 报告 | XSENSOR 可生成报告 | Tekscan 强调文档和沟通 | 明显弱 | 科研/客户交付会需要报告，建议至少 V2.1 开始定义报告模板 |
| 标定/校准 | PRD 目前不够完整 | XSENSOR 强调精度、压力范围、无需频繁重校 | Tekscan 强调可用已知力校准 | 明显弱 | 必须补“校准”和“置零”的边界，否则数据可信度不足 |
| 硬件差异化 | 纤维压力传感器、自有适配传感器 | XSENSOR 是成熟压力传感器平台 | Tekscan 是成熟薄膜压力传感器平台 | 有机会 | 不要只讲软件，要把“纤维传感器 + 软件实时分析”一起包装 |

## 7. 能打功能

| 功能 | 为什么能打 | 对外表达方式 |
|---|---|---|
| 本地离线交付 | 国内展厅、门店、科研现场不一定允许联网，离线能力有实际价值 | 支持本地离线部署，设备授权和数据采集无需依赖云端 |
| 一键连接和重连 | 现场演示最怕连接卡死，稳定连接比复杂功能更重要 | 面向非技术人员的一键连接和异常恢复 |
| 框选独立分析 | 可在实时和回放中选择区域，不耦合采集，全量数据不丢 | 支持局部压力区域分析，适合座椅分区、靠背分区和对比分析 |
| 坐垫/靠背独立处理 | 汽车座椅不是单一平面，分区有实际业务价值 | 支持座椅整体、坐垫、靠背分区展示和统计 |
| 翻转/旋转持久化 | 现场安装方向经常变化，持久化能减少售后配置成本 | 支持传感器安装方向修正，配置保存后自动生效 |
| CSV 全量导出 | 科研和研发客户需要二次分析 | 支持本地 CSV 导出，保留设备、采样率、方向、置零等 metadata |
| 纤维压力传感器 | 如果硬件确有差异化，可形成国产替代核心卖点 | 纤维压力传感器 + JQ Tools 实时显示和数据分析一体化 |

## 8. 还弱功能

| 弱项 | 竞品表现 | 对 JQ Tools 的影响 | 优先级 |
|---|---|---|---|
| 无 2D 热力图/压力云图 | XSENSOR/Tekscan 都把压力图像作为核心体验 | 展厅演示和客户理解弱，难以“看一眼就懂” | P1，若展厅是核心场景可提前 |
| 校准/标定体系不清 | 竞品强调精度、重复性、压力范围、校准 | 科研客户会质疑数据可信度 | P0 |
| 报告能力弱 | XSENSOR 可生成报告，Tekscan 强调文档沟通 | 客户交付、科研留档、销售材料不足 | P1/P2 |
| 数据对比未完全闭环 | 竞品支持多结果对比、前后对比 | 缺少座椅改款前后验证能力 | P0 基础版，P1 增强版 |
| 统计公式和单位不完整 | 竞品指标表达清楚 | QA 无法验收，客户无法解释 | P0 |
| 多传感器同步表达不足 | 竞品支持多配置和多传感器系统 | 汽车座椅/床垫大面积场景扩展受限 | P1 |
| 图片/视频/报告导出不足 | 竞品支持图片、视频或文件化输出 | 展厅和销售演示资料弱 | P1/P2 |

## 9. 产品方向建议

| 层级 | 建议 | 原因 |
|---|---|---|
| 核心定位 | JQ Tools 应定位为“国产本地化压力感知分析系统”，不是单纯传感器调试工具 | 竞品强在完整系统，JQ Tools 也必须软件硬件一体表达 |
| V2.0 主线 | 稳定连接、数据可信、实时展示、框选分析、采集回放、基础对比、CSV 导出 | 这些能力决定产品是否可交付 |
| V2.1 主线 | 2D 压力云图、框选模板增强、对比导出、对比曲线、报告雏形 | 用来补齐竞品直观展示和交付输出差距 |

## 12. 风险提醒

| 风险 | 说明 | 处理建议 |
|---|---|---|
| 过早宣称“优于竞品” | 当前缺 2D 热力图、报告、校准说明，直接说优于风险较高 | 改为“可替代、部分场景更适合国内交付” |
| 数据可信度不足 | 如果单位、校准、阈值、公式不清，科研客户难验收 | 先补数据字典和验收公式 |
| 展厅视觉弱 | 没有热力图时，客户不一定能直观看懂数字矩阵 | 先优化 3D 和数字视图，P1 做 2D 压力云图 |
| 连接卡死 | 连接失败会导致所有能力无法使用 | 连接异常处理继续保持 P0 |
| 框选耦合采集 | 如果框选影响全量采集，会破坏历史回放和后续对比 | 坚持采集保存全量矩阵，框选只作为分析视角 |
