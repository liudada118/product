# 框选功能产品需求文档

文档日期：2026-05-09  
产品名称：压力感知软件 JQ Tools  
模块名称：框选工具  
文档用途：给产品、研发、算法、QA 使用，用于明确框选功能的业务逻辑、数据口径、异常处理和验收标准。  
本次优化重点：明确框选是独立分析模块，不和采集强耦合。框选可以在实时模式和回放模式下使用；采集只负责保存全量压力数据。

## 1. 文档说明

| 项目 | 内容 |
|---|---|
| 功能定位 | 框选是 JQ Tools 的核心分析工具，用于让用户从全量压力矩阵中选取局部区域，实时查看局部压力数据、局部曲线和局部统计结果。 |
| 产品价值 | 帮助用户从“看整体压力图”升级为“分析局部压力问题”。例如汽车座椅靠背肩部、腰部、坐垫左右侧、压力峰值区域等。 |
| 当前实现基础 | 已支持 2D 模式框选、最多 4 个框选区域、手动输入坐标、有效矩阵区域限制、颜色高亮、框选统计、实时框选分析、回放框选分析。 |
| 本文重点 | 把框选的状态、边界、坐标、数据统计、实时分析、回放分析、导出和异常处理单独定义清楚。 |
| 核心原则 | **框选不绑定采集。采集保存全量矩阵；框选只决定当前分析范围。** |
| 优先级 | P0。框选是当前产品和 XSENSOR/Tekscan 对标时的重要分析能力。 |

## 2. 为什么要单独拆框选模块

| 原因 | 说明 |
|---|---|
| 框选是分析亮点 | 用户不是只看热力图，而是要知道某个区域的平均压强、最大压强、受压面积、压力曲线等。 |
| 框选影响多个模块 | 框选会影响实时统计、图表、历史回放、CSV 导出、数据对比。不能只当成一个前端画框功能。 |
| 数据口径容易混乱 | 必须明确框选只改变“分析范围”，不能改变“采集保存范围”。 |
| 多框选规则需要清楚 | 最多 4 个框选区域时，每个区域是独立统计，还是合并统计，必须定义。 |
| QA 验收需要依据 | 没有独立 PRD 时，测试只能验证“能画框”，无法验证计算结果和回放一致性。 |

## 3. 模块边界

| 边界类型 | 包含 | 不包含 |
|---|---|---|
| 交互边界 | 开启框选、拖拽框选、手动输入坐标、选中区域、删除区域、清空区域、区域高亮。 | 传感器连接、设备授权、串口异常处理。 |
| 数据边界 | 根据框选范围截取矩阵、计算局部统计、生成局部曲线、保存框选元数据。 | 原始串口帧解析、设备矩阵转换算法。 |
| 视图边界 | 2D 数字视图下使用框选，区域边框和颜色可见。 | 3D 模型旋转、3D 点图高度渲染。 |
| 采集边界 | 采集只保存全量矩阵、设备信息、时间戳、方向等采集数据。框选不是采集任务的必要条件。 | 只保存框选矩阵并丢弃全量数据。 |
| 回放边界 | 回放时用户可以重新创建框选，并基于历史全量矩阵实时计算框选数据。 | 依赖采集时保存的框选结果，导致回放不能重新分析。 |

## 4. 用户场景

| 用户角色 | 场景 | 用户目标 | 框选价值 |
|---|---|---|---|
| 汽车座椅研发工程师 | 查看坐垫左右区域压力 | 判断左右受力是否均衡 | 分别框选左右区域，对比平均压强、压力总和和面积。 |
| 汽车座椅研发工程师 | 查看靠背腰托区域 | 判断腰部支撑是否过强或不足 | 框选靠背局部区域，观察最大值和压力曲线。 |
| 科研客户 | 对实验区域做定量分析 | 导出可复核的局部数据 | 保存框选区域和全量矩阵，后续回放和导出。 |
| 展厅演示人员 | 向客户展示局部压力变化 | 让客户直观看到区域差异 | 框选局部区域，图表实时变化。 |
| QA | 验证框选数据准确性 | 确认区域、坐标、统计和回放一致 | 使用固定矩阵验证计算结果。 |

## 5. 功能目标

| 目标编号 | 目标 | 衡量标准 |
|---|---|---|
| 目标-01 | 用户可以快速选择压力矩阵中的局部区域 | 在 2D 数字视图下，用户可通过拖拽创建矩形区域。 |
| 目标-02 | 用户可以精确输入框选坐标 | 支持 X、Y、长、宽输入，并有边界校验。 |
| 目标-03 | 用户可以同时分析多个区域 | 最多支持 4 个框选区域，每个区域颜色不同。 |
| 目标-04 | 框选统计实时更新 | 压力变化时，局部平均值、最大值、最小值、压力总和、面积、点数同步更新。 |
| 目标-05 | 框选不破坏原始数据 | 采集、回放、导出必须保留全量矩阵。 |
| 目标-06 | 框选可在实时和回放中独立使用 | 用户不需要先采集框选，也可以在历史回放里重新框选分析。 |
| 目标-07 | 框选结果可导出、可复核 | 导出时可选择导出当前框选分析结果，但不能替代全量数据。 |

## 6.1 框选与采集解耦原则

| 原则编号 | 原则 | 说明 | 优先级 |
|---|---|---|---|
| 原则-01 | 框选是分析模块 | 框选的职责是选择分析范围，输出局部统计、局部曲线和局部导出结果。 | P0 |
| 原则-02 | 采集是数据保存模块 | 采集的职责是保存全量压力矩阵、时间戳、设备信息、采样率、方向等数据。 | P0 |
| 原则-03 | 框选不决定采集范围 | 无论是否框选，采集都必须保存全量矩阵。 | P0 |
| 原则-04 | 实时和回放都可框选 | 用户可以在实时模式下框选，也可以在历史回放模式下重新框选。 | P0 |
| 原则-05 | 回放框选不依赖采集时框选 | 历史回放只要有全量矩阵，就可以新建框选并重新计算区域数据。 | P0 |
| 原则-06 | 框选结果是派生数据 | 框选统计、框选曲线、框选导出都属于派生分析结果，不是原始采集数据。 | P0 |
| 原则-07 | 框选方案可选保存 | 如果用户需要复用同一组区域，可以保存为分析方案，但不影响采集数据完整性。 | P1 |

## 6.2 框选模块输入输出

| 类型 | 内容 | 来源/去向 | 说明 |
|---|---|---|---|
| 输入 | 实时全量矩阵 | 实时数据处理模块 | 实时模式下用于计算当前帧框选统计。 |
| 输入 | 历史全量矩阵 | 回放模块 | 回放模式下用于按历史帧重算框选统计。 |
| 输入 | 框选区域 | 用户拖拽或手动输入 | 包含 X、Y、长、宽、区域编号、颜色等。 |
| 输出 | 框选统计 | 数据统计模块/前端面板 | 平均、最大、最小、总和、面积、点数等。 |
| 输出 | 框选曲线 | 图表模块 | 压力总和曲线、面积曲线等局部曲线。 |
| 输出 | 框选导出结果 | CSV 导出模块 | 用户选择导出分析结果时生成。 |
| 不输出 | 采集范围 | 采集模块 | 框选不向采集模块输出“只采框选区域”的指令。 |

## 6.3 框选模板定义

框选模板是一组可复用的框选区域配置。它用于让用户快速恢复常用分析区域，例如“坐垫左侧/右侧”“靠背肩部/腰部”“前排/后排压力区”。

框选模板不是采集数据，不保存压力矩阵，不保存统计结果。模板只保存“要分析哪些区域”。

| 项目 | 规则 | 优先级 |
|---|---|---|
| 模板定位 | 一组框选区域的配置集合。 | P1 |
| 模板用途 | 在实时、回放、数据对比中快速应用同一组区域。 | P1 |
| 模板内容 | 区域名称、坐标、长宽、颜色、设备类型、矩阵尺寸、显示对象、创建时间等。 | P1 |
| 不保存内容 | 不保存 rawData、displayData、calculatedData、采集帧、统计结果。 | P0 |
| 与采集关系 | 模板不影响采集开始、停止和采集保存范围。 | P0 |
| 与历史关系 | 历史记录不强制绑定模板。用户可以在回放时临时应用任意匹配模板。 | P1 |
| 与对比关系 | 对比模式中可应用模板，同一模板区域同时作用于 A/B 和差值图。 | P1 |
| 存储方式 | 建议本地存储，跟随软件本机使用。后续可扩展导入/导出模板。 | P1 |

## 7. 框选状态机

| 状态 | 进入条件 | 用户可操作 | 系统处理 | 退出条件 |
|---|---|---|---|---|
| 未开启 | 默认状态，或用户关闭框选工具 | 开启框选 | 不显示新建框选交互 | 点击框选按钮 |
| 框选准备中 | 用户点击框选工具 | 拖拽创建区域、手动输入区域 | 显示可框选状态 | 创建区域或关闭工具 |
| 正在拖拽 | 鼠标按下并移动 | 松开鼠标 | 实时显示临时矩形 | 鼠标松开 |
| 已创建 | 拖拽结束或输入坐标成功 | 选中、删除、继续新增 | 保存区域元数据并触发统计 | 删除或清空 |
| 已选中 | 用户点击某个框选区域 | 删除、查看数据、编辑坐标 | 高亮当前区域 | 切换选中或取消 |
| 已满 | 已有 4 个区域 | 删除、清空 | 禁止继续新增 | 删除至少 1 个区域 |
| 异常 | 超出边界、尺寸为 0、视图不支持 | 修改输入或切换视图 | 提示原因，不创建区域 | 用户修正 |

## 8. 主流程

| 步骤 | 用户动作 | 前置条件 | 系统处理 | 成功结果 | 异常处理 |
|---|---|---|---|---|---|
| 1 | 点击框选工具 | 当前为 2D 数字视图 | 开启框选模式 | 鼠标可拖拽创建区域 | 非 2D 数字视图时提示“请在 2D 数字视图下使用框选” |
| 2 | 鼠标拖拽 | 框选模式已开启 | 记录起点和终点 | 生成矩形区域 | 起点或终点超出有效区域时提示 |
| 3 | 创建区域 | 拖拽范围有效 | 转换为矩阵坐标 | 区域显示在画布上 | 宽或高为 0 时不创建 |
| 4 | 更新统计 | 已存在框选区域 | 从当前矩阵截取区域数据 | 指标和图表按框选区域刷新 | 当前无有效数据时显示空状态 |
| 5 | 新增多个区域 | 已有区域少于 4 个 | 分配颜色和编号 | 多个区域同时显示 | 已有 4 个时提示“最多支持 4 个框选区域” |
| 6 | 手动输入坐标 | 用户打开区域设置 | 校验 X、Y、长、宽 | 创建或更新区域 | 输入为空、负数、越界时提示 |
| 7 | 删除区域 | 用户选中区域 | 删除区域元数据和 DOM | 统计恢复或更新 | 删除失败时保持原状态 |
| 8 | 清空区域 | 用户点击清空 | 至少存在一个区域 | 删除全部区域 | 恢复全量统计 | 无区域时按钮置灰 |

## 9. 使用条件和互斥规则

| 规则 | 说明 | 优先级 |
|---|---|---|
| 只支持 2D 数字视图 | 框选需要基于可见矩阵点位，当前只在 2D 数字视图下开放。 | P0 |
| 与量尺互斥 | 量尺和框选都依赖鼠标拖拽，不能同时开启。 | P0 |
| 与放大镜关系 | 放大镜可以查看点位，但拖拽框选时建议临时关闭放大镜。 | P1 |
| 与翻转关系 | 画布翻转后，框选坐标必须和当前显示方向一致。 | P0 |
| 与置零关系 | 置零影响计算值，框选统计应基于置零后的 calculatedData。 | P0 |
| 与可视化调节关系 | 颜色、润滑、高度等可视化调节不应改变框选原始统计口径。 | P0 |

## 10. 坐标规则

| 项目 | 规则 |
|---|---|
| 坐标原点 | 默认以当前 2D 数字矩阵左上角为原点。 |
| X | 从左到右递增，表示列坐标。 |
| Y | 从上到下递增，表示行坐标。 |
| 长 | 横向覆盖的传感点数量。 |
| 宽 | 纵向覆盖的传感点数量。 |
| 最小尺寸 | 长和宽都必须大于 0。 |
| 最大范围 | X + 长不得超过矩阵列数，Y + 宽不得超过矩阵行数。 |
| 坐标单位 | 传感点，不是像素，也不是毫米。 |
| DOM 坐标 | 只用于画布展示，保存和计算必须使用矩阵坐标。 |
| 翻转后坐标 | 保存当前显示方向的坐标。框选展示、统计和导出均以用户当前看到的方向为准。 |

## 12.1 框选模板业务逻辑

### 12.1.1 模板主流程

| 步骤 | 用户动作 | 前置条件 | 系统处理 | 成功结果 | 异常处理 |
|---|---|---|---|---|---|
| 1 | 创建框选区域 | 当前为 2D 数字视图，且区域合法 | 生成当前框选区域列表 | 画布上出现区域 | 越界、尺寸为 0、数量超过 4 时不创建 |
| 2 | 点击保存为模板 | 至少存在 1 个框选区域 | 弹出模板保存窗口 | 用户可填写模板名称、备注 | 无区域时按钮置灰 |
| 3 | 填写模板信息 | 模板名称不为空 | 校验名称、设备类型、矩阵尺寸 | 模板可保存 | 名称为空或重复时提示 |
| 4 | 保存模板 | 校验通过 | 保存区域配置和适用范围 | 模板进入模板列表 | 保存失败时提示，不影响当前框选 |

### 12.1.2 模板适用范围

推荐方案：模板必须匹配设备类型、显示对象和矩阵尺寸。这样最稳，避免把坐垫区域错误套到靠背，或把 32 x 32 模板套到 46 x 46。

| 适用字段 | 是否必须匹配 | 说明 | 优先级 |
|---|---|---|---|
| 设备类型 | 是 | 例如汽车坐垫、汽车靠背、endi 坐垫、endi 靠背。 | P1 |
| 矩阵尺寸 | 是 | 例如 32 x 32、46 x 46、50 x 64、64 x 64。 | P1 |
| 显示对象 | 是 | 坐垫模板只能用于坐垫，靠背模板只能用于靠背。 | P1 |
| 显示方向 | 建议匹配 | 当前已确认保存显示方向坐标，应用时应按当前显示方向恢复。 | P1 |
| 置零状态 | 否 | 模板只保存区域，不关心置零。 | P1 |
| 采集状态 | 否 | 模板可在未采集、采集中、回放中应用，但只影响分析。 | P1 |

### 12.1.3 模板匹配方案

| 方案 | 逻辑 | 优点 | 缺点 | 推荐 |
|---|---|---|---|---|
| 方案 A：严格匹配 | 设备类型、显示对象、矩阵尺寸全部一致才可应用。 | 最安全，数据解释清楚。 | 灵活性较低。 | 推荐 V2.0 |
| 方案 B：同尺寸可应用 | 只要矩阵尺寸一致就可应用。 | 灵活。 | 容易把业务含义不同的区域套错。 | 不推荐 |
| 方案 C：自动缩放适配 | 不同尺寸按比例缩放区域。 | 复用能力强。 | 坐标误差和解释风险大。 | P2 |

## 16. 框选与采集

### 16.1 解耦后的业务结论

| 结论 | 说明 | 优先级 |

| 框选不控制采集范围 | 点击采集后，系统始终保存全量矩阵。 | P0 |

## 23. 研发实现关注点

| 关注点 | 说明 | 优先级 |
|---|---|---|
| DOM 坐标和矩阵坐标分离 | 画布缩放、翻转后，DOM 坐标不能直接作为计算坐标。 | P0 |
| 框选数据不能污染 rawData | 框选只改变统计范围，不改变原始矩阵。 | P0 |
| 历史回放必须可重算 | 回放时不能只依赖实时计算缓存。 | P0 |
| 多框选图表性能 | 多区域实时曲线可能增加计算量，需要控制每帧计算开销。 | P1 |
| 区域删除后的订阅清理 | 删除区域后必须清理对应图表、缓存和监听，避免内存增长。 | P0 |
| 切换系统类型时清理框选 | 设备矩阵尺寸变化时，旧框选可能越界，必须清空或重新校验。 | P0 |
| 切换靠背/坐垫时处理框选 | 靠背和坐垫矩阵尺寸不同，框选需要绑定 displayType。 | P0 |
| 采集中修改框选 | 允许修改，但只影响当前分析状态，不影响采集任务和已保存全量矩阵。 | P0 |
