# 压力感知软件测试场景（test-scenarios版-基于PRD V1.6）

文档日期：2026-05-11  
产品名称：压力感知软件 JQ Tools  
产品版本：V2.0  
来源文档：`压力感知软件产品需求文档-飞书表格版-含数据处理模块.md` V1.6  
使用技能：`test-scenarios`  
文档用途：给 QA 按场景执行测试，也可直接复制到飞书表格  

## 1. 测试设计原则

| 原则 | 说明 |
|---|---|
| 覆盖主链路 | 从打开软件、连接设备、实时展示、工具分析、采集、回放、导出、对比形成闭环 |
| 优先验证 P0 | 连接、数据处理、采集、框选、CSV、异常恢复是发布阻塞项 |
| 每条用例可执行 | 每条用例都包含测试目标、前置条件、测试步骤和预期结果 |
| 数据口径可追溯 | 重点验证 rawData、displayData、calculatedData、storedData、exportedData 的影响关系 |
| 异常场景必须覆盖 | 一键连接卡死、坏帧、矩阵错误、采集中断、CSV 错误、对比不可比都需要测试 |

## 2. 测试数据准备

| 准备项 | 用途 | 必要性 |
|---|---|---|
| 合法汽车坐垫设备 | 连接、展示、采集、框选、量尺、CSV 导出 | 必须 |
| 合法汽车靠背设备 | 座椅靠背、坐垫/靠背切换、方向独立测试 | 必须 |
| 未授权或未配置 MAC 设备 | 授权失败、采集拦截测试 | 必须 |
| 正常压力帧样例 | 实时展示、统计、回放和导出 | 必须 |
| 坏帧、断帧、长度不匹配样例 | 数据异常处理测试 | 必须 |
| 超过 30 帧的历史采集记录 | 图表缓存、回放完整帧测试 | 必须 |
| 两条同类型历史记录 | 数据对比 A/B 测试 | 必须 |
| 结构正确 CSV、legacy CSV、错误 CSV | CSV 导入和兼容测试 | 必须 |
