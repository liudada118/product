## 1. 使用说明

| 项目 | 内容 |
|---|---|
| 用例编号 | 按模块编号，例如 `TC-CON-001`、`TC-DP-001`、`TC-SEL-001` |
| 场景类型 | 正向、异常、边界、数据口径、性能、兼容、UI 容器 |
| 优先级 | P0 必测；P1 建议本轮覆盖；P2 后续覆盖 |
| 是否阻塞发布 | 是表示未通过不建议发布 V2.0 |
| 状态 | 默认未执行，可在飞书中改为未执行、通过、失败、阻塞、待确认 |
| 负责人 | 复制到飞书后由 QA、研发、产品或交付补充 |
| 执行原则 | P0 用例必须尽量使用真实设备验证；算法和数据口径类用例可配合固定矩阵数据验证 |

## 2. 测试环境和通用前置条件

| 项目 | 要求 |
|---|---|
| 软件版本 | JQ Tools V2.0 待测版本 |
| PRD 口径 | 以 PRD V1.8 为准 |
| 操作系统 | Windows 必测；macOS 如本轮发布则单独覆盖 |
| 测试设备 | 至少准备汽车座椅坐垫、汽车座椅靠背；如条件允许覆盖床垫、手/坐垫、脚部、4096 点阵 |
| 异常设备 | 至少准备未配置 MAC、类型不匹配、无法读取 MAC 或串口占用场景 |
| 串口芯片 | CH340 |
| 波特率 | 921600、1000000、3000000 |
| 数据准备 | 正常历史数据、损坏帧历史数据、合法 CSV、legacy CSV、非法 CSV、A/B 对比数据 |
| UI 环境 | 常规窗口、小屏或低分辨率窗口，用于验证统一分析侧栏降级 |
| 注意事项 | 当前 V2.0 不包含 2D 热力图，不应作为本轮验收项 |
