# 一键连接与重新连接异常处理产品需求文档

文档日期：2026-05-09  
所属产品：压力感知软件 JQ Tools  
所属版本：V2.0  
所属模块：打开软件后的连接流程  
文档用途：补齐一键连接、重新连接、断开连接、异常恢复、防卡死的业务逻辑和验收标准

## 1. 文档说明

### 1.1 背景

一键连接和重新连接是压力感知软件的入口级功能。这个模块如果卡死，后面的实时展示、数据处理、采集、回放和导出都无法使用。

当前实现文档已经说明：

| 已有实现 | 说明 |
|---|---|
| `/connPort` | 一键连接 |
| `/rescanPort` | 清理死端口/僵尸设备并重连 |
| `/stopPort` | 断开所有串口 |
| `detectBaudRate(portPath)` | 自动探测波特率 |
| `newSerialPortLinkWithRetry()` | 稳定连接和重试 |
| `connectState` | 前端连接状态：空闲、连接中、已连接、重连中 |

但当前异常处理不够细，尤其是：

1. 连接中卡死。
2. 重复点击导致多次连接。
3. 串口被占用。
4. 临时串口没有释放。
5. 波特率探测卡住。
6. MAC 读取失败后状态不回收。
7. 重连失败后整体不可用。
8. 采集中设备断开后数据和状态不清楚。

### 1.2 模块目标

| 目标 | 说明 |
|---|---|
| 防止连接卡死 | 任意连接流程必须有超时和可恢复状态 |
| 防止重复连接 | 连接中、重连中不能重复触发连接 |
| 防止串口资源泄露 | 失败、超时、取消、重连前必须释放旧串口 |
| 明确错误原因 | 用户能知道是未插设备、设备占用、MAC 失败、波特率失败还是授权失败 |
| 保证主流程可恢复 | 连接失败后用户可以重新连接，不需要重启软件 |
| 保护采集数据 | 采集中断开设备时，已采集数据不能丢失 |

## 2. 角色和场景

| 用户角色 | 使用场景 | 关注点 |
|---|---|---|
| 展厅演示人员 | 现场一键连接设备 | 不能卡死，失败要能快速重试 |
| 研发测试工程师 | 高频切换设备、调试串口 | 错误原因要清楚 |
| 交付/售后人员 | 客户现场安装和排查 | 能判断是硬件、线缆、串口、配置还是软件问题 |
| QA | 验证连接稳定性 | 需要覆盖超时、失败、重试、断开、恢复 |

## 3. 连接状态机

### 3.1 状态定义

| 状态 | 状态说明 | 可执行操作 | 禁止操作 | 退出条件 |
|---|---|---|---|---|
| 未连接 | 软件已启动，没有有效串口连接 | 一键连接、设备管理、历史查看 | 采集 | 点击一键连接 |
| 连接中 | 正在扫描串口、探测波特率或读取 MAC | 取消连接（如支持）、等待 | 重复一键连接、采集、切换系统类型 | 连接成功或失败 |
| 已连接 | 设备连接成功，MAC 和类型已识别 | 采集、断开、重新连接、查看数据 | 重复一键连接 | 断开、设备异常、重新连接 |
| 重连中 | 正在清理旧连接并重新连接 | 等待、取消重连（如支持） | 采集、重复重连、系统切换 | 重连成功或失败 |
| 连接失败 | 本次连接流程失败 | 重新连接、查看错误原因 | 采集 | 用户重新连接或断开 |
| 设备异常 | 连接后设备断开、数据中断或连续坏帧 | 重新连接、断开、保留数据 | 开始新采集 | 重连成功或断开 |
| 采集中断连 | 采集中设备断开 | 停止采集、重新连接、保存已有数据 | 删除当前采集缓存、切换系统类型 | 停止采集或重连成功 |

### 3.2 状态流转

| 当前状态 | 触发动作/事件 | 下一个状态 | 处理规则 |
|---|---|---|---|
| 未连接 | 点击一键连接 | 连接中 | 锁定连接按钮，开始扫描串口 |
| 连接中 | 连接成功 | 已连接 | 展示设备状态和参数 |
| 连接中 | 连接失败 | 连接失败 | 释放所有临时资源，显示失败原因 |
| 连接中 | 连接超时 | 连接失败 | 自动终止流程，不允许一直等待 |
| 已连接 | 点击断开 | 未连接 | 关闭串口、清空实时状态 |
| 已连接 | 点击重新连接 | 重连中 | 先断开旧连接，再重新扫描 |
| 已连接 | 设备断开 | 设备异常 | 停止实时数据推送，提示断开 |
| 重连中 | 重连成功 | 已连接 | 恢复实时数据 |
| 重连中 | 重连失败 | 连接失败 | 显示失败原因，允许再次重连 |
| 采集中 | 设备断开 | 采集中断连 | 保留已采集数据，提示用户处理 |
| 采集中断连 | 重新连接成功 | 采集中或已连接 | 需确认是否继续采集，默认建议停止当前采集 |

## 4. 一键连接业务逻辑

### 4.1 主流程

| 步骤 | 处理动作 | 成功结果 | 失败处理 |
|---|---|---|---|
| 1 | 用户点击一键连接 | 状态变为连接中 | 如果已连接/连接中，则拒绝重复触发 |
| 2 | 前端调用 `/connPort` | 后端开始连接流程 | 请求失败时恢复为未连接 |
| 3 | 后端检查连接锁 | 获得连接锁 | 如果已有连接任务，返回“连接中，请稍候” |
| 4 | 清理历史残留连接 | 旧串口、旧 parser、旧 timer 被释放 | 清理失败也要进入下一步，但记录日志 |
| 5 | 扫描可用串口 | 获取端口列表 | 无端口时返回“未检测到设备” |
| 6 | 过滤 CH340 设备 | 得到候选端口 | 无 CH340 时返回“未检测到 CH340 设备” |
| 7 | 逐个端口探测波特率 | 找到匹配波特率 | 全部失败时返回“波特率探测失败” |
| 8 | 建立稳定串口连接 | 串口 open 成功 | open 失败时重试，超过次数返回失败 |
| 9 | 读取 MAC | 得到 MAC | MAC 失败时关闭串口并返回失败 |
| 10 | 解析设备类型 | 得到设备类型 | 类型未知时提示配置设备 |
| 11 | 校验授权/类型 | 校验通过 | 校验失败时禁止采集 |
| 12 | 绑定数据监听 | 开始接收数据 | 绑定失败时关闭串口 |
| 13 | 启动实时推送 | 前端收到实时数据 | 推送失败时提示实时连接异常 |
| 14 | 返回连接成功 | 状态变为已连接 | 无 |

### 4.2 连接锁规则

| 规则 | 说明 |
|---|---|
| 同一时间只允许一个连接任务 | 防止重复点击造成多个串口实例 |
| 连接中再次点击一键连接 | 前端禁用按钮，后端也返回“连接中，请稍候” |
| 重连中再次点击重新连接 | 忽略或提示“正在重连” |
| 连接流程结束必须释放锁 | 无论成功、失败、超时、异常都必须释放 |
| 连接锁需要最大存活时间 | 如果连接任务异常卡住，超过最大时间自动释放 |

### 4.3 超时规则

| 环节 | 建议超时时间 | 超时后处理 |
|---|---|---|
| 前端请求 `/connPort` | 15 秒 | 提示连接超时，允许重试 |
| 串口扫描 | 3 秒 | 返回未检测到设备 |
| 单个波特率探测 | 2 秒 | 关闭临时串口，尝试下一个波特率 |
| 单个端口总探测 | 8 秒 | 关闭端口，尝试下一个端口 |
| 稳定连接 open | 2 秒 | 重试，最多 3 次 |
| MAC 读取 | 3 秒 | 关闭串口，返回 MAC 读取失败 |
| 设备类型解析 | 2 秒 | 本地未命中时提示配置设备 |
| 授权校验 | 2 秒 | 校验失败或超时，默认禁止采集 |
| 整体一键连接 | 20 秒 | 强制结束连接流程，释放所有资源 |

## 5. 重新连接业务逻辑

### 5.1 触发场景

| 场景 | 是否允许重新连接 | 处理方式 |
|---|---|---|
| 未连接 | 允许 | 等同一键连接 |
| 连接失败 | 允许 | 清理失败状态后重新连接 |
| 已连接 | 允许 | 先断开旧连接，再重新连接 |
| 设备异常 | 允许 | 清理异常连接，再重新连接 |
| 连接中 | 不允许 | 提示正在连接 |
| 重连中 | 不允许 | 提示正在重连 |
| 采集中 | 谨慎允许 | 必须先提示风险，建议先停止采集 |
| 采集中断连 | 允许 | 重连成功后提示是否继续或结束采集 |

### 5.2 重新连接主流程

| 步骤 | 处理动作 | 成功结果 | 失败处理 |
|---|---|---|---|
| 1 | 用户点击重新连接 | 状态变为重连中 | 若连接中则拒绝 |
| 2 | 后端调用 `/rescanPort` | 开始清理旧资源 | 失败也继续后续清理 |
| 3 | 停止实时推送 timer | 旧推送停止 | 记录清理失败日志 |
| 4 | 移除旧 parser 和 data listener | 防止重复监听 | 无 |
| 5 | 关闭旧串口 | 释放串口占用 | 若关闭失败，标记为疑似僵尸端口 |
| 6 | 清理后端状态 | 清空 dataMap、macInfo、parserArr 等连接状态 | 采集状态不得误清 |
| 7 | 重新扫描端口 | 得到候选端口 | 无设备则失败 |
| 8 | 重新执行一键连接流程 | 连接成功 | 失败后进入连接失败状态 |
| 9 | 前端刷新连接状态 | 状态变为已连接 | 失败时展示原因 |

### 5.3 采集中重连规则

| 问题 | 推荐规则 |
|---|---|
| 采集中设备断开后是否继续采集 | 默认暂停或结束当前采集，不建议静默继续 |
| 重连成功后是否接着写入同一条历史 | 需要产品确认；建议 V2.0 默认结束旧采集，生成异常结束记录 |
| 已采集数据如何处理 | 必须保留，不能因为重连失败丢失 |
| 历史记录状态 | 建议增加 `interrupted` 或“异常中断”状态 |
| 用户提示 | “设备已断开，已保留当前采集数据。请重新连接或结束采集。” |

## 6. 断开连接业务逻辑

| 场景 | 前置条件 | 用户动作 | 系统处理 | 结果 |
|---|---|---|---|---|
| 未采集时断开 | 已连接，未采集 | 点击断开 | 关闭串口、停止推送、清理连接状态 | 状态变为未连接 |
| 采集中断开 | 正在采集 | 点击断开 | 弹窗确认风险 | 用户确认后停止采集或标记中断 |
| 设备已异常 | 设备断开或数据中断 | 点击断开 | 清理残留串口和状态 | 状态变为未连接 |
| 重连前自动断开 | 已连接 | 点击重新连接 | 自动清理旧连接 | 进入重连中 |

### 6.1 断开前确认规则

| 状态 | 是否需要确认 | 文案建议 |
|---|---|---|
| 未连接 | 不需要 | 无 |
| 已连接未采集 | 不需要或轻提示 | “已断开设备连接” |
| 采集中 | 必须确认 | “当前正在采集，断开连接会中断采集，已采集数据将保留。是否继续？” |
| 导出中 | 建议禁止断开 | “正在导出，请稍后再断开设备” |
| 回放中 | 不需要 | 回放使用历史数据，不依赖实时设备 |

## 7. 异常类型和处理规则

### 7.1 连接前异常

| 异常 | 判断条件 | 系统处理 | 用户提示 | 是否阻塞采集 |
|---|---|---|---|---|
| 无设备配置 | 本地缓存为空 | 跳转 MAC 配置页 | “请先配置设备 MAC 和类型” | 是 |
| 配置格式错误 | MAC 或类型解析失败 | 拒绝保存 | “设备配置格式有误” | 是 |
| 重复 MAC | 同一 MAC 重复录入 | 拒绝或覆盖前确认 | “该 MAC 已存在” | 是 |
| 类型不支持 | 类型不在支持列表 | 拒绝保存 | “设备类型不支持” | 是 |

### 7.2 连接中异常

| 异常 | 判断条件 | 系统处理 | 用户提示 | 是否允许重试 |
|---|---|---|---|---|
| 未检测到串口 | 端口列表为空 | 结束连接流程 | “未检测到设备，请检查 USB 连接” | 是 |
| 未检测到 CH340 | 无匹配芯片端口 | 结束连接流程 | “未检测到 CH340 设备” | 是 |
| 串口被占用 | open 失败且提示 busy/access denied | 关闭可释放资源 | “串口被占用，请关闭其他软件后重试” | 是 |
| 波特率探测失败 | 候选波特率均失败 | 关闭临时串口 | “波特率探测失败，请检查设备或线缆” | 是 |
| 稳定连接失败 | open 重试 3 次失败 | 释放端口 | “设备连接失败，请重新插拔后重试” | 是 |
| MAC 读取失败 | 超时或返回为空 | 关闭串口 | “MAC 读取失败，无法识别设备” | 是 |
| 设备类型未知 | 本地无映射且设备未返回 | 保持未连接或连接受限 | “设备类型未知，请先配置设备” | 是 |
| 授权失败 | 校验不通过 | 禁止采集 | “设备未授权，无法采集” | 否或需配置后重试 |
| 连接整体超时 | 超过整体连接时间 | 强制清理 | “连接超时，请重试” | 是 |

### 7.3 连接后异常

| 异常 | 判断条件 | 系统处理 | 用户提示 | 是否阻塞采集 |
|---|---|---|---|---|
| 设备断开 | 串口 close/error 或长时间无数据 | 状态变设备异常 | “设备连接已断开” | 是 |
| 连续空帧 | 连续无有效数据 | 标记数据异常 | “设备暂无有效数据” | 视情况 |
| 连续坏帧 | 坏帧超过阈值 | 暂停有效推送 | “设备数据异常，请检查连接” | 是 |
| WebSocket 断开 | 前端实时通道断开 | 尝试重连或提示 | “实时数据连接异常” | 不一定 |
| 采集中断连 | 采集中设备断开 | 保留数据，标记异常 | “采集中断，已保留当前数据” | 是 |

## 9. 后端资源释放规则

| 资源 | 释放时机 | 释放要求 |
|---|---|---|
| 临时探测串口 | 每个波特率探测结束后 | 必须 close，防止端口占用 |
| 稳定连接串口 | 断开、重连、失败、异常时 | 必须 close 并移除引用 |
| parser | 断开、重连、失败、异常时 | 移除监听，避免重复解析 |
| data listener | 断开、重连、失败、异常时 | 移除旧监听，避免重复推送 |
| timer | 断开、重连、失败、异常时 | 清理实时推送定时器 |
| dataMap | 重连或断开时 | 清理旧设备实时数据 |
| macInfo | 重连或断开时 | 清理旧 MAC 信息 |
| connection lock | 成功、失败、超时、异常时 | 必须释放 |

## 11. API 行为要求

### 11.1 `/connPort`

| 项目 | 要求 |
|---|---|
| 用途 | 一键连接 |
| 前置条件 | 当前不在连接中或重连中 |
| 成功返回 | 连接成功、设备信息、MAC、类型、波特率、采样率 |
| 失败返回 | 错误码、错误阶段、用户可读文案 |
| 超时要求 | 整体不超过 20 秒 |
| 幂等要求 | 连接中重复请求不能启动第二个连接任务 |

### 11.2 `/rescanPort`

| 项目 | 要求 |
|---|---|
| 用途 | 清理旧连接并重新连接 |
| 前置条件 | 当前不在连接中或重连中 |
| 成功返回 | 重连成功和新设备信息 |
| 失败返回 | 清理阶段或连接阶段失败原因 |
| 特殊要求 | 必须先释放旧串口、parser、listener、timer |
| 采集状态 | 采集中调用需要弹窗确认或后端拒绝 |

### 11.3 `/stopPort`

| 项目 | 要求 |
|---|---|
| 用途 | 断开所有串口 |
| 前置条件 | 任意连接状态 |
| 成功返回 | 已断开 |
| 特殊要求 | 采集中调用需要确认，并保留已采集数据 |
| 资源释放 | 必须清理串口、parser、listener、timer |

## 14. 发布准入

| 准入项 | 通过标准 | 是否阻塞 |
|---|---|---|
| 连接不会卡死 | 任意失败场景都能在超时后恢复到可操作状态 | 是 |
| 连接互斥 | 重复点击不会产生多个连接任务 | 是 |
| 资源释放 | 失败、超时、重连、断开后串口资源被释放 | 是 |
| 错误提示 | 主要失败原因都有用户可读提示 | 是 |
| 重连可恢复 | 重连失败后仍可再次重试，不需要重启软件 | 是 |
| 采集中断连保护 | 已采集数据不会丢失 | 是 |
| 非法设备禁止采集 | 未授权、类型未知、类型不匹配均不能采集 | 是 |
