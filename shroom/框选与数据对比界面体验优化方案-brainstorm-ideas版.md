# 框选与数据对比界面体验优化方案（brainstorm-ideas版）

文档日期：2026-05-11  
产品名称：压力感知软件 JQ Tools  
来源文档：`压力感知软件产品需求文档-飞书表格版-含数据处理模块.md` V1.6  
使用技能：`brainstorm-ideas-existing`  
优化范围：框选功能、数据对比功能  

## 1. 机会判断

| 项目 | 内容 |
|---|---|
| 当前问题 | 框选和数据对比已经有业务逻辑，但界面偏工具化、信息层级弱，用户不容易快速理解“当前分析范围、当前对比对象、当前指标变化” |
| 目标用户 | 汽车座椅研发、科研客户、展厅演示人员、内部 QA |
| 期望结果 | 让用户更快完成区域选择、局部分析、A/B 对比、差值判断和结果导出 |
| 设计原则 | 不做花哨大改，优先提升可读性、状态清晰度和分析效率 |
| 研发原则 | 尽量复用现有 2D 数字视图、框选数据结构、统计指标、历史回放和 CSV 导出能力 |

## 2. Opportunity Solution Tree

| 机会 | 解决方向 | 可验证实验 |
|---|---|---|
| 用户不知道当前分析的是全量还是框选 | 增加“分析范围状态条”和框选区域列表 | 用 1 版低保真原型让内部研发/QA 执行框选统计任务，看是否能说清当前范围 |
| 框选创建后缺少管理感 | 做框选区域面板：名称、颜色、坐标、指标、显示/隐藏、删除 | 对比旧版和新版完成“创建 3 个区域并导出”的耗时 |
| A/B 对比信息分散 | 做三栏对比布局：A 数据、差值图、B 数据 | 让用户判断“哪组压力更大、差在哪里”，记录判断时间 |
| 差值图解释成本高 | 加图例、正负差值说明、当前帧指标摘要 | 让非研发用户解释红蓝含义，检查是否误读 |
| 数据对比和框选割裂 | 在对比页复用框选，并显示区域差值指标 | 测试用户是否能完成“只比较腰部区域压力变化” |

## 3. PM 视角：产品价值和业务闭环

| 编号 | 想法 | 说明 | 价值 | 复杂度 |
|---|---|---|---|---|
| PM-01 | 分析范围状态条 | 在主视图区上方显示“当前分析：全量 / 框选 1 / 多框选 3 个 / 回放框选” | 降低误读，提升数据可信度 | 低 |
| PM-02 | 框选区域管理面板 | 用右侧面板管理每个框选区域的名称、颜色、坐标、统计指标和导出状态 | 把框选从“画个框”升级成“区域分析工具” | 中 |
| PM-03 | 数据对比三步流程 | 选择基准数据 A、选择对比数据 B、进入对比分析 | 降低对比入口复杂度 | 中 |
| PM-04 | 对比结论摘要 | 自动生成当前帧结论：B 总压力高/低多少、面积变化多少、最大压力点移动到哪里 | 让研发和客户快速判断变化 | 中 |
| PM-05 | 对比结果导出预览 | 导出前展示将导出的字段范围：当前帧、框选指标、A/B/差值 | 降低导出误解 | 低 |

## 4. Designer 视角：布局、交互和状态反馈

| 编号 | 想法 | 说明 | 价值 | 复杂度 |
|---|---|---|---|---|
| UX-01 | 统一分析侧栏 | 框选、历史、对比、导出共用同一个右侧侧栏，通过标签页切换 | 避免多个抽屉互相遮挡，保持画布主导 | 中 |
| UX-02 | 框选颜色和编号一致 | 画布边框、区域列表、图表曲线使用同一颜色和编号 | 多框选时更容易对应 | 低 |
| UX-03 | 框选创建微交互 | 拖拽时显示尺寸、行列范围；松手后出现命名输入和确认按钮 | 降低坐标不确定感 | 中 |
| UX-04 | 数据对比三栏布局 | 左 A、 中差值、右 B，下方统一时间轴和指标表 | 对比关系最直观 | 中 |
| UX-05 | 差值图图例固定显示 | 红色 = B 大于 A，蓝色 = B 小于 A，灰色 = 接近无变化，并显示范围 | 避免误读差值图 | 低 |

## 5. Engineer 视角：技术实现和复用

| 编号 | 想法 | 说明 | 价值 | 复杂度 |
|---|---|---|---|---|
| ENG-01 | 统一 SelectionStore | 把实时、回放、对比页的框选状态统一成一个 selection store | 避免三处重复实现 | 中 |
| ENG-02 | SelectionStats 派生计算层 | 框选统计统一走派生计算，不改 rawData，不影响采集 | 符合 PRD 数据层原则 | 中 |
| ENG-03 | CompareViewModel | 对比页先生成标准 ViewModel：A 当前帧、B 当前帧、差值矩阵、指标差值 | 降低 UI 复杂度 | 中 |
| ENG-04 | 图表颜色映射复用 | 框选颜色、曲线颜色、差值图图例使用统一色板配置 | 减少 UI 不一致 | 低 |
| ENG-05 | 对比校验前置组件 | A/B 选择后先展示可比性校验结果，不通过时禁止进入分析 | 让错误提前暴露 | 中 |

## 6. 推荐优先做的 5 个方向

| 排名 | 方案 | 一句话描述 | 为什么优先 | 关键假设 | 建议优先级 |
|---|---|---|---|---|---|
| 1 | 框选区域管理面板 | 在 2D 数字视图右侧增加框选列表，集中展示名称、颜色、坐标、指标、显示/隐藏、删除 | 框选是功能亮点，当前最需要从“操作”变成“分析模块” | 用户会同时使用 2-4 个框选区域，并需要快速比较区域数据 | P0 |
| 2 | 数据对比三栏布局 | 对比页采用 A 数据、差值图、B 数据三栏结构，下方统一时间轴和指标表 | 最直接解决“对比看不清”的问题 | 用户主要任务是判断 B 相比 A 哪里变大、哪里变小 | P0 |
| 3 | 分析范围状态条 | 在页面顶部显示当前分析范围、当前帧、过滤值、置零状态、方向状态 | 防止用户误以为框选控制采集或误解当前统计范围 | 用户在实时、回放、对比之间切换时容易混淆当前口径 | P0 |
| 4 | 差值图图例和结论摘要 | 差值图固定显示红蓝含义，并给出当前帧关键指标变化 | 提升客户演示和研发判断效率 | 用户需要快速读懂差值，而不是只看两张图 | P1 |
| 5 | 框选模板轻量入口 | 支持保存和套用常用区域模板，例如腰部、肩部、左侧、右侧 | 适合座椅/靠背重复测试场景 | 客户会反复分析相同区域，手动画框会降低效率 | P1 |

## 7. 框选界面建议

### 7.1 推荐布局

| 区域 | 内容 | 说明 |
|---|---|---|
| 顶部状态条 | 当前模式、分析范围、过滤值、置零状态、方向状态 | 例如：实时分析 / 框选 2 个区域 / 过滤值 10 / 已置零 / 坐垫方向 90 度 |
| 左侧/主区域 | 2D 数字矩阵画布 | 保持当前核心画布，不被卡片遮挡 |
| 画布浮层 | 框选矩形、编号、颜色、拖拽尺寸提示 | 拖拽时显示行列范围 |
| 右侧面板 | 框选区域列表、坐标、实时指标、操作按钮 | 用于管理和查看区域数据 |
| 底部图表 | 全量曲线、多框选独立曲线 | 曲线颜色和框选颜色一致 |

### 7.2 框选区域面板字段

| 字段/控件 | 说明 | 是否必做 |
|---|---|---|
| 区域编号 | 区域 1、区域 2、区域 3、区域 4 | 是 |
| 区域名称 | 默认“区域 1”，可改名为“腰部左侧”等 | 建议 |
| 颜色标识 | 与画布框线和曲线颜色一致 | 是 |
| 坐标信息 | X、Y、长、宽，展示行区间和列区间 | 是 |
| 实时指标 | 平均压强、最大压强、压力总和、受压面积、有效点数 | 是 |
| 显示/隐藏 | 隐藏某个框选的画布边框和曲线 | 建议 |
| 删除 | 删除单个框选 | 是 |
| 应用模板 | 从模板快速生成多个区域 | P1 |

### 7.3 框选交互流程

| 步骤 | 用户动作 | 系统反馈 | 关键规则 |
|---|---|---|---|
| 1 | 点击框选工具 | 进入框选模式，光标和工具按钮状态变化 | 仅 2D 数字视图可编辑 |
| 2 | 在画布拖拽 | 实时显示矩形、X/Y/长/宽、行列范围 | 不允许超出有效矩阵 |
| 3 | 松开鼠标 | 生成临时框选，右侧面板出现区域卡片 | 默认名称为区域 N |
| 4 | 用户确认或修改坐标 | 区域高亮并开始实时统计 | 坐标非法时给出错误提示 |
| 5 | 用户切换实时/回放 | 框选可继续作为分析范围 | 框选不影响采集保存全量矩阵 |

### 7.4 框选状态反馈

| 状态 | UI 表现 | 说明 |
|---|---|---|
| 未框选 | 状态条显示“当前分析：全量” | 统计和曲线使用全量矩阵 |
| 正在拖拽 | 框线使用虚线，显示尺寸标签 | 帮助用户确认区域 |
| 已创建 | 实线边框 + 编号角标 | 与右侧列表对应 |
| 选中区域 | 边框加粗或黄色描边 | 便于编辑当前区域 |
| 区域越界 | 边框变红，保存按钮不可用 | 防止无效框选 |
| 达到上限 | 新建入口置灰，提示最多 4 个区域 | 与 PRD 上限一致 |

## 8. 数据对比界面建议

### 8.1 推荐布局

| 区域 | 内容 | 说明 |
|---|---|---|
| 顶部选择区 | 基准数据 A、对比数据 B、可比性校验状态 | 未通过校验时禁止进入对比 |
| 顶部状态条 | 当前帧、进度、采样率提示、置零口径、方向状态 | 明确当前对比口径 |
| 左栏 | A 数据视图 | 2D 数字或压力矩阵 |
| 中栏 | 差值图 | 红蓝差值，固定图例 |
| 右栏 | B 数据视图 | 与 A 同步帧 |
| 下方 | 指标差值表、框选指标、时间轴 | 当前帧指标和框选指标集中展示 |
| 统一分析侧栏 | 框选区域管理、对比指标、导出预览 | 对比页复用同一个侧栏，不再叠加多个抽屉 |

### 8.2 数据对比交互流程

| 步骤 | 用户动作 | 系统反馈 | 关键规则 |
|---|---|---|---|
| 1 | 选择基准数据 A | 显示 A 的设备类型、矩阵尺寸、帧数、采样率 | A/B 不能是同一条记录 |
| 2 | 选择对比数据 B | 自动执行可比性校验 | 类型、矩阵、坐垫/靠背必须一致 |
| 3 | 点击开始对比 | 进入三栏对比页 | 不可比时展示失败原因 |
| 4 | 拖动时间轴 | A/B 按进度百分比同步 | 帧数不同按百分比映射 |
| 5 | 查看差值图 | 展示 B-A 的红蓝差值 | 红色表示 B 大于 A，蓝色表示 B 小于 A |
| 6 | 创建框选 | 同一坐标作用于 A、B、差值图 | 输出局部差值指标 |
| 7 | 导出结果 | 展示导出预览后导出 | V2.1 只导出当前帧和框选指标 |

### 8.3 对比指标表建议

| 指标 | A 值 | B 值 | 差值 | 变化率 | 说明 |
|---|---|---|---|---|---|
| 平均压强 | 显示 | 显示 | B - A | (B - A) / A | A 为 0 时显示 N/A |
| 最大压强 | 显示 | 显示 | B - A | (B - A) / A | 同时显示最大值坐标 |
| 压力总和 | 显示 | 显示 | B - A | (B - A) / A | 支持全量和框选 |
| 受压面积 | 显示 | 显示 | B - A | (B - A) / A | 使用当前过滤值 |
| 有效受压点数 | 显示 | 显示 | B - A | (B - A) / A | 与面积口径一致 |
| 压力重心 | 显示坐标 | 显示坐标 | 位移距离 | 不适用 | 可用箭头表示移动方向 |

### 8.4 差值图视觉规则

| 元素 | 规则 | 说明 |
|---|---|---|
| 红色 | B 压力大于 A | 表示对比数据压力增加 |
| 蓝色 | B 压力小于 A | 表示对比数据压力降低 |
| 灰色/白色 | 接近无变化 | 用于低于差值阈值的点 |
| 图例 | 固定在差值图右上角或底部 | 必须展示正负含义和当前范围 |
| 悬浮提示 | 显示 A 值、B 值、差值、变化率、坐标 | 辅助定位具体点 |
| 当前范围 | 默认按当前帧最大绝对差值对称映射 | 避免正负颜色偏置 |

## 9. 研发实现建议

| 建议 | 说明 | 优先级 |
|---|---|---|
| 建立统一 `selectionStore` | 保存框选 ID、名称、颜色、坐标、矩阵对象、可见状态、选中状态 | P0 |
| 抽象 `SelectionPanel` | 实时页、回放页、对比页都复用同一个框选管理面板 | P0 |
| 抽象 `SelectionStats` | 输入矩阵和 selection，输出区域指标；不修改 rawData | P0 |
| 建立 `CompareViewModel` | 统一生成 A 当前帧、B 当前帧、差值矩阵、指标差值、变化率 | P0 |
| 抽象 `CompareValidator` | A/B 选择后先输出可比性校验结果和失败原因 | P0 |
| 统一颜色系统 | 框选边框、列表色块、曲线颜色、差值图色阶使用配置化色板 | P1 |
| 建立导出预览对象 | 对比导出前先生成导出范围和字段说明，避免导出逻辑散落在 UI | P1 |

## 10. 建议落地顺序

| 阶段 | 范围 | 目标 | 产出 |
|---|---|---|---|
| 第一阶段 | 框选区域管理面板 + 分析范围状态条 | 先把框选功能做得清楚、可信、可管理 | 框选 UI PRD 补充、低保真原型、研发字段说明 |
| 第二阶段 | 数据对比三栏布局 + 可比性校验 | 先让对比看得懂、进得去、错得明白 | 数据对比 UI PRD 补充、对比页布局图 |
| 第三阶段 | 差值图图例 + 指标摘要 + 框选对比 | 提升分析效率和演示价值 | 差值图规则、指标表规则、框选对比规则 |
| 第四阶段 | 框选模板 + 对比导出预览 | 提升重复测试效率和交付完整度 | 模板管理规则、导出预览规则 |

## 11. 需要产品确认的问题

| 编号 | 问题 | 推荐方案 |
|---|---|---|
| UI-DEC-01 | 框选、历史、对比是否使用多个抽屉 | 推荐不使用多个抽屉，统一为一个分析侧栏 |
| UI-DEC-02 | 框选选中态颜色 | 推荐黄色描边，区域原色保留 |
| UI-DEC-03 | 多框选指标是否全部展开 | 推荐默认只展开选中区域，其余折叠显示核心指标 |
| UI-DEC-04 | 数据对比默认视图 | 推荐 A / 差值 / B 三栏，差值图居中 |
| UI-DEC-05 | 对比页是否默认显示框选面板 | 推荐默认打开“对比”标签，点击框选工具后切到“框选”标签 |
| UI-DEC-06 | 差值图是否支持用户手动设置色阶 | 推荐 V2.0 不做，先按当前帧最大绝对差值自动映射 |
