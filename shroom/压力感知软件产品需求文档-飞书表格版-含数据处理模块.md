### 7.3.1 连接异常处理重点

一键连接和重新连接是入口级能力。该模块如果卡死，后续展示、数据处理、采集、回放、导出都不可用。因此连接模块必须按“不会卡死、可恢复、可解释”的标准设计。

| 风险 | 影响 | 处理原则 | 优先级 |
|---|---|---|---|
| 连接中卡死 | 用户无法进入实时展示 | 必须有全流程超时和连接锁释放 | P0 |
| 重复点击连接 | 串口状态混乱 | 前端按钮置灰，后端连接锁拦截 | P0 |
| 串口被占用 | 无法打开设备 | 明确提示串口占用，并释放临时资源 | P0 |
| 波特率探测失败 | 设备无法识别 | 尝试候选波特率后返回明确失败原因 | P0 |
| MAC 读取失败 | 无法授权和识别类型 | 禁止采集，提示重新插拔或重连 | P0 |
| 旧连接未释放 | 重连失败或僵尸串口 | 重连前必须关闭旧端口、监听器、parser、定时器 | P0 |
| 采集中断连 | 数据可能丢失 | 保留已采集数据，生成异常中断状态 | P0 |

### 7.3.3 一键连接主流程

| 步骤 | 系统动作 | 成功结果 | 异常处理 |
|---|---|---|---|
| 1 | 前端点击一键连接，按钮进入连接中 | 防止重复点击 | 已连接或连接中时直接提示 |
| 2 | 后端创建连接锁 | 同一时间只有一个连接任务 | 锁已存在时返回连接中 |
| 3 | 清理残留串口资源 | 避免旧连接影响新连接 | 清理失败记录日志但继续尝试 |
| 4 | 扫描串口 | 得到候选串口列表 | 无串口提示未检测到设备 |
| 5 | 筛选 CH340 设备 | 得到目标设备端口 | 无 CH340 提示检查连接线 |
| 6 | 探测波特率 | 找到可用波特率 | 候选波特率均失败则返回失败 |
| 7 | 建立稳定串口连接 | 打开正式连接 | 打开失败释放资源 |
| 8 | 读取 MAC | 获取设备身份 | MAC 读取失败则禁止采集 |
| 9 | 匹配设备类型 | 得到系统类型 | 未配置或不匹配则提示 |
| 10 | 绑定数据监听 | 开始接收实时帧 | 绑定失败释放连接 |
| 11 | 推送连接成功状态 | 前端进入实时展示 | 任一步失败都释放连接锁 |

### 7.3.4 连接锁和超时规则

| 项目 | 规则 | 优先级 |
|---|---|---|
| 前端按钮锁 | 连接中/重连中按钮置灰，防止重复点击 | P0 |
| 后端连接锁 | `/connPort`、`/rescanPort` 同一时间只允许一个任务 | P0 |
| 锁释放 | 成功、失败、超时、异常都必须释放锁 | P0 |
| 全流程超时 | 一键连接建议 20 秒内返回结果 | P0 |
| 串口扫描超时 | 建议 3 秒 | P0 |
| 单个波特率探测超时 | 建议 2 秒 | P0 |
| 稳定连接打开超时 | 建议 2 秒，最多重试 3 次 | P0 |
| MAC 读取超时 | 建议 30 秒 | P0 |
| 前端请求超时 | 建议 15-20 秒，超时后提示用户 | P0 |

### 7.3.5 重新连接规则

| 场景 | 是否允许重连 | 处理规则 | 优先级 |
|---|---|---|---|
| 未连接 | 允许 | 等同一键连接 | P0 |
| 连接失败 | 允许 | 清理旧状态后重新扫描 | P0 |
| 已连接 | 允许 | 先断开旧连接，再重新连接 | P0 |
| 设备异常 | 允许 | 清理旧 parser、listener、timer 后重连 | P0 |
| 连接中 | 不允许 | 提示连接中，请稍候 | P0 |
| 重连中 | 不允许 | 提示重连中，请稍候 | P0 |
| 采集中 | 谨慎允许 | 建议先提示会中断采集，保留已采集数据 | P0 |
| 回放中 | 允许但不影响回放 | 回放使用历史数据，不依赖实时连接 | P1 |

### 7.3.6 后端资源释放规则

| 资源 | 释放时机 | 规则 |
|---|---|---|
| 临时探测串口 | 波特率探测结束后 | 成功或失败都必须 close |
| 稳定连接串口 | 断开、重连、连接失败后 | 必须 close 并清空引用 |

| 定时器 | 断开、重连、异常后 | 必须 clear |
| 数据缓存 | 重连前 | 按设备和系统类型清理 |
| 连接锁 | 成功、失败、超时、异常后 | 必须释放 |

### 7.3.8 `/startCol` 采集前强校验

`/startCol` 是进入数据采集的最后一道门。即使前端按钮状态异常，后端也必须再次校验设备是否合法，不能只依赖前端状态。

| 校验项 | 通过条件 | 失败处理 | 错误码 | 用户提示 | 优先级 |
|---|---|---|---|---|---|
| 设备连接状态 | 当前存在稳定串口连接，且设备在线 | 拒绝开始采集 | DEVICE_DISCONNECTED | 设备未连接，请先连接设备 | P0 |
| MAC 已读取 | 当前连接已成功读取 MAC | 拒绝开始采集 | MAC_FAIL | 设备识别失败，请重新连接 | P0 |
| MAC 已配置 | MAC 命中本地设备配置 | 拒绝开始采集 | TYPE_UNKNOWN | 当前设备未配置，请先录入设备信息 | P0 |
| 设备已授权 | 当前 MAC 或设备密钥通过授权校验 | 拒绝开始采集 | AUTH_FAIL | 当前设备未授权，不能采集 | P0 |
| 类型匹配 | 当前连接设备类型与当前系统类型一致 | 拒绝开始采集 | TYPE_MISMATCH | 当前设备类型与系统类型不匹配 | P0 |
| 矩阵尺寸有效 | 设备类型能映射到明确矩阵尺寸 | 拒绝开始采集 | MATRIX_SIZE_INVALID | 设备点阵配置异常，不能采集 | P0 |
| 采集未进行中 | 当前没有未结束采集任务 | 拒绝重复采集 | COLLECT_BUSY | 正在采集，请勿重复开始 | P0 |
| 方向状态稳定 | 当前不处于方向修改中 | 拒绝开始采集 | DIRECTION_UPDATING | 方向配置正在更新，请稍后采集 | P1 |

### 7.3.9 连接错误码落地规则

| 规则 | 说明 | 优先级 |
|---|---|---|
| 前后端共用同一套错误码 | 后端接口返回 `code/message/detail`，前端只负责展示和状态切换 | P0 |

## 8. 模块二：设备数据接收与处理

### 8.1 模块目标

| 项目 | 内容 |
|---|---|
| 模块目标 | 把设备串口原始数据转换成后续展示、统计、采集和导出可使用的标准压力数据 |
| 模块位置 | 位于“设备连接流程”和“展示视图”之间 |
| 业务价值 | 保证后续所有压力图、统计指标、采集数据和 CSV 数据可信 |
| 成功结果 | 生成结构稳定、尺寸正确、可追溯的原始压力矩阵和推送数据 |

### 8.2 为什么需要单独拆这个模块

| 问题 | 说明 |
|---|---|
| 连接成功不等于数据可用 | 串口连接只是建立通道，还需要确认设备帧能被正确解析 |
| 展示视图依赖处理后数据 | 2D/3D 视图不能直接消费串口原始字节，需要标准矩阵 |
| 统计指标依赖数据口径 | 平均压强、最大压强、面积等都依赖矩阵尺寸、单位和有效点规则 |
| 采集和导出依赖原始数据 | 如果处理层没有定义 rawData，后续很容易把显示数据误当原始数据保存 |
| 异常定位需要分层 | 设备断连、坏帧、矩阵错位、显示异常需要区分是连接问题、解析问题还是渲染问题 |

### 8.3 当前已实现

| 功能 | 当前状态 | 备注 |
|---|---|---|
| 串口数据接收 | 已实现 | 由串口监听和数据帧处理负责 |
| 数据帧分隔 | 已实现 | 根据协议分隔符识别帧 |
| 波特率探测时帧校验 | 已实现 | 用于判断候选波特率是否可用 |
| 设备类型对应解析函数 | 已实现 | hand、bed、endi、carY 等存在不同转换函数 |
| 一维点阵转矩阵 | 已实现 | 按系统类型转换为对应尺寸 |
| 多系统矩阵尺寸配置 | 已实现 | 如 32x32、46x46、50x64、64x64 |
| WebSocket 实时推送 | 已实现 | 后端推送前端消费 |
| 采样率/HZ 处理 | 已实现或部分实现 | V2.0 以后端按帧时间计算为准，硬件返回值用于记录和校验 |
| 历史回放数据结构 | 已实现 | 与实时数据结构需保持一致 |

### 8.4 输入输出

| 类型 | 内容 | 来源/去向 | 说明 |
|---|---|---|---|
| 输入 | 串口原始字节流 | 设备串口 | 设备持续发送的原始数据 |
| 输入 | 设备类型 | 连接模块 | 决定使用哪套解析规则 |
| 输入 | 系统类型 | 前端/后端状态 | 决定矩阵尺寸和展示结构 |
| 输入 | 当前波特率 | 连接模块 | 影响数据帧接收速度 |
| 输出 | 原始帧 rawFrame | 数据处理模块内部 | 用于调试和追溯 |
| 输出 | 原始矩阵 rawData | 展示、统计、采集模块 | 设备原始方向和原始压力数据 |
| 输出 | 推送数据 payload | WebSocket | 前端实时展示使用 |
| 输出 | 采样率 HZ | 展示、历史、CSV | 后端按帧时间计算，用于显示、回放和导出 |
| 输出 | 硬件返回采样率 | 历史、CSV、日志 | 仅作为记录和校验，不作为回放主时钟 |
| 输出 | 异常状态 | 前端提示、日志 | 如坏帧、断帧、矩阵长度不匹配 |

### 8.5 主处理流程

| 步骤 | 处理动作 | 输入 | 输出 | 异常处理 |
|---|---|---|---|---|
| 1 | 接收串口原始字节 | 设备字节流 | 缓冲区数据 | 无数据时保持等待 |
| 2 | 查找帧分隔符 | 缓冲区数据 | 候选数据帧 | 找不到分隔符则继续累积或丢弃异常片段 |
| 3 | 校验帧完整性 | 候选数据帧 | 完整数据帧 | 长度不匹配则丢弃并记录 |
| 4 | 识别设备类型 | MAC 映射/设备类型 | 解析规则 | 类型未知则不进入展示和采集 |
| 5 | 解析压力点数组 | 完整数据帧 | 一维压力数组 | 解析失败则丢弃该帧 |
| 6 | 转换矩阵 | 一维压力数组 + 系统类型 | rawData 矩阵 | 长度和矩阵尺寸不匹配则报错 |
| 7 | 计算采样率 | 后端接收帧时间戳 | HZ | 以后端帧间隔计算为准，异常波动时标记；硬件返回采样率仅记录和校验 |
| 8 | 生成推送数据 | rawData + metadata | WebSocket payload | 推送失败时记录连接状态 |
| 9 | 分发下游模块 | payload | 展示、统计、采集 | 下游失败不应破坏原始数据 |

### 8.7 数据层业务逻辑

| 数据层 | 生成位置 | 业务含义 | 是否可被修改 | 下游用途 |
|---|---|---|---|---|
| rawFrame | 后端串口接收后 | 设备原始帧或原始字节片段 | 不应修改 | 调试、异常定位 |
| rawPointArray | 后端解析后 | 一维压力点数组 | 不应修改 | 矩阵转换 |
| rawData | 后端或前端矩阵转换后 | 业务原始压力矩阵，经过方向修正后作为全链路基准矩阵 | 可被翻转/旋转修改；不受颜色、润滑、噪点消除、框选污染 | 展示、统计、采集、导出 |
| backendProcessedData | 后端处理数据流 | 在 rawData 基础上做后端级处理后的矩阵，例如噪点消除 | 可由后端数据处理规则生成 | 展示、统计、采集、导出 |
| displayData | 前端处理后 | 用于界面展示的矩阵 | 可受颜色、润滑、高度、响应速度影响 | 2D/3D 展示 |
| calculatedData | 统计模块处理后 | 用于指标计算的矩阵 | 受置零、框选、后端处理数据流影响 | 平均值、最大值、面积等 |
| storedData | 采集模块处理后 | 入库保存的数据 | 口径必须明确 | 历史回放 |
| exportedData | 导出模块处理后 | CSV 输出数据 | 口径必须明确 | 科研分析、外部导入 |

说明：V2.0 将 `rawFrame/rawPointArray` 作为设备原始输入保留；`rawData` 是软件业务原始矩阵。用户执行翻转/旋转后，`rawData` 需要按修正后的方向重建，后续可视化、统计、采集、回放、导出都基于修正后的 `rawData`。

### 8.8 异常处理业务逻辑

| 异常类型 | 判断条件 | 系统处理 | 用户提示 | 是否阻塞采集 |
|---|---|---|---|---|
| 空帧 | 收到空数据或无有效压力点 | 丢弃该帧 | 不一定提示，可记录日志 | 否 |
| 坏帧 | 帧头/帧尾/分隔符异常 | 丢弃该帧并记录 | 高频出现时提示设备数据异常 | 视出现频率而定 |
| 帧长度不匹配 | 解析长度不符合设备类型 | 丢弃该帧 | 提示数据长度异常 | 是 |
| 设备类型未知 | MAC 未命中且设备未返回类型 | 不解析为有效矩阵 | 提示设备类型未知 | 是 |
| 矩阵尺寸不匹配 | 点数无法映射到配置矩阵 | 拒绝进入展示/采集 | 提示设备配置异常 | 是 |
| 采样率异常 | 后端计算 HZ 长时间为 0 或明显跳变 | 记录并提示，同时保留硬件返回采样率用于排查 | 视情况而定 |
| WebSocket 推送失败 | 前端连接断开 | 保留后端最新数据，等待重连 | 提示实时连接异常 | 不一定 |
| 连续坏帧 | 一段时间内坏帧超过阈值 | 标记设备数据异常 | 提示检查设备或连接 | 是 |

### 8.8.1 异常帧阈值和提示规则

| 场景 | 建议阈值 | 系统处理 | 用户提示 | 日志要求 | 优先级 |
|---|---|---|---|---|---|
| 单个坏帧 | 1 帧 | 丢弃，不中断实时展示 | 不提示 | 记录 bad_frame_count | P0 |
| 短时间坏帧增多 | 1 秒内坏帧率超过 10% | 保持连接，标记数据质量下降 | 设备数据不稳定，请检查连接 | 记录窗口内总帧、坏帧、比例 | P0 |
| 连续坏帧 | 连续 10 帧 | 暂停采集写入异常帧，状态进入设备异常 | 设备数据异常，请重新连接 | 记录起止时间和最后一帧错误 | P0 |
| 帧长度不匹配 | 任意 1 帧 | 丢弃该帧；连续出现按设备异常处理 | 数据长度异常，请检查设备类型 | 记录期望长度、实际长度、设备类型 | P0 |
| 矩阵尺寸不匹配 | 任意 1 次转换失败 | 禁止展示为有效矩阵，禁止采集 | 点阵配置异常，不能采集 | 记录 matrix_key、width、height、point_count | P0 |
| 采样率为 0 | 连续 2 秒 | 保持连接，标记 HZ 异常 | 数据刷新异常，请检查设备 | 记录最近帧时间戳 | P1 |
| WebSocket 推送失败 | 连续 3 次 | 保留后端数据，前端重连 | 实时显示连接异常 | 记录连接 ID 和失败原因 | P1 |

说明：阈值是 V2.0 默认建议值，研发可根据真实设备帧率微调，但必须保证“单帧异常不打断，连续异常可提示和恢复”。

| 采集是保存模块 | 采集保存全量压力矩阵、时间戳、设备信息、采样率、方向等数据 | P0 |
| 框选不决定采集范围 | 无论是否框选，采集都必须保存全量矩阵 | P0 |
| 实时和回放都可框选 | 用户可以在实时模式框选，也可以在历史回放里重新框选 | P0 |
| 回放框选不依赖采集时框选 | 历史回放只要有全量矩阵，就可以新建框选并重新计算区域数据 | P0 |
| 回放框选需要先锁定帧 | 历史回放中创建或编辑框选时，系统应自动暂停并锁定当前帧，避免边播放边画框 | P0 |
| 回放框选默认应用整段历史 | 用户在某一帧画出的区域默认应用到整段历史记录，用同一坐标重算每一帧区域统计和曲线 | P0 |
| 框选结果是派生数据 | 框选统计、框选曲线、框选导出都不是原始采集数据 | P0 |

### 10.4.2.1 历史回放框选交互优化

历史数据中直接框选较难操作，核心原因是回放画面会随时间变化，用户难以稳定定位到想分析的一帧。因此 V2.0 增加“历史框选模式”：先暂停并锁定当前帧，再让用户画框，最后把框选区域应用到历史数据。

| 项目 | 规则 | 优先级 |
|---|---|---|
| 入口 | 回放界面提供“框选分析”入口 | P0 |
| 进入方式 | 用户点击“框选分析”后，系统自动暂停播放并锁定当前帧 | P0 |
| 当前帧状态 | 顶部或回放控件显示“已锁定第 N 帧 / 当前时间 T” | P0 |
| 定位方式 | 支持拖动时间轴、输入帧号、上一帧、下一帧定位到目标帧 | P0 |
| 画框对象 | 用户在锁定帧的 2D 数字矩阵上创建框选 | P0 |
| 默认应用范围 | 框选默认应用到整段历史记录，基于同一坐标重算每一帧的区域统计和曲线 | P0 |
| 当前帧分析 | 支持只查看当前帧框选统计，但不作为默认保存范围 | P1 |
| 时间段分析 | 支持选择起止帧后只分析某一段历史 | P2 |
| 播放恢复 | 用户完成框选后，可点击继续播放，框选区域随历史帧同步统计 | P0 |
| 数据影响 | 不修改历史 rawData，不修改 storedData，只生成派生 selectionData 和 selectionStats | P0 |

### 10.4.3 框选模板业务逻辑

框选模板是一组可复用的框选区域配置。它用于让用户快速恢复常用分析区域，例如“坐垫左侧/右侧”“靠背肩部/腰部”“前排/后排压力区”。

框选模板不保存压力矩阵，不保存统计结果，不影响采集。模板只保存“要分析哪些区域”。

| 项目 | 规则 | 优先级 |
|---|---|---|
| 模板定位 | 一组框选区域的配置集合 | P1 |
| 模板用途 | 在实时、回放、数据对比中快速应用同一组区域 | P1 |
| 模板内容 | 区域名称、坐标、长宽、颜色、设备类型、矩阵尺寸、显示对象、创建时间 | P1 |
| 不保存内容 | 不保存 rawData、displayData、calculatedData、采集帧、统计结果 | P0 |
| 与采集关系 | 模板不影响采集开始、停止和采集保存范围 | P0 |
| 与历史关系 | 历史记录不强制绑定模板，回放时可临时应用匹配模板 | P1 |
| 与对比关系 | 对比模式中可应用模板，同一区域同时作用于 A/B 和差值图 | P1 |
| 存储方式 | V2.0 建议本地存储，后续可扩展导入/导出模板 | P1 |

### 10.4.4 框选模板主流程

| 步骤 | 用户动作 | 系统处理 | 成功结果 | 异常处理 |
|---|---|---|---|---|
| 1 | 创建框选区域 | 生成当前框选区域列表 | 画布上出现区域 | 越界、尺寸为 0、超过 4 个时不创建 |
| 2 | 点击保存为模板 | 弹出模板保存窗口 | 用户可填写模板名称、备注 | 无区域时按钮置灰 |
| 3 | 保存模板 | 校验名称、设备类型、矩阵尺寸 | 模板进入模板列表 | 名称为空或重复时提示 |
| 4 | 应用模板 | 校验当前设备/历史/对比数据是否匹配 | 模板区域恢复到画布 | 不匹配时禁止应用 |
| 5 | 编辑模板 | 加载模板区域并允许修改 | 保存后更新模板 | 模板不存在时提示 |
| 6 | 删除模板 | 二次确认后删除 | 模板从列表移除 | 已应用区域不受影响 |

### 10.4.5 框选模板匹配规则

推荐：严格匹配。设备类型、显示对象、矩阵尺寸全部一致才可应用。

| 匹配项 | 是否必须匹配 | 说明 | 优先级 |
|---|---|---|---|
| 设备类型 | 是 | 例如汽车坐垫、汽车靠背、endi 坐垫、endi 靠背 | P1 |
| 矩阵尺寸 | 是 | 例如 32x32、46x46、50x64、64x64 | P1 |
| 显示对象 | 是 | 坐垫模板只能用于坐垫，靠背模板只能用于靠背 | P1 |
| 显示方向 | 建议匹配 | 当前按显示方向坐标保存，应用时按当前显示方向恢复 | P1 |
| 置零状态 | 否 | 模板只保存区域，不关心置零 | P1 |
| 采集状态 | 否 | 采集中应用模板只影响当前分析，不影响采集 | P1 |

### 10.4.6 框选模板待确认项和方案

| 待确认项 | 选项一简介 | 选项二简介 | 选项三简介 | 当前采用/推荐 |
|---|---|---|---|---|
| 模板匹配规则 | 设备类型、显示对象、矩阵尺寸严格匹配 | 只要矩阵尺寸一致就允许 | 自动缩放适配不同矩阵 | 当前采用严格匹配 |

### 10.5 翻转业务逻辑

画布翻转/旋转用于让用户把传感器实际摆放方向和软件分析方向对齐。当前产品口径确认：翻转/旋转不是单纯显示效果，而是方向修正能力。方向修正后，后续展示、统计、采集、回放、导出都应基于修正后的矩阵。

说明：底层串口原始帧 rawFrame 可保留用于排查；但业务上的 rawData / storedData / exportedData 应按用户确认后的方向保存和使用。

| 项目 | 业务规则 | 优先级 |
|---|---|---|
| 触发入口 | 用户点击左右翻转或上下翻转按钮 | P0 |
| 旋转入口 | 用户点击 90 度旋转按钮；V2.0 采用单按钮，每次点击顺时针旋转 90 度 | P0 |
| 影响对象 | 当前对象的业务矩阵方向，坐垫和靠背分开处理 | P0 |
| 左右翻转 | 当前矩阵按横向镜像修正 | P0 |
| 上下翻转 | 当前矩阵按纵向镜像修正 | P0 |
| 90 度旋转 | 当前矩阵每次按顺时针 90 度旋转，连续点击按 0/90/180/270 循环累加 | P0 |
| 双向翻转 | 同时开启左右和上下翻转时，相当于旋转 180 度 | P0 |
| 坐垫/靠背独立 | 座椅坐垫和靠背分别保存方向状态，互不影响 | P0 |
| 是否影响 rawFrame | 不影响，保留设备原始帧用于排查 | P0 |
| 是否影响 rawData | 影响。业务 rawData 应按修正后的方向生成和保存 | P0 |
| 是否影响 displayData | 影响。展示基于修正后的矩阵 | P0 |
| 是否影响 calculatedData | 影响。统计、坐标、重心基于修正后的矩阵 | P0 |
| 是否影响采集 | 影响。采集保存修正方向后的全量矩阵，并记录方向配置 | P0 |
| 是否影响回放 | 影响。回放直接使用已持久化的修正方向矩阵 | P0 |
| 是否影响 CSV | 影响。CSV 导出修正方向后的矩阵，并导出方向配置字段 | P0 |
| 是否持久化 | 是。方向配置需要按设备类型 + 显示对象保存到本地 | P0 |

### 10.6 量尺业务逻辑

量尺用于测量两个传感点或一个区域之间的真实距离。量尺是查看和分析工具，不影响原始数据、采集数据和统计数据。

| 项目 | 业务规则 | 优先级 |
|---|---|---|
| 使用条件 | 仅 2D 数字视图下可用 | P0 |
| 触发入口 | 用户点击量尺按钮进入测量模式 | P1 |
| 互斥规则 | 量尺和框选不能同时启用 | P0 |
| 测量对象 | 两个传感点之间的距离，或矩形区域的宽高距离 | P1 |
| 计算依据 | `pointWidthDistance` 和 `pointHeightDistance`，由软件按设备类型配置维护 | P0 |
| 默认单位 | 推荐使用 cm，必要时展示 mm | P1 |
| 缩放影响 | 缩放只影响显示，不影响真实距离 | P0 |
| 翻转影响 | 翻转只改变显示位置，不改变真实距离 | P0 |
| 采集影响 | 不影响采集，不入库为采集数据 | P0 |
| 回放影响 | 回放中可使用量尺测量历史画面 | P1 |
| 导出影响 | V2.0 不导出量尺结果，后续可支持截图/报告 | P2 |

### 10.6.1 量尺主流程

| 步骤 | 用户动作 | 系统处理 | 成功结果 | 异常处理 |
|---|---|---|---|---|
| 1 | 点击量尺按钮 | 进入量尺模式，鼠标指针变为测量状态 | 可开始测量 | 非 2D 数字视图提示不可用 |
| 2 | 点击或拖拽起点 | 记录起点传感点坐标 | 显示起点 | 起点超出有效矩阵时不记录 |
| 3 | 点击或拖拽终点 | 记录终点传感点坐标 | 显示测量线和距离 | 终点超出有效矩阵时提示 |
| 4 | 查看测量结果 | 根据点距计算横向、纵向、直线距离 | 展示距离数值 | 点距缺失时提示无法计算 |
| 5 | 新增多条量尺 | 保存当前量尺并继续测量 | 画布显示多条量尺 | 超过上限时提示 |
| 6 | 选中量尺 | 高亮当前量尺 | 可删除或查看详情 | 无 |
| 7 | 删除量尺 | 删除当前量尺 | 画布更新 | 删除失败时保持原状 |

### 10.6.2 量尺计算规则

| 项目 | 公式/规则 | 优先级 |
|---|---|---|
| 点坐标 | 使用矩阵坐标，不使用 DOM 像素坐标 | P0 |
| 横向距离 | abs(x2 - x1) x pointWidthDistance | P0 |
| 纵向距离 | abs(y2 - y1) x pointHeightDistance | P0 |
| 直线距离 | sqrt(横向距离² + 纵向距离²) | P1 |
| 单位换算 | mm 转 cm 时除以 10 | P1 |
| 缩放后计算 | 仍按矩阵坐标和点距计算，不按屏幕像素计算 | P0 |
| 翻转后计算 | 翻转前后同两点真实距离一致 | P0 |
| 坐垫/靠背切换 | 使用当前 displayType 对应的点距配置 | P0 |
| 不同设备 | 使用当前设备类型对应点距配置 | P0 |

### 12.5.4 回放损坏帧处理

| 场景 | 判定条件 | 系统处理 | 用户提示 | 日志/统计 | 优先级 |
|---|---|---|---|---|---|
| 单帧矩阵为空 | 当前帧缺少 real_data 或矩阵为空 | 跳过该帧，播放下一帧 | 不弹窗，可在回放控件显示“已跳过异常帧” | skipped_frame_count +1 | P0 |
| 单帧矩阵长度不匹配 | 当前帧点数不等于历史记录矩阵配置 | 跳过该帧 | 当前数据存在异常帧，已自动跳过 | 记录 frame_index、期望点数、实际点数 | P0 |
| 单帧数值非法 | 出现非数字、无法解析 JSON、负值是否非法按设备规则判断 | 跳过该帧 | 当前数据存在异常帧，已自动跳过 | 记录解析错误 | P0 |
| 连续损坏帧 | 连续 10 帧损坏 | 暂停回放，允许用户继续或退出 | 历史数据损坏较多，建议检查文件 | 记录连续损坏区间 | P0 |

### 12.6.1 采集中强退恢复流程

V2.0 采用默认推荐恢复策略：如果存在有效帧，默认推荐“恢复为异常中断历史记录”；只有用户明确确认丢弃时，才删除临时任务。

| 步骤 | 系统动作 | 判断结果 | 用户选择 | 处理规则 | 优先级 |
|---|---|---|---|---|---|
| 1 | 软件启动时检查采集状态 | 存在 `collecting=true` 或未完成记录 | 无 | 进入异常恢复状态，不自动覆盖 | P0 |
| 2 | 检查临时缓存和数据库帧数 | 有可恢复帧 | 恢复为历史记录 | 生成一条“异常中断”历史记录 | P0 |
| 3 | 检查临时缓存和数据库帧数 | 无有效帧 | 丢弃 | 清理空任务并记录日志 | P0 |
| 4 | 用户选择恢复 | 有效帧数大于 0 | 恢复 | 补结束时间、异常结束标记、帧数、metadata | P0 |
| 5 | 用户选择暂不处理 | 有可恢复帧 | 稍后处理 | 保留缓存，不允许新采集覆盖同一缓存 | P1 |
| 6 | 用户选择丢弃 | 用户确认二次弹窗 | 丢弃 | 删除临时任务，保留异常日志 | P0 |
| 7 | 恢复失败 | 数据库写入失败或缓存损坏 | 重试/导出日志 | 不删除原缓存，提示联系售后 | P0 |

### 12.6.3 历史数据与本地缓存定义

| 类型 | 定义 | 是否允许自动清理 | 说明 |
|---|---|---|---|
| 历史数据 | 用户点击采集后形成的传感器压力记录，包括全量压力帧、时间戳、设备信息、方向、置零状态、框选元数据、名称和备注 | 待确认，当前默认不自动删除 | 这是用户实验或演示结果，不等同于普通缓存 |
| 临时采集缓存 | 采集中断、写库失败或强退恢复前保存的临时帧和任务状态 | 待确认 | 用于防丢失，恢复后可转成历史数据 |

### 13.6.2 CSV legacy 字段映射规则

V2.0 采用推荐兼容范围：兼容无版本号、常见旧字段别名和缺省 metadata；不做完整字段映射配置器。

| legacy 字段/形态 | 标准字段 | 兼容规则 | 不兼容处理 | 优先级 |

### 13.7.3 帧同步和差值计算规则

| 项目 | 规则 | 优先级 |
|---|---|---|
| 对齐方式 | 按播放进度百分比同步 A/B | P0 |
| 当前帧计算 | A 当前帧 = A 总帧数 x 当前进度；B 当前帧 = B 总帧数 x 当前进度 | P0 |
| 帧数不同 | 允许对比，按百分比映射到最近帧 | P0 |
| 采样率不同 | 允许对比，但提示“采样率不同，按进度对齐” | P1 |
| 单点差值 | B 点值 - A 点值 | P0 |
| 指标差值 | B 指标 - A 指标 | P0 |
| 变化率 | (B - A) / A x 100% | P0 |
| A=0 | 变化率显示 N/A，不报错 | P0 |

## 14. 全局数据规则

### 14.1 数据分层

| 数据层 | 定义 | 是否必须保留 | 影响因素 |
|---|---|---|---|
| 设备原始输入 | rawFrame/rawPointArray，设备原始帧和一维点数组 | 必须保留 | 不受任何工具影响，用于排查 |
| 业务原始矩阵 | rawData，软件全链路使用的基准压力矩阵 | 必须保留 | 受翻转/旋转影响，不受颜色、润滑、噪点消除、框选影响 |
| 后端处理数据 | backendProcessedData，后端在 rawData 基础上处理后的矩阵 | 按处理状态生成 | 受噪点消除影响 |
| 置零后数据 | 扣除置零基线后的矩阵 | 根据置零状态生成 | 受置零影响 |
| 显示数据 | 用于界面展示的矩阵 | 必须可重建 | 受颜色、润滑、高度、响应速度影响 |
| 统计数据 | 用于计算指标的矩阵 | 口径必须明确 | 受置零、框选、噪点消除影响 |
| 入库数据 | 采集保存到数据库的数据 | 必须可追溯 | 方向和置零口径需明确 |
| 导出数据 | CSV 导出的数据 | 必须标明口径 | 需和导入兼容 |

### 14.2 功能对数据层影响

| 功能 | 原始数据 | 显示数据 | 统计数据 | 入库数据 | 导出数据 | 结论 |
|---|---|---|---|---|---|---|
| 翻转/旋转 | rawFrame 不影响；业务 rawData 按修正方向重建 | 影响 | 影响坐标、框选、重心 | 保存修正后业务矩阵和方向 metadata | 导出修正后业务矩阵和方向 metadata | V2.0 已确认 |
| 置零 | 不影响设备原始帧 | 影响 | 影响 | 保存置零后业务矩阵和置零 metadata | 导出置零后业务矩阵和置零 metadata | V2.0 已确认 |
| 颜色调节 | 不影响 | 影响 | 不影响 | 不影响 | 不影响 | 清晰 |
| 高斯润滑 | 不影响 | 影响 | V2.0 不影响 | 不影响 | 不影响 | V2.0 只做显示 |
| 噪点消除 | 不影响 rawData，生成 backendProcessedData | 影响 | 影响 | 保存后端处理后的业务矩阵和处理标记 | 导出后端处理后的业务矩阵和处理标记 | V2.0 已确认 |
| 框选 | 不影响 | 影响局部显示 | 影响统计范围 | 保存全量矩阵 + 框选元数据 | 可导出派生数据 | V2.0 已确认 |

### 14.3 原始数据原则

| 编号 | 原则 | 优先级 |
|---|---|---|
| 原则-01 | 框选不得导致全量原始矩阵丢失 | P0 |
| 原则-02 | 颜色、高斯润滑、高度、响应速度不得污染 rawData | P0 |
| 原则-03 | 翻转/旋转会修正业务 rawData，必须保存方向并保证可还原 | P0 |
| 原则-04 | 置零如影响统计或导出，必须记录置零状态和置零时间 | P0 |
| 原则-05 | CSV 导出必须支持后续导入、回放和对比 | P0 |
| 原则-06 | 噪点消除不覆盖 rawData，但会生成后端处理数据流并影响统计、采集和导出 | P0 |
| 原则-07 | 任何导入的 legacy 数据如果缺失关键 metadata，必须提示用户其可比性受限 | P1 |
