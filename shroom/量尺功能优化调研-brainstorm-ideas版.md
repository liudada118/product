# 量尺功能优化调研

文档日期：2026-05-20  
产品名称：压力感知软件 JQ Tools  
功能模块：量尺工具  
使用技能：`brainstorm-ideas-existing`、`competitor-analysis`  
关联文档：`D:\pm-skill\shroom\量尺与画布翻转产品需求文档.md`、`D:\pm-skill\汽车定制\PRD1_1.md`

## 1. 结论先行

当前量尺功能的核心问题不是“能不能算距离”，而是“用户能不能快速、准确、可复用地完成测量”。

现有 PRD 已经定义了点距、缩放、翻转、最多 8 条量尺、回放可用等基础规则，但还缺少以下体验闭环：

| 问题 | 影响 |
|---|---|
| 入口弱 | 用户不知道什么时候可以用量尺，也不知道当前是否处于量尺模式 |
| 操作成本高 | 两点选择、微调、删除、重命名、查看详情不够顺手 |
| 结果弱 | 只看到距离，不知道横向、纵向、直线距离、坐标、单位和点距口径 |
| 管理弱 | 多条量尺没有清晰列表、颜色、锁定、隐藏、复制、批量删除 |
| 和框选割裂 | 框选是分析亮点，但量尺不能辅助框选区域尺寸判断 |
| 和回放割裂 | 回放中可用，但缺少“随帧保留/当前查看/报告引用”的规则 |
| 不能交付 | 量尺结果暂不导出、不进报告，客户看完很难沉淀结论 |

推荐方向：把量尺从“临时测距工具”升级为“空间标注与尺寸辅助分析工具”。

## 2. 竞品和行业启发

### 2.1 XSENSOR

XSENSOR 强调压力数据的测量、可视化、AI/ML 分析、用户工作流和可导出数据。它的启发不是单个量尺按钮，而是工具要服务“测量-分析-优化”闭环。

| 观察 | 对 JQ Tools 的启发 |
|---|---|
| 强调 measure、visualize、analyze、optimize | 量尺不能只是画线，应服务区域分析和报告 |
| 强调用户工作流和可操作时间序列数据 | 量尺需要进入回放、对比和报告工作流 |
| 强调导出 ready data | 量尺结果后续应能导出到报告或截图 |

### 2.2 Tekscan I-Scan

Tekscan I-Scan 强调真实压力测量、2D/3D 显示、中心力轨迹、峰值、接触面积、图形分析和每帧数据查看。它的核心是“测量工具和分析指标绑定”。

| 观察 | 对 JQ Tools 的启发 |
|---|---|
| 支持实时和历史数据图形分析 | 量尺应支持实时、回放、对比页面一致使用 |
| 每帧包含总力、中心力、峰值、压力分布 | 量尺结果应能和当前帧指标绑定 |
| Sensor MAP 体现传感器形状和尺寸 | JQ Tools 需要显示当前设备点距、矩阵尺寸、真实尺寸口径 |

### 2.3 PPS Chameleon

PPS Chameleon 强调实时分析、记录、回放、注释、视频同步、导出、复制到设计文档。它对量尺功能最有启发的是“注释和交付”。

| 观察 | 对 JQ Tools 的启发 |
|---|---|
| 支持 recording、annotating、editing | 量尺应支持命名、备注、编辑、锁定 |
| 可把可视化复制到设计文档 | 量尺结果应进入截图、报告、复制输出 |
| 可导出部分数据用于 Excel/MATLAB | 量尺可以作为局部分析边界或报告标注 |

### 2.4 Novel pliance

Novel pliance 面向汽车座椅优化，强调坐垫和靠背同步、2D/3D/isobar、多传感垫、动态压力变化和自定义分析方法。

| 观察 | 对 JQ Tools 的启发 |
|---|---|
| 面向汽车座椅优化 | 量尺应支持坐垫/靠背不同点距和对象切换 |
| 可自定义分析方法 | 量尺应能服务研发自定义测量，如腰托高度、坐垫前后距离 |
| 同步坐垫和靠背 | 量尺需要明确整体视图、坐垫、靠背的测量边界 |

## 3. 当前量尺功能诊断

### 3.1 已有能力

| 已有能力 | 当前规则 |
|---|---|
| 使用视图 | 仅 2D 数字视图下可用 |
| 测量方式 | 两点直线量尺，P1 可做矩形宽高 |
| 计算方式 | 按 `pointWidthDistance`、`pointHeightDistance` 计算 |
| 单位 | 默认 cm，内部建议 mm |
| 缩放影响 | 缩放不影响真实距离 |
| 翻转影响 | 翻转不改变真实距离 |
| 回放使用 | 回放中可用，只影响查看 |
| 多量尺 | 最多 8 条 |
| 导出 | V2.0 不导出，P2 评估导出到报告 |

### 3.2 主要缺口

| 缺口 | 说明 | 优先级 |
|---|---|---|
| 缺少量尺面板 | 多条量尺没有统一管理区 | P0 |
| 缺少端点微调 | 用户点错一个传感点后只能重画 | P0 |
| 缺少吸附规则 | 起点/终点是否吸附传感点中心不清 | P0 |
| 缺少详情信息 | 没有显示起点、终点、横向、纵向、直线距离 | P0 |
| 缺少可见状态 | 是否正在量尺模式、当前选中哪条不清晰 | P0 |
| 缺少隐藏/锁定 | 多量尺时容易误删、误拖动、遮挡数据 | P1 |
| 缺少矩形量尺 | 无法快速测量区域宽高 | P1 |
| 缺少量尺模板 | 常用座椅测量位置不能复用 | P2 |
| 缺少报告输出 | 量尺无法成为交付结论的一部分 | P2 |

## 5. 推荐优化方向

### 5.1 P0：把量尺做成稳定可用的工具

| 功能 | 说明 | 为什么要做 |
|---|---|---|
| 量尺模式状态提示 | 开启后按钮高亮，画布显示“选择起点/选择终点” | 用户知道当前在干什么 |
| 端点吸附传感点中心 | 鼠标移动时预览当前传感点，点击后落点到传感点中心 | 提高测量准确性 |
| 起点/终点可拖拽 | 选中量尺后可拖动端点重新定位 | 点错不用重画 |
| 量尺悬浮标签 | 线段中间展示直线距离，hover 展示横向/纵向距离 | 结果更直观 |
| 量尺列表面板 | 显示最多 8 条量尺，支持选中、删除、清空 | 多量尺可管理 |
| 详情信息 | 显示起点坐标、终点坐标、横向距离、纵向距离、直线距离、单位 | 结果可信 |
| 点距缺失拦截 | 无点距配置时禁用量尺并解释原因 | 避免错误结果 |

### 5.2 P1：让量尺参与分析工作流

| 功能 | 说明 | 为什么要做 |
|---|---|---|
| 矩形宽高量尺 | 拖出矩形，显示宽、高、对角线 | 辅助判断区域尺寸 |
| 与框选联动 | 从量尺矩形一键生成框选，或查看框选宽高 | 框选和量尺互补 |
| 回放量尺 | 回放中量尺保留在当前会话，可随进度查看 | 复盘更方便 |
| 对比量尺 | A/B 对比时同一量尺同时覆盖 A、B、差值图 | 对比口径一致 |
| 量尺隐藏/锁定 | 多条量尺支持隐藏、锁定、防误操作 | 降低画布混乱 |
| 坐垫/靠背对象隔离 | 量尺属于当前对象，切换对象时不混用 | 防止错误测量 |

### 5.3 P2：让量尺成为可交付资产

| 功能 | 说明 | 为什么要做 |
|---|---|---|
| 量尺命名和备注 | 例如“腰托高度”“坐垫前缘距离” | 便于报告表达 |
| 量尺模板 | 保存常用测量配置 | 重复测试更快 |
| 量尺导出到报告 | 截图中保留量尺，报告表格列出结果 | 客户交付更完整 |
| 量尺复制 | 复制量尺结果为文本 | 方便粘贴到飞书/报告 |
| 量尺历史保存 | 可选择随历史记录保存为分析标注 | 复盘可追溯 |

## 6. 推荐交互流程

### 6.1 两点直线量尺

| 步骤 | 用户动作 | 系统反馈 |
|---|---|---|
| 1 | 点击量尺按钮 | 按钮变蓝，画布出现十字光标，提示“选择起点” |
| 2 | 鼠标移动到矩阵 | 当前传感点轻微高亮，显示坐标 |
| 3 | 点击起点 | 起点标记出现，提示“选择终点” |
| 4 | 移动鼠标 | 预览线段和实时距离 |
| 5 | 点击终点 | 生成量尺，线段中间显示距离 |
| 6 | 选中量尺 | 量尺高亮，右侧面板展示详情 |
| 7 | 拖动端点 | 实时更新距离和坐标 |
| 8 | 删除或清空 | 画布和列表同步更新 |

### 6.1.1 拖拽整条量尺和移动端点

量尺编辑分为两种：拖拽整条量尺、移动单个端点。两者都必须基于矩阵坐标计算，不能基于屏幕像素保存结果。

| 编辑方式 | 用户意图 | 坐标变化 | 距离变化 | 推荐优先级 |
|---|---|---|---|---|
| 拖拽整条量尺 | 把同一段测量线移动到另一个位置 | 起点和终点一起移动 | 不变 | P0 |
| 移动起点 | 修正起点位置 | 起点变，终点不变 | 变化 | P0 |
| 移动终点 | 修正终点位置 | 终点变，起点不变 | 变化 | P0 |

#### 拖拽整条量尺

| 规则 | 说明 |
|---|---|
| 触发方式 | 鼠标按住量尺线段或距离标签后拖动 |
| 命中优先级 | 端点优先于线段；如果鼠标同时命中端点和线段，默认移动端点 |
| 位移计算 | 根据鼠标在矩阵中的 row/column 变化计算 delta |
| 新坐标 | 起点和终点同时加上 delta |
| 吸附 | 拖拽结果吸附到传感点中心，坐标保存为整数 row/column |
| 边界 | 如果起点或终点会越界，量尺停在最后合法位置 |
| 距离 | 整条拖拽不改变横向距离、纵向距离、直线距离 |
| 反馈 | 拖动时实时显示预览线和距离标签 |

#### 移动单个端点

| 规则 | 说明 |
|---|---|
| 触发方式 | 鼠标按住起点或终点圆点后拖动 |
| 固定点 | 拖动起点时终点固定；拖动终点时起点固定 |
| 吸附 | 被拖动端点吸附到最近有效传感点中心 |
| 边界 | 越界或落在无效点时不更新坐标 |
| 距离 | 拖动过程中实时更新横向、纵向、直线距离 |
| 相同点 | 起点和终点相同可显示 0，但需要提示用户重新选择 |
| 反馈 | 被拖动端点高亮，固定端点保持普通选中态 |

#### 状态保护

| 场景 | 推荐处理 |
|---|---|
| 拖拽中按 Esc | 取消本次拖拽，恢复拖拽前坐标 |
| 拖拽中切换视图 | 取消本次拖拽 |
| 拖拽中设备断开 | 保留当前画面量尺，提示设备已断开 |
| 量尺已锁定 | 不允许拖动整条量尺或端点 |
| 量尺已隐藏 | 不参与命中检测 |

### 6.2 矩形宽高量尺

| 步骤 | 用户动作 | 系统反馈 |
|---|---|---|
| 1 | 切换到矩形量尺 | 工具栏二级模式显示“直线/矩形” |
| 2 | 拖拽矩形 | 显示宽、高、对角线 |
| 3 | 松开鼠标 | 生成矩形量尺 |
| 4 | 点击“转为框选” | 使用同一坐标生成框选区域 |
| 5 | 修改边界 | 宽高和框选坐标同步更新 |

## 9. 和框选的关系

建议不要让量尺和框选完全割裂。互斥只解决鼠标冲突，但产品价值应该互补。

| 场景 | 推荐处理 |
|---|---|
| 框选已开启，再点量尺 | 提示切换工具，保留已有框选但停止编辑框选 |
| 量尺已开启，再点框选 | 提示切换工具，保留量尺但停止编辑量尺 |
| 矩形量尺转框选 | 支持一键转为框选 |
| 框选查看尺寸 | 框选详情显示宽、高、面积估算 |
| 数据对比 | 一个量尺同步展示在 A/B/差值图上 |

推荐文案：

| 场景 | 文案 |
|---|---|
| 工具切换 | 当前正在使用量尺，切换到框选后将保留量尺但停止编辑。 |
| 转框选 | 是否将当前矩形量尺转为框选区域？ |
| 框选尺寸 | 当前框选区域宽 12.5 cm，高 8.0 cm。 |

## 10. 和回放、对比、报告的关系

| 模块 | 推荐规则 |
|---|---|
| 实时 | 量尺用于当前查看，不影响采集 |
| 采集 | 采集不保存量尺为原始数据 |
| 回放 | 回放中可新建量尺，默认只保留当前会话 |
| 历史记录 | V2.1 可选择保存为历史标注 |
| 数据对比 | 同一量尺作用于 A/B/差值图，确保位置一致 |
| 报告 | V2.2 支持截图和表格导出量尺结果 |
