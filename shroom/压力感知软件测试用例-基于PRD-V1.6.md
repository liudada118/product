# 压力感知软件测试用例（基于 PRD V1.6）

文档日期：2026-05-11  
产品名称：压力感知软件 JQ Tools  
版本范围：V2.0

文档用途：给 QA、研发、产品用于功能测试、回归测试和发布验收  

## 1. 测试范围

| 模块 | 测试重点 | 是否发布阻塞 |
|---|---|---|
| 打开软件后的连接流程 | 首次配置、一键连接、重新连接、连接锁、异常提示、采集前强校验 | 是 |
| 设备数据接收与处理 | 帧解析、矩阵转换、采样率、坏帧处理、数据层隔离 | 是 |
| 连接之后的展示视图 | 2D 数字、3D 点图、3D 数字、3D 模型、缩放、视图切换 | 是 |
| 功能工具模块 | 置零、翻转/旋转、框选、量尺、放大镜、可视化调节 | 是 |
| 数据统计与图表 | 平均、最大、最小、总和、面积、重心、曲线、框选统计 | 是 |
| 采集、历史数据与回放 | 开始/停止采集、历史管理、回放、倍速、异常恢复 | 是 |
| 数据导出与数据对比 | CSV 导出、CSV 导入、字段校验、A/B 对比、差值图 | 是 |
| 全局数据规则 | rawData、displayData、storedData、exportedData 口径一致 | 是 |

## 2. 测试环境与数据准备

| 类型 | 准备项 | 说明 |
|---|---|---|
| 硬件 | 至少 1 套合法汽车坐垫设备、1 套合法汽车靠背设备 | 用于连接、展示、采集、方向、分区测试 |
| 硬件 | 至少 1 个未录入或未授权设备 | 用于非法设备、MAC 未配置、授权失败测试 |
| 串口 | CH340 串口设备 | 覆盖串口扫描和波特率探测 |
| 数据 | 正常压力帧样例 | 用于实时展示、统计、采集、导出 |
| 数据 | 坏帧、断帧、长度不匹配样例 | 用于异常处理测试 |
| 数据 | 至少 2 条同类型历史采集记录 | 用于回放和数据对比 |
| 数据 | 结构正确 CSV、legacy CSV、结构错误 CSV | 用于导入兼容和错误提示 |
| 系统 | Windows 打包版本、macOS 打包版本 | 用于发布验收 |

## 3. P0 测试用例总表

### 3.1 模块一：打开软件后的连接流程

| 用例编号 | 优先级 | 测试场景 | 测试目标 | 前置条件 | 测试步骤 | 预期结果 |
|---|---|---|---|---|---|---|
| TC-CONN-001 | P0 | 首次启动无设备配置 | 验证无配置时进入 MAC 配置页 | 清空本地设备配置 | 1. 打开软件；2. 观察启动后页面 | 自动进入 `/macConfig` 或 MAC 配置页；不能直接开始采集 |
| TC-CONN-002 | P0 | MAC 批量录入成功 | 验证 `MAC地址:类型` 可保存并离线使用 | 进入 MAC 配置页；准备合法 MAC 和类型 | 1. 输入多行 `MAC地址:类型`；2. 点击保存；3. 重启软件 | 配置保存成功；重启后进入主界面；设备配置仍存在 |
| TC-CONN-003 | P0 | MAC 格式错误 | 验证错误格式被拦截 | 进入 MAC 配置页 | 1. 输入缺少类型、重复 MAC 或非法类型；2. 点击保存 | 保存失败；提示用户修改；不写入错误配置 |
| TC-CONN-004 | P0 | 一键连接合法设备 | 验证合法设备可完成连接闭环 | 已录入合法 MAC；设备已插入 | 1. 点击一键连接；2. 等待连接完成 | 串口连接成功；显示在线；显示 MAC、类型、串口、波特率、采样率 |
| TC-CONN-005 | P0 | 连接中重复点击 | 验证连接锁和按钮置灰 | 设备已插入 | 1. 点击一键连接；2. 连接中连续点击一键连接或重新连接 | 前端按钮置灰或拦截；后端只保留一个连接任务；不会创建多个串口监听 |
| TC-CONN-006 | P0 | 无串口设备连接失败 | 验证无设备提示明确 | 拔掉所有串口设备 | 1. 点击一键连接；2. 等待结果 | 连接失败；提示未检测到设备或串口；连接锁释放；可再次点击连接 |
| TC-CONN-007 | P0 | 串口被占用 | 验证端口占用时可恢复 | 使用其他程序占用目标串口 | 1. 点击一键连接；2. 等待失败；3. 关闭占用程序后重试 | 提示串口被占用；释放临时资源；重试后可连接成功 |
| TC-CONN-008 | P0 | 波特率探测失败 | 验证候选波特率均失败后的处理 | 使用异常设备或模拟无效帧 | 1. 点击一键连接；2. 等待探测结束 | 依次尝试候选波特率；最终返回通信失败；连接锁释放；不进入在线状态 |
| TC-CONN-009 | P0 | MAC 读取失败 | 验证 MAC 失败禁止采集 | 设备连接异常或模拟 MAC 超时 | 1. 点击一键连接；2. 触发 MAC 读取失败；3. 点击采集 | 提示设备识别失败；设备不能开始采集；日志记录失败阶段 |
| TC-CONN-010 | P0 | 非法设备采集拦截 | 验证 `/startCol` 强校验 | 连接未授权或未配置 MAC 设备 | 1. 完成连接或模拟在线；2. 点击采集 | 后端拒绝采集；返回 `AUTH_FAIL` 或 `TYPE_UNKNOWN`；前端提示不可采集 |
| TC-CONN-011 | P0 | 设备类型不匹配拦截 | 验证系统类型和设备类型一致性 | 连接坐垫设备，当前系统切到靠背 | 1. 点击采集；2. 查看提示 | 拒绝采集；提示当前设备类型与系统类型不匹配 |
| TC-CONN-012 | P0 | 重新连接释放旧资源 | 验证重连前清理串口、监听器、定时器 | 设备已连接 | 1. 点击重新连接；2. 观察连接状态；3. 查看数据是否重复推送 | 旧连接被关闭；不会重复收到同一帧；重连成功后状态正常 |
| TC-CONN-013 | P0 | 采集中设备断开 | 验证断开时不丢已有采集数据 | 正在采集 | 1. 拔掉设备；2. 查看状态；3. 停止或恢复采集 | 状态进入异常或断开；保留已采集帧；提示重新连接或保存异常中断记录 |

| TC-DATA-005 | P0 | 矩阵长度不匹配 | 验证点数不匹配被阻断 | 模拟点数不等于宽 x 高 | 1. 发送长度错误数据；2. 点击采集 | 拒绝展示为有效矩阵；禁止采集；提示点阵配置异常 |
| TC-DATA-006 | P0 | 不同设备矩阵尺寸 | 验证设备类型到矩阵映射正确 | 准备 endi、carY、bed、hand 等类型数据 | 1. 分别连接或导入样例；2. 查看矩阵尺寸 | 生成对应尺寸矩阵；坐垫/靠背 key 正确 |
| TC-DATA-007 | P0 | 采样率计算 | 验证 HZ 后端计算口径 | 设备稳定发送数据 | 1. 连接设备；2. 观察 HZ；3. 导出 CSV 查看字段 | HZ 按后端帧时间计算；界面、历史、CSV 一致 |
| TC-DATA-008 | P1 | 硬件采样率记录 | 验证硬件返回值只做记录校验 | 设备返回硬件采样率 | 1. 连接设备；2. 采集并导出 CSV | `hardware_sample_rate_hz` 被记录；不覆盖主 `sample_rate_hz` |
| TC-DATA-009 | P0 | 翻转/旋转影响业务 rawData | 验证方向修正后的 rawData 作为基准 | 设备在线；有稳定压力矩阵 | 1. 执行左右翻转或 90 度旋转；2. 查看展示、统计、采集、导出 | 展示和统计使用修正后矩阵；采集和 CSV 保存修正后业务矩阵与方向 metadata |
| TC-DATA-010 | P0 | 颜色和润滑不污染 rawData | 验证显示调节不改变业务原始矩阵 | 设备在线；记录一帧 rawData | 1. 调整颜色、润滑、高度；2. 导出或查看统计 | rawData 数值不变；仅 displayData 变化 |
| TC-DATA-011 | P0 | 噪点消除数据流 | 验证噪点消除生成后端处理数据 | 设备在线；开启噪点消除 | 1. 开启噪点消除；2. 查看展示、统计、导出标记 | 不覆盖 rawData；生成 backendProcessedData；统计、采集、导出带处理标记 |
| TC-DATA-012 | P1 | WebSocket 推送失败 | 验证前端断开时后端状态稳定 | 模拟前端 WebSocket 断开 | 1. 断开前端连接；2. 继续接收设备数据；3. 恢复前端 | 后端保留最新数据；前端提示实时连接异常；恢复后继续展示 |

### 3.5 模块五：数据统计与图表

| 用例编号 | 优先级 | 测试场景 | 测试目标 | 前置条件 | 测试步骤 | 预期结果 |
