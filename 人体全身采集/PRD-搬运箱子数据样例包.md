# PRD：搬运箱子全身压力数据样例包

## 1. Summary

本 PRD 定义一个面向机器人公司的 MVP：采集人类完成“搬运箱子”任务时的全身压力数据，并通过视频与压力帧对齐回放，展示人体在真实物理任务中的接触、承重和发力变化。

第一版目标不是让机器人直接学会搬箱子，而是证明：全身压力传感衣可以稳定采集普通视频看不到的触觉/受力信息，并让机器人团队判断这些数据是否有训练、分析或仿真价值。

## 3. Background

机器人公司和具身智能团队正在大量采集视频、遥操作、动作捕捉等数据，但这些数据通常缺少一个关键维度：真实人体与环境接触时的压力和受力变化。

对于“搬运箱子”这类真实物理任务，普通视频可以看到人做了什么，但很难回答：

- 身体哪个部位开始接触物体？
- 抱起箱子时压力如何从手臂、胸腹、腿部变化？
- 2kg、5kg、10kg 的压力模式有什么差异？
- 行走和放下时身体承重如何变化？
- 哪些动作阶段可能代表风险或负荷过高？

公司已有全身压力传感衣能力：

- 采样率：100Hz
- 点位数量：1024
- 压力精度：0.01N
- 可做人身体部位映射
- 可输出标准数据格式

因此第一版产品应聚焦在“压力数据价值验证”，而不是一开始做完整的视觉、听觉、触觉三模态平台。

## 4. Objective

### 4.1 产品目标

在 30 天内完成一个“搬运箱子”数据样例包，用于向机器人公司展示全身压力数据的价值，并验证其是否愿意为定制数据采集服务付费。

### 4.2 客户目标

让机器人公司在 3 分钟内理解：

> 普通视频只能看到人做了什么；这个数据包可以看到人搬箱子时身体哪里受力、什么时候受力、压力如何变化。

### 4.3 公司目标

验证一个可商业化方向：

> 用全身压力传感衣，为机器人公司提供真实人体动作中的触觉/压力数据采集服务。

## 5. Market Segments

### 5.1 首要客户：机器人公司

目标用户：

- 机器人算法团队
- 具身智能数据团队
- 人形机器人研发团队
- 机器人仿真/训练团队

他们的核心问题：

- 有视频数据，但缺接触和受力数据。
- 有动作轨迹，但不知道动作过程中身体如何承重。
- 很难采到真实人类任务中的高频触觉数据。
- 不确定哪些人体动作数据对机器人训练有用。

他们希望完成的任务：

> 获取真实人体完成物理任务时的接触和受力数据，用于动作理解、模型训练、仿真校准、任务分析或数据可行性评估。

### 5.2 次要客户：XR/仿真训练公司

这类客户可能关心真实动作中的身体负荷、接触反馈和压力变化，用于提升训练仿真真实性。但第一版不作为主要销售对象。

### 5.3 暂不服务的客户

第一版暂不服务：

- C 端健身用户
- 普通智能穿戴消费者
- 医疗诊断场景
- 大规模标准数据集采购方

原因：这些方向会引入新的合规、体验、算法和销售复杂度，不适合第一版 MVP。

## 6. Value Propositions

### 6.1 核心价值主张

> 我们用 100Hz、1024 点位、0.01N 精度的全身压力传感衣，采集人类完成真实物理任务时的身体接触和受力过程，帮助机器人团队获得普通视频看不到的触觉/压力数据。

### 6.2 主卖点

主卖点不是“三模态采集”，而是：

> 视频看到动作，压力数据看到受力。

或者更适合销售表达：

> 让机器人团队看到人类动作背后的接触、承重和发力过程。

### 6.3 客户购买理由

机器人公司可能购买的理由：

- 普通视频无法告诉模型“哪里受力”。
- 压力数据可以补足视觉数据缺失的物理接触信息。
- 搬运、扶持、推拉、坐下起身等任务都依赖接触和承重。
- 客户自己搭建全身压力采集系统成本高、周期长。
- 先通过样例包判断数据价值，再决定是否做定制采集。

### 6.4 与替代方案对比

| 替代方案 | 客户当前能得到什么 | 缺口 |
| --- | --- | --- |
| 普通视频采集 | 动作画面 | 缺身体受力和接触强度 |
| 动作捕捉 | 姿态和轨迹 | 缺压力和接触信息 |
| 遥操作机器人数据 | 机器人自身动作 | 缺人类真实身体负荷 |
| 局部压力传感器 | 单点或局部受力 | 缺全身分布 |
| 自建采集系统 | 可定制 | 成本高、周期长、难维护 |

## 7. Solution

### 7.1 产品范围

第一版交付一个数据样例包和一个回放 demo。

产品包含：

- 搬运箱子任务采集协议
- 90 条动作样本
- 压力原始数据
- 压力清洗数据
- 第三视角视频
- 动作阶段标签
- 人体区域映射
- 数据说明文档
- 质量报告
- 视频与压力帧对齐回放 demo

产品不包含：

- 自研视频硬件
- 自研音频硬件
- 完整云端数据平台
- 机器人训练模型
- 让机器人直接复现动作
- 医疗诊断结论
- 大规模商业数据集

### 7.2 采集任务设计

任务名称：搬运箱子

任务描述：

被试从起点站立，双手接触箱子，将箱子抱起，行走到目标点，将箱子放下，并结束任务。

采集规模：

| 项目 | 要求 |
| --- | --- |
| 被试人数 | 3 人 |
| 箱子重量 | 2kg、5kg、10kg |
| 每人每档重量重复次数 | 10 次 |
| 总样本数 | 90 条 |
| 单条样本时长 | 建议 8-20 秒 |

动作阶段标签：

- start：任务开始
- contact_box：身体/手臂接触箱子
- lift：箱子离开地面
- walk：抱箱行走
- put_down：箱子开始放下
- end：任务结束

每条样本必须包含：

| events.json | 动作阶段标签 |
| video.mp4 | 第三视角视频 |
| preview.mp4 | 可选，样本预览视频 |

### 7.4 回放功能

回放功能是样例包的核心展示工具。

它必须让客户直观看到：

> 视频播放到某个动作瞬间时，压力数据也显示同一时刻的受力状态。

最低功能：

- 打开一个样本。
- 左侧播放第三视角视频。
- 右侧显示压力热力图或人体区域压力曲线。
- 底部显示时间轴。
- 时间轴显示动作阶段标签。
- 播放、暂停、拖动时，视频和压力数据同步变化。
- 暂停时显示当前时间、视频帧、压力帧编号。
- 可切换不同重量样本进行观察。

第一版可视化形式：

1. 人体区域压力曲线
   - 胸部
   - 背部
   - 肩部
   - 手臂
   - 腿部

2. 人体区域热力图
   - 如果已有精确 2D 点位映射，则显示 1024 点热力图。
   - 如果暂时没有精确 2D 映射，则显示按人体区域聚合的热力图。

回放验收要求：

- 视频与压力数据同步误差 < 500ms。
- 拖动时间轴后，视频和压力图一起跳转。
- 当前动作标签随时间正确变化。
- 能导出 1-2 分钟客户 demo 视频。

### 7.5 数据格式

第一版优先支持：

- CSV

后续根据客户需求扩展：

pressure_clean.csv 建议字段：

| region_shoulder | 肩部区域聚合压力 |
| region_arm | 手臂区域聚合压力 |
| region_leg | 腿部区域聚合压力 |

### 7.6 Acceptance Criteria

#### 7.6.1 数据采集验收

- 采集 90 条搬运箱子样本。
- 至少 80 条样本通过质量检查。
- 每条样本都有压力数据、视频、标签、元数据。
- 压力数据采样率达到 100Hz，允许 ±5% 波动。
- 每条压力数据都有完整时间戳。
- 单条样本缺失帧比例 < 1%。
- 1024 点位中不可用点位比例 < 2%。
- 压力值单位明确，按 0.01N 精度输出。
- 2kg、5kg、10kg 样本之间能看到可解释的压力差异。

#### 7.6.2 标签验收

- 每条样本都有 6 个核心标签：start、contact_box、lift、walk、put_down、end。
- 标签命名统一。
- 人工抽检 20 条样本，标签与视频动作误差 < 500ms。
- 标签文件可被回放工具读取。

#### 7.6.3 视频同步验收

- 每条样本都有对应 video.mp4。
- 视频文件和压力数据文件一一对应。
- 回放工具中视频与压力帧同步误差 < 500ms。
- 暂停时可以看到当前压力帧编号。
- 拖动时间轴后，视频画面和压力可视化同步跳转。

#### 7.6.4 回放 demo 验收

- 可以打开任意一条合格样本。
- 可以播放、暂停、拖动。
- 可以看到视频、压力可视化、时间轴、动作标签。
- 可以切换不同重量样本。
- 可以导出 1-2 分钟 demo.mp4。
- 客户看 demo 时，能在 3 分钟内理解压力数据相对普通视频的增量价值。

#### 7.6.5 文档验收

- README.md 能说明这个数据集是什么、怎么打开。
- data_schema.md 能说明每个字段的含义、单位和格式。
- collection_protocol.md 能说明采集场地、箱子重量、相机位置、动作流程。
- quality_report.md 能说明采集数量、合格数量、缺失率、同步误差、异常样本。
- dataset-card.md 能说明数据用途、限制、授权和隐私边界。

#### 7.6.6 客户验证验收

- 约访至少 10 家机器人相关公司。
- 至少 5 家愿意看 demo。
- 至少 3 家明确说出该数据可能有用的场景。
- 至少 2 家愿意继续讨论付费定制采集。
- 至少 1 家给出下一步希望采集的具体任务清单。

### 7.7 Non-Goals

第一版不承诺：

- 机器人可以直接用该数据学会搬箱子。
- 数据能直接提升某个模型指标。
- 数据可用于医疗诊断。
- 数据包含完整三模态平台能力。
- 数据已经覆盖所有机器人搬运动作。

第一版只验证：

> 全身压力数据能否作为机器人真实动作数据中的一个有价值模态，被客户理解并愿意继续付费验证。

### 7.8 Assumptions

当前未验证但需要验证的假设：

- 机器人公司认为人体全身压力数据有价值。
- 搬运箱子是机器人公司容易理解的起始场景。
- 500ms 同步误差足以用于第一版销售 demo。
- 客户愿意先看样例包，再讨论定制采集。
- 100Hz、1024 点位、0.01N 精度足以形成差异化。
- 客户愿意接受 CSV/JSON 作为第一版交付格式。

## 9. Open Questions

需要后续确认的问题：

1. 是否已有 1024 点位到人体区域的稳定映射？
2. 压力原始数据当前输出格式是什么？
3. 视频同步用什么方法：拍手、蜂鸣器、时间码，还是软件同步？
4. 是否需要采集音频？如果采集，第一版是否只作为辅助信息？
5. 被试的身高、体重、性别等信息是否需要匿名化？
6. 数据是否允许对外展示？是否需要签署被试授权？
7. 客户 demo 是网页工具、桌面工具，还是导出视频即可？
8. 第一批访谈机器人公司名单从哪里来？

## 10. Definition of Done

- `carry-box-dataset-v0.1` 数据包目录结构完整。
- 至少 80 条合格样本可打开、可读、可追溯。
- 每条合格样本都有压力数据、视频、标签、元数据。
- 回放 demo 可以同步展示视频和压力变化。
- 1-2 分钟客户展示视频已导出。
- README、schema、采集协议、质量报告齐全。
- 内部成员可以用 3 分钟讲清楚主卖点。
- 已准备好客户访谈问题和反馈记录表。
