# 硬件清单：搬运箱子数据样例包采集系统

## 1. 结论

第一版 MVP 可以做到“压力衣 + 视频 + 可选音频”同时采集，并通过软件时间戳和同步事件完成对齐。

第一版不要追求电影级或工业级帧同步。目标是：

- 压力数据稳定采集：100Hz、1024 点位
- 视频能清楚看到动作
- 压力帧和视频帧能回放对齐
- 同步误差小于 500ms
- 能生成客户可理解的 demo

后续客户试点再升级到硬件触发、timecode 或 ROS2。

## 2. MVP 推荐硬件清单

| 模块 | 推荐硬件 | 数量 | 作用 | 软件接入方式 |
| --- | --- | ---: | --- | --- |
| 压力衣 | 现有全身压力传感衣 | 1 套 | 采集全身压力数据 | 自研 SDK、串口、USB、BLE、Wi-Fi 或 Ethernet |
| 采集电脑 | Windows/Linux 笔记本或小主机 | 1 台 | 统一启动采集、保存数据 | Python/C++/Node 均可 |
| 主相机 | Intel RealSense D455/D457 | 1 台 | 第三视角 RGB 视频，可选深度 | RealSense SDK / ROS2 |
| 备选相机 | USB 工业相机或 UVC 摄像头 | 1 台 | 只采 RGB 视频 | OpenCV / UVC / 厂商 SDK |
| 麦克风 | USB 麦克风 | 1 个 | 记录环境声音和蜂鸣同步点 | PortAudio / sounddevice |
| 同步触发器 | Arduino 或 ESP32 | 1 个 | 控制 LED 闪烁、蜂鸣器、同步日志 | Serial 串口 |
| LED 灯 | 高亮 LED 或小灯板 | 1 个 | 视频同步锚点 | 由 Arduino/ESP32 控制 |
| 蜂鸣器 | 有源蜂鸣器 | 1 个 | 音频同步锚点 | 由 Arduino/ESP32 控制 |
| 三脚架 | 普通三脚架 | 1-2 个 | 固定相机 | 无 |
| 存储 | 1TB SSD | 1 个 | 存储视频和压力数据 | 文件系统 |

## 3. SDK 与接口判断

### 3.1 压力衣

压力衣是主设备，必须能被采集软件控制。

最低要求：

- 可以启动采集
- 可以停止采集
- 每帧输出时间戳
- 每帧输出 1024 点压力值
- 可以记录同步事件，或至少能在采集日志中写入同步时间点

建议接口优先级：

1. Ethernet：最适合稳定采集
2. USB：开发方便
3. Wi-Fi：可用，但要注意丢包和延迟
4. BLE：不建议传 100Hz x 1024 点完整数据，带宽可能不够

### 3.2 相机

第一版推荐 RealSense D455/D457。

原因：

- 可通过 SDK 采集 RGB 帧
- 可选深度数据
- 机器人客户容易理解
- 后续可接 ROS2

官方 SDK：

- RealSense SDK/librealsense：https://github.com/IntelRealSense/librealsense
- ROS2 realsense2_camera：https://docs.ros.org/en/ros2_packages/humble/api/realsense2_camera/

注意：

- 第一版只需要 RGB 视频即可。
- 深度数据是加分项，不是第一版核心卖点。
- 如果用 RealSense 的帧时间戳，软件侧仍然要记录本机接收时间，便于后处理排查。

### 3.3 工业相机备选

如果要更稳定的硬件同步，后续可以用 Basler 或 Teledyne FLIR 工业相机。

Basler：

- pylon SDK 支持 C、C++、.NET。
- 支持软件触发、硬件触发、相机时间戳。
- 官方文档：https://docs.baslerweb.com/pylonapi/index

Teledyne FLIR：

- Spinnaker SDK 支持工业相机采集。
- 官方页面：https://www.teledynevisionsolutions.com/products/spinnaker-sdk

第一版不一定需要工业相机。只有当客户要求更低同步误差、更稳定帧率、更强触发能力时再升级。

### 3.4 GoPro / Insta360

这类相机适合拍摄好看的 demo，但不建议作为第一版主采集相机。

原因：

- 通常更适合控制录制和取文件
- 不一定方便逐帧实时进采集软件
- 精确时间戳和同步链路更复杂

GoPro 有 Open GoPro API：

- https://gopro.github.io/OpenGoPro/

如果只是拍客户展示视频，可以用。若要做可复现数据采集，优先用 RealSense、工业相机或 USB 摄像头。

### 3.5 音频

第一版音频不是核心卖点。

用途：

- 记录蜂鸣同步点
- 记录箱子落地、碰撞、脚步等环境事件

建议：

- 使用普通 USB 麦克风
- 采集 WAV
- 保存音频开始时间戳
- 可选接入回放工具

## 4. 同步方案

### 4.1 MVP 同步方案

使用“软件时间戳 + 可见/可听同步事件”。

流程：

1. 采集软件创建样本目录。
2. 用户选择被试编号、箱子重量、任务编号。
3. 点击开始采集。
4. 压力衣开始记录压力帧。
5. 相机开始录制视频。
6. 麦克风开始录音，可选。
7. 采集软件通过串口控制 Arduino/ESP32。
8. LED 闪 3 次，视频能看到。
9. 蜂鸣器响 3 次，音频能听到。
10. 同步事件写入 `sync_events.json`。
11. 采集结束后，后处理对齐压力、视频、音频时间轴。

第一版验收目标：

- 视频和压力同步误差 < 500ms
- 能在回放工具里同步播放
- 能通过 LED 闪烁点人工校验同步

### 4.2 后续工程化同步方案

客户试点后，如果需要更高同步精度，再升级：

- 工业相机硬件 trigger
- 相机 timecode
- PTP 网络时间同步
- ROS2 bag 统一记录
- 多设备共用硬件同步脉冲

目标可以提升到：

- <100ms
- <50ms
- 帧级同步

## 5. 采集软件需要连接哪些设备

第一版采集软件至少连接：

1. 压力衣
2. 相机
3. Arduino/ESP32 同步触发器

可选连接：

4. USB 麦克风
5. 深度流
6. 第二相机

采集软件要做的事：

- 统一开始/停止
- 给每路数据写时间戳
- 保存设备状态
- 写入同步事件
- 自动生成样本目录
- 采集失败时给出错误提示

## 6. 样本目录输出

每次采集结束后生成：

```text
carry_box_s01_5kg_003/
  pressure_raw.csv
  pressure_clean.csv
  video.mp4
  audio.wav
  events.json
  sync_events.json
  metadata.json
  capture_log.json
```

`capture_log.json` 至少记录：

- 采集开始时间
- 采集结束时间
- 压力衣设备编号
- 相机设备编号
- 采样率
- 视频帧率
- 丢帧数量
- 同步事件时间
- 异常信息

## 7. 采购优先级

### 必买/必备

- 压力衣
- 采集电脑
- RealSense D455/D457 或 USB 相机
- Arduino/ESP32
- LED
- 蜂鸣器
- 三脚架
- SSD

### 可以后买

- 工业相机
- 多相机同步设备
- 专业 timecode 设备
- 动作捕捉系统
- 专业录音设备

### 第一版不建议买

- 昂贵电影机
- 专业动捕系统
- 多台复杂工业相机
- 自研视频/音频硬件
- 大型云端存储和标注平台

## 8. 推荐第一版组合

如果只选一套，推荐：

- 压力衣：现有设备
- 相机：RealSense D455 或 D457
- 音频：USB 麦克风
- 同步：ESP32 + LED + 蜂鸣器
- 软件：Python 采集控制台
- 存储：本地 SSD

这个组合足够完成：

- 搬运箱子样本采集
- 视频与压力数据同步
- 回放 demo
- 客户验证
