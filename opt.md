本文基于当前 PRD：`PRD-JQ-Tools-V2-Development.md`，对照实现文档：`shroom/FUNCTION_IMPLEMENTATION.md`，按产品 PRD 的六个模块重新整理需要优化、补充或修正的地方。

标记说明：

- <span style="color:#d32f2f"><strong>【需修改】</strong></span>：当前实现文档需要重点补充或修正。
- <strong>优先级 P0</strong>：影响研发理解、测试验收、数据可信度或交付风险，建议优先处理。
- <strong>优先级 P1</strong>：影响文档完整性、后续扩展或跨模块理解，建议本版补齐。
- <strong>优先级 P2</strong>：不阻塞当前交付，但建议后续优化。

### 2.4 可视化调节是否只影响展示

对应实现文档位置：

- `9. 可视化调节`
- `7. 前端矩阵处理`
- `settingValue`
- localStorage 配置

当前情况：

实现文档写了高斯润滑、颜色调节、噪点消除、高度调节、响应速度调节，但没有明确这些参数影响哪一层数据。

<span style="color:#d32f2f"><strong>【需修改】PRD 原则是可视化调节不能污染原始采集数据。实现文档必须明确这些调节只影响展示，还是也影响统计、采集、导出。</strong></span>

建议新增数据层说明：

| 数据层 | 说明 |
|---|---|

| `calculatedData` | 用于统计指标，需要说明是否受置零、滤波、框选影响。 |
| `storedData` | 采集入库数据，需要说明保存原始矩阵还是显示矩阵。 |
| `exportedData` | CSV 导出数据，需要说明与入库数据是否一致。 |

优先级：**P0**

## 模块三：连接之后的功能工具模块

对应 PRD 模块：预压力置零、框选、量尺、画布翻转、放大镜、可视化编辑、视角切换。

### 3.1 预压力置零

对应实现文档位置：

- `9. 可视化调节`
- `pageInfo.changeWsLocalData()`
- 置零按钮相关逻辑

当前情况：

实现文档只说明预压力置零通过 `pageInfo.changeWsLocalData()` 实现，没有解释数据流。

<span style="color:#d32f2f"><strong>【需修改】预压力置零是数据可信度核心功能，必须写清楚它对显示、统计、采集、导出的影响。</strong></span>

建议补充：

| 问题 | 需要在实现文档中明确 |
|---|---|
| 置零时保存什么 | 是否保存一份 baseline 基线矩阵。 |
| 后续数据如何处理 | 是否每一帧都减去 baseline。 |
| 是否影响统计 | 平均压强、最大压强、压力总和是否基于置零后数据。 |
| 是否影响采集 | 入库保存原始值，还是置零后值。 |
| 是否影响 CSV | CSV 导出原始值，还是置零后值。 |
| 取消置零 | 是否恢复原始数据，baseline 是否清空。 |

建议在文档中增加流程：

```text
设备原始矩阵 -> 记录置零基线 -> 当前帧减基线 -> 展示矩阵 -> 统计矩阵 -> 采集/导出
```

优先级：**P0**

<span style="color:#d32f2f"><strong>【需修改】PRD 要求默认始终保存全量原始数据。实现文档必须明确框选采集不会替代全量矩阵保存。</strong></span>

|---|---|

## 模块四：连接之后的数据可视化和计算数据

对应 PRD 模块：平均压强、最大压强、最小压强、压力总和、受压面积、传感点数、压力重心、正态分布、趋势图表、框选统计。

### 4.1 统计指标公式和单位

对应实现文档位置：

- `7. 前端矩阵处理`
- `12.2 统计字段`

当前情况：

实现文档列出了 `pressAver`、`pressMax`、`pressMin`、`pressTotal`、`areaTotal`、`pointTotal`、`pressureCenter`、`μ`、`Var`、`Skew`、`Kurt` 等字段，但没有逐项公式。

<span style="color:#d32f2f"><strong>【需修改】统计指标必须补充公式、单位、是否排除 0、是否基于框选区域，否则 QA 无法验收，客户也无法理解数据含义。</strong></span>

建议新增指标字典：

| 指标 | 需要补充 |
|---|---|
| 平均压强 `pressAver` | 公式、单位、是否排除 0。 |
| 最大压强 `pressMax` | 是否基于全量/框选、坐标如何记录。 |
| 最小压强 `pressMin` | 是否排除 0，建议明确为非零最小值。 |
| 压力总和 `pressTotal` | 是否所有有效点求和，单位是什么。 |
| 受压面积 `areaTotal` | 是点数面积还是物理面积，点距如何换算。 |
| 传感点数 `pointTotal` | 是总点数、有效点数，还是受压点数。 |
| 压力重心 `pressureCenter` | X/Y 公式、坐标原点、是否受翻转影响。 |
| 正态分布指标 | μ、方差、偏度、峰度的统计对象和计算窗口。 |

优先级：**P0**

<span style="color:#d32f2f"><strong>【需修改】这是强业务规则，必须产品化解释，否则研发、测试、客户交付都会难以判断数据是否正确。</strong></span>

| 需要补充的问题 | 建议写法 |
|---|---|
