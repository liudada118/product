# 使用传感器做舒适性评估的调研

日期：2026-05-14

## 1. 结论摘要

使用传感器进行舒适性评估是可行的，但不应只依赖单一传感器或单一指数。当前较成熟的路线是“环境传感器 + 人体/行为传感器 + 主观反馈标签 + 标准模型/机器学习模型”的融合方案。

最适合作为第一阶段落地的方向是热舒适与室内环境舒适，因为它们有较明确的标准、指标和开源计算工具。座椅、穿戴、交通座舱等场景也可做，但需要更多人体差异、姿态、振动和长时间疲劳数据。

推荐先做一个 4-8 周的小规模试点：部署温湿度、黑球温度或辐射温度、风速、CO2、PM2.5、照度、噪声、座椅压力或 IMU、可穿戴心率/皮温等传感器；同时用手机或手表收集简短主观投票。先建立 PMV/PPD、SET、IAQ、声光舒适、姿态/振动等分项评分，再训练个人化舒适模型。

## 2. 舒适性评估对象

舒适性不是一个单一物理量，而是人对环境、身体状态和任务场景的综合主观感受。可拆成以下维度：

| 维度 | 典型场景 | 关键传感器 | 常用指标 |
| --- | --- | --- | --- |
| 热舒适 | 办公室、教室、座舱、住宅 | 空气温度、相对湿度、平均辐射温度、风速、皮肤温度、心率/HRV | PMV、PPD、SET、热感觉投票 TSV、热偏好 |
| 空气舒适/室内空气质量 | 室内空间、车舱、睡眠空间 | CO2、PM2.5、TVOC、甲醛、温湿度、通风状态 | IAQ 评分、CO2 超标时间、污染物暴露量 |
| 声舒适 | 办公、居住、交通 | 声级计/麦克风、频谱、事件噪声 | dBA、LAeq、峰值噪声、语音干扰 |
| 光舒适 | 办公、教室、驾驶舱 | 照度、色温、眩光、屏幕亮度、日照 | 照度合规、眩光风险、昼夜节律照明指标 |
| 姿态/压力舒适 | 座椅、床垫、人体工学产品 | 压力阵列、IMU、摄像头/深度相机、肌电 | 压力分布、姿态变化频率、局部压强、疲劳代理指标 |
| 振动/乘坐舒适 | 汽车、轨道交通、工程机械 | 三轴加速度计、陀螺仪、座椅/地板振动传感器 | 频率加权 RMS、VDV、ISO 2631、Sperling 指数 |
| 生理舒适/压力 | 长时间工作、驾驶、睡眠 | PPG/ECG、HRV、EDA、皮温、呼吸、眼动 | 压力水平、疲劳、觉醒度、舒适偏好预测 |

## 3. 主要标准和参考框架

### 3.1 热舒适

ASHRAE Standard 55-2023 是建筑热舒适中最常用的标准之一，关注环境因素：温度、热辐射、湿度、风速；以及个人因素：活动代谢率和服装热阻。其核心用途是判断室内热环境是否能被多数健康成年人接受。

ISO 7730 使用 PMV/PPD 体系评估热环境。PMV 预测群体平均热感觉，PPD 预测不满意比例。它适合机械空调、相对稳定的室内场景，但对个体差异、动态环境和非均匀环境的解释能力有限。

EN 16798-1 把室内环境输入参数扩展到热环境、室内空气质量、照明和声学，适合作为多维 IEQ 评估框架。

### 3.2 室内环境质量

IEQ 通常包括 thermal comfort、indoor air quality、acoustic comfort、visual comfort 四大类。近期综述指出，舒适性评估不应只看环境测量，还要结合主观感知与生理响应；机器学习模型依赖连续数据，但真实场景中的数据缺失、标注不足和个体差异仍是主要限制。

WELL Building Standard 也把空气、光、声、热舒适等作为建筑健康和福祉评估的重要维度，并支持部分策略通过连续传感器监测来验证。

### 3.3 乘坐/振动舒适

ISO 2631 系列用于全身振动暴露评估。ISO 2631-4:2001 面向固定导向交通系统中的乘客和乘务人员舒适性，强调根据运动环境评价相对乘坐舒适性。工程落地中通常使用座椅或地板三轴加速度、频率加权 RMS、振动剂量值等指标。

## 4. 传感器方案

### 4.1 最小可行传感器组合

如果目标是快速验证舒适性评估，建议先采用：

| 模块 | 传感器 | 目的 |
| --- | --- | --- |
| 热环境 | 温湿度、黑球温度或辐射温度、风速 | 计算 PMV/PPD、SET，判断热舒适边界 |
| 空气质量 | NDIR CO2、PM2.5、TVOC | 判断闷、异味、通风不足等空气舒适问题 |
| 声光环境 | dBA 噪声、照度/色温 | 建立声舒适和视觉舒适分项 |
| 人体状态 | 手环 PPG、皮温、EDA 可选 | 捕捉个体化热反应、压力、疲劳 |
| 姿态行为 | 座椅压力垫、IMU、门窗/空调控制事件 | 判断人体工学、活动水平和主动调节行为 |
| 主观反馈 | 手机/手表微问卷 | 建模标签，避免“只有传感器无真值” |

第一阶段不建议上来就使用 EEG、复杂肌电或高成本热成像作为主传感器。它们能提供研究价值，但佩戴负担、成本、隐私和场景适配问题更大。

### 4.2 采样与布点

环境传感器应布在人实际停留区域，而不是只布在墙面或空调回风口。热舒适测量要尽量靠近人体活动高度；座舱或座椅场景要区分头部、胸腹、脚部和座面区域。

建议采样频率：

| 数据类型 | 频率建议 |
| --- | --- |
| 温湿度、CO2、PM2.5、照度 | 0.1-1 Hz 通常足够 |
| 风速、噪声 | 1 Hz 或事件级统计 |
| IMU/振动 | 50-200 Hz，按 ISO/场景需要调整 |
| PPG/HRV、EDA、皮温 | 1-64 Hz，取决于设备 |
| 主观反馈 | 每 30-120 分钟一次，或环境突变后触发 |

## 5. 模型路线

### 5.1 标准模型路线

先使用标准模型建立基线：

1. 热舒适：PMV、PPD、SET、adaptive comfort。
2. 空气舒适：CO2、PM2.5、TVOC、温湿度超限时间与暴露量。
3. 声舒适：等效连续声级 LAeq、峰值噪声、事件噪声频次。
4. 光舒适：照度、色温、眩光代理指标、日照过强时段。
5. 振动舒适：三轴加速度、频率加权 RMS、VDV、ISO 2631 相关方法。

可直接使用 CBE Thermal Comfort Tool 或 pythermalcomfort 计算 PMV/PPD、SET、adaptive comfort 等热舒适指标。

### 5.2 数据驱动路线

在有足够主观标签后，建立机器学习模型：

| 模型目标 | 输入 | 输出 |
| --- | --- | --- |
| 群体舒适预测 | 环境传感器 + 场景信息 | 舒适/不舒适、冷热/闷/吵/刺眼 |
| 个人化舒适预测 | 环境 + 生理 + 历史反馈 | 个人热偏好、控制建议 |
| 异常检测 | 时间序列传感器 | 某个区域长期不舒适或设备异常 |
| 控制优化 | 舒适评分 + 能耗 + 控制动作 | HVAC/风扇/遮阳/座椅调节策略 |

建议先用可解释模型作为基线，例如逻辑回归、随机森林、XGBoost，再考虑 LSTM/Transformer 等深度时序模型。舒适性数据通常样本少、噪声大，先追求可解释和稳定。

### 5.3 推荐评分体系

可以构建一个 0-100 的综合舒适评分：

```
综合舒适分 = 0.35 * 热舒适分
          + 0.20 * 空气舒适分
          + 0.15 * 声舒适分
          + 0.15 * 光舒适分
          + 0.15 * 姿态/振动/生理舒适分
```

权重应按场景调整。办公建筑更重视热、空气、声光；座椅和床垫更重视压力、姿态、皮温和长时间疲劳；车辆更重视热、振动、噪声和晕动风险。

## 6. 研究发现

### 发现 1：热舒适最适合作为起点

信心：高。

原因是热舒适已有成熟标准和工具，传感器数据也相对容易获取。PMV/PPD 可以作为群体基线，但不能直接代表个人偏好。近期研究强调，皮温、心率变异性、EDA、行为反馈等生理/行为数据可提高个人化热舒适预测能力。

### 发现 2：只测环境不足以评估“人是否舒服”

信心：高。

同样的温度、湿度、噪声和光照，对不同人、不同任务和不同状态会产生不同感受。舒适性本质上需要主观标签校准。没有主观反馈的系统，只能评估“环境是否达标”，不能可靠判断“人是否舒服”。

### 发现 3：可穿戴传感器有价值，但会带来接受度和隐私问题

信心：中高。

PPG、HRV、EDA、皮温等可用于识别热压力、紧张、疲劳和个体偏好。但企业办公、车舱、医疗康复等场景对隐私敏感，需要明确数据最小化、边缘计算、匿名化和用户授权机制。

### 发现 4：多维舒适要避免“强行合成一个分数”

信心：中高。

综合评分便于展示，但实际诊断时应保留分项。否则可能出现“总体分数不错，但用户因噪声或局部过热明显不满”的情况。产品上应显示主要不舒适来源和建议动作。

### 发现 5：传感器质量和布点比算法更容易决定成败

信心：高。

低成本传感器存在漂移、交叉敏感、安装位置偏差和响应滞后。CO2 建议优先使用 NDIR；TVOC 传感器更适合趋势判断而非精确定量；温度传感器要避开直射阳光、设备热源和空调直吹。

## 7. 建议试点方案

### 7.1 目标

建立一个可解释、可复现的舒适性评估原型，回答三个问题：

1. 当前环境是否满足标准舒适范围？
2. 用户实际是否感觉舒适？
3. 哪些传感器组合能以较低成本预测不舒适？

### 7.2 样本与周期

建议招募 15-30 人，连续采集 4-8 周。若是办公或教室场景，至少覆盖早中晚、晴雨天、空调开启/关闭、人员密度变化等条件。若是座椅/车舱场景，应覆盖不同体型、性别、穿着、姿态和使用时长。

### 7.3 数据表结构

| 字段 | 示例 |
| --- | --- |
| timestamp | 2026-05-14 10:00:00 |
| user_id_hash | 匿名用户 ID |
| zone_id | 房间、座位、车舱区域 |
| air_temp_c | 空气温度 |
| globe_temp_c / mrt_c | 黑球温度或平均辐射温度 |
| rh_pct | 相对湿度 |
| air_speed_mps | 风速 |
| co2_ppm / pm25 / tvoc | 空气质量 |
| lux / cct | 光环境 |
| dba | 噪声 |
| pressure_features | 座椅压力分布特征 |
| imu_features | 振动/姿态特征 |
| hr / hrv / skin_temp / eda | 生理特征 |
| subjective_thermal_vote | -3 到 +3 |
| subjective_comfort | 1 到 7 或舒适/不舒适 |
| preference | 希望更冷/不变/更热等 |

### 7.4 评估指标

| 目标 | 指标 |
| --- | --- |
| 标准合规 | PMV 在目标范围内的时间占比、PPD、SET、IAQ 超限时长 |
| 预测准确性 | Accuracy、F1、AUROC、MAE |
| 个人化效果 | 个人模型相对群体模型提升 |
| 传感器价值 | 去掉某类传感器后的性能下降 |
| 用户体验 | 反馈完成率、佩戴中断率、隐私顾虑反馈 |

## 8. 产品化建议

1. 第一版先做“监测 + 解释 + 建议”，不要直接闭环控制。
2. 展示分项评分和主要原因，例如“偏热：辐射温度高 + 风速低”。
3. 主观反馈尽量轻量化，例如三键：太冷、正好、太热；或不舒服原因多选。
4. 个人化模型需要冷启动机制，前 1-2 周以标准模型为主，随后逐步按个人反馈校准。
5. 数据隐私默认采用匿名 ID；生理数据只做特征化，不展示原始波形。
6. 对外报告用标准模型，对内优化用个人化模型。

## 9. 风险与限制

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 主观标签不足 | 模型无法知道真实舒适感 | 微问卷、事件触发反馈、低打扰采样 |
| 传感器漂移 | 分数不稳定 | 定期校准、冗余传感器、异常检测 |
| 个体差异大 | 群体模型误判 | 个人化校准、分人群建模 |
| 隐私顾虑 | 用户拒绝佩戴或反馈 | 数据最小化、匿名化、边缘计算 |
| 场景迁移差 | 一个办公室模型不能直接用于车舱/座椅 | 分场景建模，保留标准指标基线 |
| 综合分误导 | 掩盖局部不舒适 | 同时输出分项与主要不舒适原因 |
