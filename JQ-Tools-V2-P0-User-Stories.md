# JQ Tools V2.0 P0 用户故事

## 1. 范围说明

本文把 JQ Tools V2.0 P0 功能拆成研发可执行的用户故事。范围以 V2.0 主线闭环为准：设备连接、授权校验、设备参数、预压力清零、实时 2D 可视化、采集、统计、框选、历史回放、CSV 导出、本地缓存和异常恢复。

V2.0 暂不包含：3D 模型上传、双组数据对比、CSV 导入、macOS、多语言、复杂报告、多量尺管理。

## 2. 用户角色

| 角色 | 说明 |
|---|---|
| 测试工程师 | 在研发或实验室中连接设备、采集数据、回放并导出 |
| 展示人员 | 在展厅或门店中打开软件并展示实时压力图 |
| 科研用户 | 关注数据可信度、记录完整性、导出格式和复现实验 |
| 交付/售后人员 | 负责设备安装、授权校验、故障排查和客户现场支持 |

## 3. Epic A: 软件安装与启动

### Story A1: Windows 安装包

**Description:** 作为交付人员，我希望 JQ Tools 可以通过 Windows 安装包安装，所以客户不用配置复杂环境就能使用软件。

**Design:** 暂无。

**Acceptance Criteria:**
1. 用户可以在 Windows 系统上运行安装包。
2. 安装完成后桌面或开始菜单中出现 JQ Tools 启动入口。
3. 软件首次启动时不要求联网。
4. 安装失败时显示明确错误信息。
5. 卸载后不会删除用户已采集的数据，除非用户明确选择删除。

### Story A2: 中文界面

**Description:** 作为国内用户，我希望软件默认显示中文，所以我可以直接理解按钮、状态和错误提示。

**Design:** 暂无。

**Acceptance Criteria:**
1. 软件默认语言为中文。
2. 主流程按钮显示中文，包括“一键连接”“采集”“停止”“下载”“回放”。
3. 设备连接、采集、导出、错误提示均显示中文。
4. 不出现中英文混杂的关键操作文案。
5. 中文文案不超出按钮、弹窗和表格边界。

## 4. Epic B: 设备连接与设备参数

### Story B1: 一键连接设备

**Description:** 作为测试工程师，我希望点击“一键连接”后软件自动连接压力采集设备，所以我不用手动选择复杂串口参数。

**Design:** 暂无。

**Acceptance Criteria:**
1. 未连接设备时，“一键连接”按钮可点击。
2. 点击后软件开始扫描可用串口。
3. 找到可用设备后显示“已连接”状态。
4. 连接成功后显示设备 MAC 地址和设备类型。
5. 连接失败时显示失败原因和重试入口。
6. 扫描期间按钮显示连接中状态，避免重复点击。

### Story B2: 串口自动检测

**Description:** 作为测试工程师，我希望软件自动检测 CH340 串口设备，所以我不用在系统设备管理器中查端口。

**Design:** 暂无。

**Acceptance Criteria:**
1. 软件可以识别可用串口列表。
2. 软件优先识别 CH340 相关串口设备。
3. 未检测到设备时显示“未检测到设备”。
4. 检测到多个串口时，软件只连接符合协议响应的设备。
5. 连接日志中记录使用的串口号。

### Story B3: 波特率自动尝试

**Description:** 作为测试工程师，我希望软件能自动尝试 921600、1000000、3000000 三个波特率，所以不同硬件版本都能连接。

**Design:** 暂无。

**Acceptance Criteria:**
1. 软件按预设顺序尝试 921600、1000000、3000000。
2. 某个波特率握手成功后停止继续尝试。
3. 连接成功后记录实际波特率。
4. 全部失败时提示“波特率探测失败”。
5. 探测过程有超时限制，不能无限等待。

### Story B4: 读取设备 MAC 地址

**Description:** 作为交付人员，我希望插入设备后软件读取 MAC 地址，所以我可以确认设备身份和授权状态。

**Design:** 暂无。

**Acceptance Criteria:**
1. 连接成功后软件自动读取设备 MAC 地址。
2. MAC 地址显示在设备信息区。
3. MAC 地址写入本次采集记录。
4. 读取失败时显示明确错误。
5. 软件不允许手工修改设备 MAC 地址。

### Story B5: 设备类型选择

**Description:** 作为测试工程师，我希望连接后选择设备类型，所以软件能用正确的矩阵、方向和显示模板处理数据。

**Design:** 暂无。

**Acceptance Criteria:**
1. 软件支持选择设备类型：手-坐垫、床-床垫、汽车坐垫、汽车靠背、工学椅-坐垫、工学椅-靠背。
2. 设备类型变更后，热力图布局和分区名称同步更新。
3. 采集开始后不允许随意切换设备类型。
4. 设备类型写入采集记录和 CSV 文件。
5. 未选择设备类型时，软件提示用户选择后再采集。

### Story B6: 本地离线授权/固件校验

**Description:** 作为交付人员，我希望软件离线校验设备是否合法，所以客户在无网络环境中也能使用授权设备。

**Design:** 暂无。

**Acceptance Criteria:**
1. 软件连接设备后执行本地授权或固件校验。
2. 校验通过后允许采集。
3. 校验失败时禁止采集，并显示失败原因。
4. 校验过程不依赖互联网。
5. 校验结果写入软件日志。

### Story B7: 设备参数页

**Description:** 作为科研用户，我希望查看设备参数，所以我能确认本次测试使用的硬件规格。

**Design:** 暂无。

**Acceptance Criteria:**
1. 设备参数页显示 MAC 地址、设备类型、固件版本、连接串口、波特率。
2. 设备参数页预留点阵数量、采样率、压力范围字段。
3. 设备参数可随采集记录保存。
4. 未连接设备时，参数页显示空状态。
5. 参数页内容不可被普通用户随意编辑。

## 5. Epic C: 预压力清零与数据可信度

### Story C1: 执行预压力清零

**Description:** 作为测试工程师，我希望开始测试前执行预压力清零，所以软件能消除初始压力偏移。

**Design:** 暂无。

**Acceptance Criteria:**
1. 设备连接成功后显示预压力清零按钮。
2. 点击清零后，软件以当前压力矩阵作为零点基线。
3. 清零成功后按钮进入已置零状态。
4. 清零失败时显示错误原因。
5. 未连接设备时清零按钮不可用。

### Story C2: 取消预压力清零

**Description:** 作为测试工程师，我希望能取消置零状态，所以我可以回到原始压力数据观察。

**Design:** 暂无。

**Acceptance Criteria:**
1. 已置零状态下，用户可以点击取消置零。
2. 取消置零后，软件恢复显示未扣除基线的数据。
3. 已置零时图标按钮置灰。
4. 取消置零后图标按钮置蓝。
5. 置零和取消置零状态写入采集记录。

### Story C3: 记录清零信息

**Description:** 作为科研用户，我希望清零信息被记录，所以后续回放和导出时能知道数据是否经过清零。

**Design:** 暂无。

**Acceptance Criteria:**
1. 每次采集记录包含是否清零。
2. 每次采集记录包含清零时间。
3. 每次采集记录包含清零基线标识或摘要。
4. 回放时显示该数据是否经过清零。
5. CSV 导出包含清零状态字段。

## 6. Epic D: 实时采集控制

### Story D1: 开始采集

**Description:** 作为测试工程师，我希望点击“采集”开始记录压力数据，所以我可以保存测试过程。

**Design:** 暂无。

**Acceptance Criteria:**
1. 设备连接、授权通过、设备类型已选择后，“采集”按钮可点击。
2. 点击“采集”后软件开始写入采集数据。
3. 采集开始后按钮文字变为“停止”。
4. 采集期间显示采集时长。
5. 采集开始时创建本地采集记录。

### Story D2: 停止采集

**Description:** 作为测试工程师，我希望点击“停止”结束采集，所以本次测试可以保存成历史数据。

**Design:** 暂无。

**Acceptance Criteria:**
1. 采集期间点击“停止”后停止写入数据。
2. 停止后按钮文字变为“采集”。
3. 软件保存本次采集记录。
4. 停止后该记录出现在历史数据列表。
5. 停止失败时提示用户并避免损坏已采集数据。

### Story D3: 采集状态保护

**Description:** 作为测试工程师，我希望采集期间关键设置被保护，所以不会因为误操作导致数据异常。

**Design:** 暂无。

**Acceptance Criteria:**
1. 采集期间不允许切换设备类型。
2. 采集期间不允许断开当前设备，除非用户确认。
3. 采集期间可以继续缩放、翻转和查看热力图。
4. 采集期间可以框选区域查看统计。
5. 采集期间出现设备断开时，软件显示异常状态并保存已采集数据。

## 7. Epic E: 实时 2D 可视化

### Story E1: 2D 热力图显示

**Description:** 作为展示人员，我希望实时看到 2D 压力热力图，所以客户能直观看到受压区域。

**Design:** 暂无。

**Acceptance Criteria:**
1. 设备连接后，软件实时显示压力热力图。
2. 压力越高的区域颜色越明显。
3. 无压力或低压力区域显示为低值颜色。
4. 热力图随传感器数据变化实时更新。
5. 热力图显示不阻塞数据采集。

### Story E2: 坐垫/靠背分区显示

**Description:** 作为汽车座椅测试工程师，我希望区分坐垫和靠背数据，所以我能分别分析不同区域的压力。

**Design:** 暂无。

**Acceptance Criteria:**
1. 汽车坐垫设备显示为坐垫视图。
2. 汽车靠背设备显示为靠背视图。
3. 分区名称在界面中清晰显示。
4. 切换设备类型后分区显示同步变化。
5. 采集记录中保存对应设备类型和分区信息。

### Story E3: 2D 数字矩阵开关

**Description:** 作为科研用户，我希望能查看 2D 数字矩阵，所以我可以检查具体传感点数值。

**Design:** 暂无。

**Acceptance Criteria:**
1. 用户可以打开或关闭 2D 数字矩阵。
2. 打开后每个传感点显示当前数值。
3. 关闭后回到普通热力图视图。
4. 高密度矩阵下数字不应导致界面不可操作。
5. 数字矩阵只影响显示，不影响采集数据。

### Story E4: 画布左右/上下翻转

**Description:** 作为测试工程师，我希望能左右或上下翻转画布，所以软件显示方向能匹配实际设备摆放。

**Design:** 暂无。

**Acceptance Criteria:**
1. 用户可以点击左右翻转按钮。
2. 用户可以点击上下翻转按钮。
3. 翻转只影响显示方向，不改变原始数据存储。
4. 翻转状态在当前测试中保持。
5. 回放时能使用采集时的显示方向或手动重新设置。

### Story E5: 鼠标滚轮缩放

**Description:** 作为测试工程师，我希望用鼠标滚轮放大或缩小画布，所以我可以查看整体或局部细节。

**Design:** 暂无。

**Acceptance Criteria:**
1. 鼠标滚轮向上放大画布。
2. 鼠标滚轮向下缩小画布。
3. 缩放范围限制在 10% 到 1000%。
4. 达到边界后继续滚动不再超出限制。
5. 缩放不影响采集和导出数据。

### Story E6: 视图重置

**Description:** 作为测试工程师，我希望一键重置视图，所以我可以快速回到默认显示状态。

**Design:** 暂无。

**Acceptance Criteria:**
1. 用户点击视图重置后，缩放比例恢复默认。
2. 用户点击视图重置后，画布位置恢复默认。
3. 是否恢复翻转状态需按产品配置执行，并保持一致。
4. 重置视图不影响采集状态。
5. 重置视图不改变原始数据。

## 8. Epic F: 实时统计与框选

### Story F1: 实时统计指标

**Description:** 作为测试工程师，我希望实时查看核心压力指标，所以我可以用数字理解当前压力状态。

**Design:** 暂无。

**Acceptance Criteria:**
1. 软件实时显示平均压强、最大压强、最小压强。
2. 软件实时显示压力总和、受压面积、传感点数。
3. 指标随压力数据变化更新。
4. 指标单位在界面上明确显示。
5. 最小压强是否包含 0 值需按产品口径固定。
6. 受压面积的计算阈值需可配置或有默认说明。

### Story F2: 单矩形框选

**Description:** 作为科研用户，我希望在热力图上框选一个矩形区域，所以我可以只分析局部压力。

**Design:** 暂无。

**Acceptance Criteria:**
1. 用户可以启用框选工具。
2. 用户可以拖拽创建一个矩形框选区域。
3. 框选区域不能超出传感器边界。
4. 框选区域在热力图上高亮显示。
5. 用户可以取消框选，恢复全局统计。

### Story F3: 精确坐标输入框选

**Description:** 作为科研用户，我希望通过 X/Y/长/宽输入框选区域，所以我可以复现相同区域的统计。

**Design:** 暂无。

**Acceptance Criteria:**
1. 用户可以输入 X、Y、长、宽。
2. 输入值必须为有效数字。
3. 输入区域超出边界时，软件提示错误并不应用。
4. 应用成功后热力图显示对应框选区域。
5. 坐标口径与传感器矩阵坐标保持一致。

### Story F4: 框选区域统计

**Description:** 作为测试工程师，我希望框选后统计只显示框选区域数据，所以我可以分析局部压力变化。

**Design:** 暂无。

**Acceptance Criteria:**
1. 框选后统计指标基于框选区域计算。
2. 未框选时统计指标基于全部传感点计算。
3. 框选状态下仍保存全量原始数据。
4. 框选统计显示区域名称或坐标。
5. 取消框选后统计恢复为全局数据。

### Story I1: 本地采集缓存

## 12. Epic J: 性能与基础可靠性

### Story J1: 实时显示低延迟

**Description:** 作为展示人员，我希望压力图变化延迟低，所以用户坐下或移动时画面能及时响应。

**Design:** 暂无。

### Story J2: 60fps 渲染目标验证

**Description:** 作为产品和研发团队，我希望验证 60fps 渲染目标，所以我们知道 V2.0 的性能边界。

3. 测试包含普通热力图、数字矩阵开启、缩放、框选四种状态。

## 13. 建议研发拆分顺序

1. 设备连接最小闭环：串口检测、波特率尝试、MAC 读取、一键连接。
2. 设备参数与授权：设备类型、本地授权/固件校验、设备参数页。
3. 实时数据通路：接收压力矩阵、开始/停止采集、本地缓存。
4. 2D 可视化：热力图、坐垫/靠背分区、翻转、缩放、重置。
5. 数据可信度：预压力清零、取消清零、清零记录。
6. 统计与框选：全局统计、单矩形框选、区域统计。
7. 历史数据：列表、重命名、搜索、删除、回放入口。
8. 回放控件：播放、暂停、进度条、倍速、退出。
9. CSV 导出：路径选择、格式版本、成功提示。
10. 异常恢复和性能验收：缓存恢复、清空缓存、FPS 测试。

## 14. 需要产品确认的问题

1. V2.0 是否必须支持汽车坐垫和汽车靠背同时显示，还是先单设备显示？
    - 产品确认：设备同时显示。
2. 设备类型是由固件自动识别，还是由用户手动选择？
    - 产品确认：设备类型自动识别。
3. 采样率字段是否由设备返回，还是由软件配置记录？
    - 产品确认：采样率字段由设备返回。
4. 预压力清零是否要求无负载状态下执行？
    - 产品确认：要求用户点击执行。
5. 受压面积的压力阈值是多少？

6. 最小压强是否排除 0 值？
    - 产品确认：排除 0 值。
7. CSV 中压力矩阵采用宽表、长表，还是逐帧块状结构？
    - 产品确认：一维数组表。
8. 历史数据默认保存路径在哪里？
    - 产品确认：默认保存在软件安装目录下的历史数据文件夹中，可以在设置中修改保存路径。
9. 异常恢复保留多久的缓存？
    - 产品确认：默认保留10秒缓存。
10. V2.0 验收时 60fps 是硬性指标还是目标指标？
    - 产品确认：目标指标。可能硬件达不到60HZ，但是软件会努力实现。
