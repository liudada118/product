# JQ Tools V2.0 功能需求整理

## 1. 产品目标

JQ Tools V2.0 的目标是把压力感知系统从“实时显示工具”升级为“可连接设备、可实时采集、可视化分析、可回放、可导出、可本地管理数据”的完整压力分析软件。

核心场景包括：

1. 汽车座椅研发测试。
2. 汽车座椅/靠背压力展示。
3. 工学椅、床垫、手部等多设备类型压力采集。
4. 展厅或门店实时压力展示。
5. 科研客户的数据采集、分析、回放和导出。

## 2. 需求主题分组

### A. 设备连接与授权

用户问题：用户需要插入设备后快速连接，并确认设备合法、类型正确、参数可用。

需求项：

1. 打开软件后支持“一键连接”。
2. 串口自动检测，识别 CH340 芯片。
3. 波特率自动探测，支持 921600、1000000、3000000。
4. 自动读取设备 MAC 地址。
5. 插入设备后自动读取 MAC。
6. 每次录入时只能在一个串口插入水晶头。
7. 用户配置软件从“传感器配对”改为“固件校验”。
8. 支持无联网使用。
9. 出货前录入数据库管理系统。
10. 设备授权加密。
11. 设备类型切换。
12. 支持多种传感器硬件。

设备类型：

1. 手-坐垫。
2. 床-床垫。
3. 汽车坐垫。
4. 汽车靠背。
5. 工学椅-坐垫。
6. 工学椅-靠背。

优先级建议：P0。

理由：没有稳定连接、MAC 识别、设备类型和授权，后面的采集、显示、回放和导出都不可用。

风险点：

1. 自动波特率探测可能增加连接耗时。
2. “固件校验”和“数据库录入”的离线授权逻辑需要定义清楚。
3. 多设备类型会影响数据矩阵尺寸、坐标方向、可视化映射和报告模板。

需要确认：

1. 本地软件是否需要内置授权白名单，还是读取本地授权文件？
2. 出货前数据库管理系统由谁维护，JQ Tools 是否需要访问？
3. 一个设备是否只绑定一个设备类型，还是可在软件内切换？

### B. 采集控制与数据状态

用户问题：用户需要明确地开始、停止、保存压力数据，并知道当前是否正在采集。

需求项：

1. 点击“采集”按钮后开始数据采集。
2. 开始后按钮文字变为“停止”。
3. 点击“停止”按钮后停止数据采集。
4. 停止后按钮文字变为“采集”。
5. 数据采集结果本地存储。
6. 异常关机后重启自动恢复。
7. 增加本地缓存。
8. 支持清空本地缓存。
9. 数据防丢失。

优先级建议：P0。

理由：采集状态是系统主流程。按钮状态、缓存和异常恢复直接影响用户信任。

风险点：

1. 异常关机恢复需要设计缓存写入频率，否则会影响性能。
2. “采集数据”和“显示数据”是否总是同步，需要明确。
3. 框选状态会影响采集范围，需定义清楚。

需要确认：

1. 采集时是否保存原始全量传感点，还是只保存框选区域？
2. 采集文件默认存储路径在哪里？
3. 是否需要采集前填写项目名称、设备类型和备注？

### C. 预压力清零

用户问题：用户需要在测试开始前消除预压力或零点偏移，保证压力数据可信。

需求项：

1. 预压力清零按钮。
2. 置零后图标按钮置灰。
3. 取消置零后图标按钮置蓝。

优先级建议：P0。

理由：清零直接影响数据可信度，是压力测量系统的基础能力。

风险点：

1. “置零”和“取消置零”的数据逻辑不能只做 UI 状态。
2. 需要定义置零是全局置零、区域置零，还是按设备类型置零。
3. 取消置零后是否恢复原始基线，需要明确。

需要确认：

1. 置零时是否要求传感器无负载？
2. 置零值是否写入采集文件？
3. 回放时是否显示当时的置零状态？

### D. 2D 可视化

用户问题：用户需要实时、直观地看到压力分布，并能用热力图和数字矩阵理解数据。

需求项：

1. 2D 可视化。
2. 热力图显示。
3. 靠背/坐垫分区显示。
4. 2D 数字矩阵。
5. 颜色映射算法。
6. 画布左右翻转。
7. 画布上下翻转。
8. 通过鼠标滚轮放大和缩小，范围 10% 到 1000%。
9. 视图重置。
10. 显示模式切换。
11. 2D 数字视图。

优先级建议：P0。

理由：2D 热力图是压力感知系统最核心的价值表达，也是展厅、研发和科研都需要的基础能力。

风险点：

1. 不同设备类型的方向、坐标和分区不同，翻转逻辑要和数据坐标分开。
2. 10% 到 1000% 缩放范围很大，性能和交互需要验证。
3. 数字矩阵可能在高密度传感点下难以阅读。

需要确认：

1. 热力图颜色是否需要和竞品或客户标准一致？
2. 压力单位是什么？
3. 坐垫和靠背是否同时显示，还是切换显示？

### E. 3D 可视化

用户问题：用户希望用更直观、更有科技感的 3D 方式展示压力云图。

需求项：

1. 3D 可视化。
2. 3D 点图模型，使用 Three.js。
3. 3D 数字显示。
4. 3D 模型视图。
5. 压力云图高度调节。
6. 云图响应速度调节。
7. 3D 模型上传。
8. 视角切换。
9. 视图重置。

优先级建议：P1。

理由：3D 对展示和高端感有帮助，但在数据可信、采集、导出、回放之前不是第一优先级。

风险点：

1. Three.js 渲染可能影响 60fps 目标。
2. 3D 模型上传涉及格式、比例、坐标对齐和安全校验。
3. 3D 数字显示容易遮挡或影响阅读。

需要确认：

1. V2.0 是否必须支持 3D 模型上传，还是先内置标准坐垫/靠背模型？
2. 3D 是展示模式核心，还是科研分析辅助？
3. 支持哪些模型格式，比如 glb、gltf、obj？

### F. 可视化设置与调节

用户问题：不同传感器、不同座椅和不同场景下，用户需要调整视觉效果，减少噪声并增强可读性。

需求项：

1. 可视化设置。
2. 通用设置。
3. 只 2D 视图下可用的设置。
4. 只 3D 视图下可用的设置。
5. 图像润滑度调节。
6. 视图颜色层次调节。
7. 噪点消除。
8. 压力云图高度调节。
9. 云图响应速度调节。
10. 恢复默认可视化调节值。
11. 放大镜。
12. 局部放大查看，且放大的数据可实时变化。

优先级建议：P1。

理由：这些能力能提升专业感和可用性，但需要在核心采集和基础显示稳定后再做完整。

风险点：

1. “润滑度”建议改名为“平滑度”。
2. 噪点消除可能改变原始数据，必须区分“显示滤波”和“原始数据”。
3. 局部放大实时变化对性能要求高。

需要确认：

1. 调节项是否只影响显示，不影响原始采集数据？
2. 是否需要保存每个设备类型的默认可视化配置？
3. 放大镜是否只用于 2D，还是 3D 也需要？

### G. 框选与区域分析

用户问题：用户需要只分析某个区域的压力，例如坐垫局部、靠背局部或受压高峰区。

需求项：

1. 框选工具。
2. 矩形区域框选。
3. 精确坐标输入，支持 X/Y/长/宽。
4. 框选后显示框选数据。
5. 框选后采集框选数据。
6. 未框选时显示并采集所有传感点数据。
7. 框选数据统计。
8. 区域强调显示。
9. 边界校验。

优先级建议：P0/P1。

理由：框选分析是科研和研发测试的重要能力。V2.0 至少应支持单个矩形框选和区域统计；多区域或复杂交互可延后。

风险点：

1. “框选后采集框选数据”可能导致原始全量数据丢失，不建议作为默认行为。
2. 坐垫/靠背翻转后，框选坐标和原始矩阵坐标需要一致。
3. 多设备类型下边界校验不同。

建议方案：

1. 默认始终保存全量原始数据。
2. 框选只影响显示统计和导出筛选。
3. 如客户明确选择“只导出框选区域”，再生成区域数据文件。

### H. 量尺工具

用户问题：用户需要在压力图上测量区域距离、尺寸或辅助定位。

需求项：

1. 量尺工具。
2. 多量尺管理。
3. 选中量尺高亮，颜色为黄色。
4. 支持单个删除。
5. 支持批量删除。

优先级建议：P2。

理由：量尺对专业分析有价值，但不是采集和可视化主流程的必要条件。

风险点：

1. 需要明确量尺单位，是传感点、毫米，还是像素。
2. 如果要显示实际长度，需要传感器物理尺寸和点距。

### I. 实时统计指标

用户问题：用户需要把热力图转成可理解的数字指标。

需求项：

1. 平均压强。
2. 最大压强。
3. 最小压强。
4. 压力总和。
5. 受压面积。
6. 传感点数。

优先级建议：P0。

理由：这些指标是报告、趋势图、区域统计和对比分析的基础。

风险点：

1. 压强、压力、压力总和的单位和换算需要统一。
2. 受压面积需要定义阈值，例如大于多少压力算受压。
3. 最小压强是否包含 0 值，需要明确。

### J. 趋势图表与分析

用户问题：用户需要观察压力随时间变化，而不是只看某一帧。

需求项：

1. 趋势图表可拖拽。
2. 趋势图表可调整大小。
3. 压力总和曲线。
4. 面积变化曲线。
5. 压力正态分布图。
6. 压力重心点轨迹。

优先级建议：P1。

理由：趋势图对研发和科研价值高，但依赖采集、缓存、时间轴和指标计算。

风险点：

1. 图表拖拽和调整大小会增加界面复杂度。
2. 正态分布图需要明确统计对象，是当前帧、时间段，还是框选区域。
3. 压力重心轨迹需要处理噪声，否则会抖动。

### K. 历史数据与回放

用户问题：用户需要查看过去的采集数据，并像视频一样回放压力变化。

需求项：

1. 历史数据。
2. 采集数据存储。
3. 历史列表浏览。
4. 数据搜索。
5. 重命名与备注。
6. 数据删除。
7. 进入回放界面后，下方弹出回放控件。
8. 历史回放播放。
9. 开始。
10. 暂停。
11. 播放/暂停控制。
12. 滑块。
13. 帧进度控制。
14. 倍速调节。
15. 退出。

优先级建议：P0/P1。

理由：如果产品要服务研发和科研，回放和历史数据是必须能力。V2.0 应至少支持历史列表、播放、暂停、进度条和倍速。

风险点：

1. 数据文件体积可能很大。
2. 回放时是否复现原始采集频率，需要定义。
3. 回放界面和实时界面是否共用可视化组件，需要提前设计。

### L. 数据导入、导出与对比

用户问题：用户需要把数据拿出去做报告，也需要把历史或外部数据导入做对比。

需求项：

1. 导出 CSV 文件。
2. 点击“下载”按钮后弹出路径选择对话框。
3. CSV 格式导出。
4. 下载进度显示。
5. 下载成功后弹窗。
6. 单击弹窗可打开文档。
7. 文件路径可点击打开。
8. CSV 文件上传。
9. 外部数据加载。
10. 双组数据选择。
11. 对比热力图显示。
12. 分压对比分析。

优先级建议：导出 P0，导入/对比 P1/P2。

理由：CSV 导出是科研和招标刚需。数据导入和双组对比有价值，但可以在基础采集闭环稳定后再做。

风险点：

1. CSV 文件格式必须定义版本，否则后续导入会混乱。
2. 外部数据加载需要校验格式、设备类型和矩阵尺寸。
3. 双组对比需要处理不同采样率、不同尺寸和不同方向。

### M. 安装、版本与兼容性

用户问题：用户需要像普通软件一样安装、升级和使用，不依赖复杂环境。

需求项：

1. 软件以安装包形式交付。
2. Windows 系统兼容。
3. macOS 系统兼容。
4. 产品版本为 V2.0。
5. 后续每升级一版，版本号自动加一。
6. 语言切换。
7. 中文。

优先级建议：Windows 安装包 P0，macOS P2，中文 P0，多语言 P2。

理由：当前客户和硬件串口生态大概率优先 Windows。macOS 可作为后续兼容目标。

风险点：

1. CH340、串口和驱动在 macOS 上适配成本更高。
2. 自动版本号需要配合构建流水线。

### N. 非功能需求

用户问题：软件必须稳定、低延迟、安全、可靠，否则客户不会信任数据。

需求项：

1. 实时数据低延迟。
2. 60fps 渲染目标。
3. 数据防丢失。
4. 设备授权加密。
5. 兼容 Windows 系统。
6. 兼容 macOS 系统。
7. 支持多种传感器硬件。

优先级建议：P0/P1。

风险点：

1. 60fps 对高密度传感点、3D 渲染、趋势图和局部放大同时开启时有挑战。
2. 数据防丢失会增加本地缓存和文件写入压力。
3. 多传感器兼容会明显增加测试矩阵。

## 3. 重复、冲突和需澄清项

### 3.1 重复项

1. 回放控制中“开始/暂停”和“播放/暂停控制”重复，可合并。
2. 数据导出中“导出 CSV 文件”和“CSV 格式导出”重复，可合并。
3. “下载成功后弹窗，单击可打开文档”和“文件路径可点击打开”可合并为导出成功提示。
4. “靠背/坐垫分区显示”和后面的“坐垫/靠背”重复，需要归入设备分区视图。
5. “3D 模型视图”和“3D 可视化”重复，需要合并。
6. “数据采集”和“采集数据存储”应合并为采集闭环。

### 3.2 潜在冲突

1. 框选后“采集框选数据”与数据防丢失冲突。建议始终保存全量原始数据，框选只影响显示、统计和导出。
2. “无联网使用”与“出货前录入数据库管理系统”需要明确授权数据如何同步到本地。
3. “60fps 渲染目标”与 3D、趋势图、局部放大、数字矩阵同时开启存在性能冲突。
4. macOS 兼容与 CH340 串口、驱动、安装包交付存在适配冲突。
5. “后续每升级一版版本号自动加一”属于构建和发布流程，不是用户功能，需要归入工程需求。

### 3.3 术语建议

1. “预压力清零按扭”改为“预压力清零按钮”。
2. “图像润滑度”改为“图像平滑度”。
3. “采框选数据”改为“采集框选区域数据”。
4. “插如设备”改为“插入设备”。
5. “分压对比分析”需定义为“分区压力对比”还是“压力差值对比”。

## 5. Top 3 优先功能

### Top 1: 设备连接、授权与设备类型管理

Rationale:

这是所有功能的入口。用户必须先连接设备、识别 MAC、确认设备类型，才能采集和显示数据。

Alternative Solutions:

1. V2.0 先做本地授权文件，不做实时数据库访问。
2. 先支持 Windows + CH340，macOS 后置。
3. 先支持手动选择设备类型，后续再做自动识别。

High-Risk Assumptions:

1. 自动波特率探测能稳定识别设备。
2. MAC 地址读取能覆盖所有硬件版本。
3. 离线固件校验能满足防伪和授权要求。

Fast Test:

用 3 到 5 台真实设备做连接测试，记录识别耗时、失败率、波特率识别准确率和 MAC 读取成功率。

### Top 2: 实时 2D 压力显示、采集和清零

Rationale:

这是用户感知产品价值的核心闭环。用户插入设备后，需要看到实时压力图，能清零，能开始和停止采集。

Alternative Solutions:

1. V2.0 优先做 2D，3D 作为增强功能。
2. 默认保存全量数据，框选只影响统计。
3. 数字矩阵可作为开关，避免影响主视图性能。

High-Risk Assumptions:

1. 高密度数据下 2D 热力图能接近 60fps。
2. 清零逻辑能稳定处理预压力。
3. 颜色映射能让客户直观理解压力差异。

Fast Test:

做一个 2D 热力图性能样机，使用真实或模拟压力矩阵，测试不同点数、缩放、翻转、数字矩阵开启时的帧率。

### Top 3: 历史数据、回放和 CSV 导出

Rationale:

研发和科研客户不仅要看实时图，还要保存、回放、导出并用于报告。没有这部分，产品很难进入科研和招标场景。

Alternative Solutions:

1. V2.0 先支持 CSV 导出，不先做复杂报告。
2. 先支持单组历史回放，数据对比后置。
3. 先固定 CSV 格式，后续加格式版本和导入。

High-Risk Assumptions:

1. 本地存储能承载长时间采集数据。
2. 回放和实时显示可复用同一套渲染逻辑。
3. CSV 格式能满足客户二次分析需求。

Fast Test:

用 10 分钟、30 分钟、60 分钟三种采集时长测试文件大小、写入稳定性、回放流畅度和导出耗时。
