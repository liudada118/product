## 5. 主流程业务逻辑

| 步骤 | 用户动作 | 前置条件 | 系统处理 | 成功结果 | 失败/异常处理 |
|---|---|---|---|---|---|
| 1 | 打开软件 | 软件已安装 | 检查本地设备配置 | 有配置进入主界面 | 无配置进入 MAC 配置页 |
| 2 | 录入设备配置 | 首次使用或设备未配置 | 批量保存 MAC 和设备类型 | 配置写入本地缓存 | 格式错误提示用户修改 |
| 3 | 点击一键连接 | 设备已插入 | 扫描串口、识别 CH340、探测波特率 | 建立串口连接 | 未找到设备或波特率失败时提示 |
| 4 | 读取设备信息 | 串口连接成功 | 读取 MAC、识别设备类型、读取采样率 | 显示设备在线和参数 | MAC 读取失败时禁止采集 |
| 5 | 授权/类型校验 | 已读取设备信息 | 校验设备是否允许采集、类型是否匹配当前系统 | 进入实时展示 | 校验失败时禁止采集 |
| 6 | 接收设备数据 | 设备在线且持续推送 | 接收串口原始帧、校验帧完整性、解析点阵 | 生成原始压力矩阵 | 断帧、坏帧、长度不匹配时丢弃并记录 |
| 7 | 处理压力矩阵 | 原始帧解析成功 | 按设备类型转换矩阵、计算采样率、生成推送数据 | 生成可供展示和统计的数据对象 | 矩阵尺寸不匹配时提示设备或配置异常 |
| 8 | 查看实时压力图 | 数据处理成功 | WebSocket 推送压力矩阵，前端渲染 | 2D/3D 视图实时变化 | 数据中断时提示设备异常 |
| 9 | 使用分析工具 | 有实时数据 | 置零、翻转、框选、量尺、可视化调节 | 视图和统计变化 | 工具不可用时给出提示 |
| 10 | 点击采集 | 设备在线且校验通过 | 创建采集任务，持续写入本地数据库 | 按钮变为停止，数据持续保存 | 设备不合法或类型不匹配时拒绝 |
| 11 | 点击停止 | 正在采集 | 停止写入，生成历史记录 | 历史列表出现记录 | 保存失败时保留缓存并提示 |
| 12 | 回放历史 | 存在历史记录 | 加载历史帧，按进度播放 | 热力图、统计、图表同步变化 | 历史损坏时提示并跳过损坏帧 |
| 13 | 导出 CSV | 选择历史记录 | 选择路径，生成 CSV | 导出成功，可打开文件 | 导出失败提示原因 |
| 14 | 导入 CSV | 选择 CSV 文件 | 校验文件结构和矩阵长度 | 导入成功进入本地数据列表 | 不符合结构提示“数据有误” |

## 6. 全局状态机

| 状态 | 进入条件 | 可执行操作 | 禁止操作 | 退出条件 |
|---|---|---|---|---|
| 未配置 | 本地无设备配置 | 进入 MAC 配置、录入设备 | 采集、回放实时数据 | 配置保存成功 |
| 未连接 | 有配置但未连接设备 | 一键连接、设备管理、历史查看 | 实时采集 | 连接成功 |
| 连接中 | 用户点击一键连接 | 等待、取消连接（如支持） | 重复点击连接、采集 | 成功或失败 |
| 已连接 | 设备在线且校验通过 | 查看视图、使用工具、开始采集 | 无 | 断开设备或开始采集 |
| 采集中 | 用户点击采集且校验通过 | 停止采集、查看视图、框选统计 | 切换系统类型、删除当前采集 | 停止采集或异常中断 |
| 回放中 | 用户选择历史记录 | 播放、暂停、拖动、倍速、框选回放 | 实时采集 | 退出回放 |
| 导出中 | 用户点击导出 | 等待导出 | 重复导出、删除当前记录 | 导出成功或失败 |
| 异常恢复 | 软件启动发现未完成采集 | 恢复、丢弃、查看缓存 | 直接覆盖缓存 | 用户选择处理方式 |

### 7.4 设备配置业务逻辑

| 规则 | 说明 | 优先级 |
|---|---|---|
| 配置格式必须可解析 | 当前 `MAC地址:类型` 容易与 MAC 冒号冲突，建议改为 `MAC=类型` | P1 |
| 每次录入只能插一个设备 | 避免多个串口同时插入导致 MAC 识别混乱 | P0 |
| MAC 必须唯一 | 同一个 MAC 不允许重复配置不同类型 | P0 |
| 类型必须在支持列表内 | 如 carY-sit、carY-back、endi-sit、endi-back 等 | P0 |
| 配置保存后写入本地缓存 | 支持离线使用 | P0 |
| 清空缓存不应删除历史数据 | 设备缓存和采集历史是两类数据 | P0 |

## 8. 模块二：设备数据接收与处理

### 8.1 模块目标

| 项目 | 内容 |
|---|---|
| 模块目标 | 把设备串口原始数据转换成后续展示、统计、采集和导出可使用的标准压力数据 |
| 模块位置 | 位于“设备连接流程”和“展示视图”之间 |
| 业务价值 | 保证后续所有压力图、统计指标、采集数据和 CSV 数据可信 |
| 成功结果 | 生成结构稳定、尺寸正确、可追溯的原始压力矩阵和推送数据 |

### 8.2 为什么需要单独拆这个模块

| 问题 | 说明 |
|---|---|
| 连接成功不等于数据可用 | 串口连接只是建立通道，还需要确认设备帧能被正确解析 |
| 展示视图依赖处理后数据 | 2D/3D 视图不能直接消费串口原始字节，需要标准矩阵 |
| 统计指标依赖数据口径 | 平均压强、最大压强、面积等都依赖矩阵尺寸、单位和有效点规则 |
| 采集和导出依赖原始数据 | 如果处理层没有定义 rawData，后续很容易把显示数据误当原始数据保存 |
| 异常定位需要分层 | 设备断连、坏帧、矩阵错位、显示异常需要区分是连接问题、解析问题还是渲染问题 |

### 8.3 当前已实现

| 功能 | 当前状态 | 备注 |
|---|---|---|
| 串口数据接收 | 已实现 | 由串口监听和数据帧处理负责 |
| 数据帧分隔 | 已实现 | 根据协议分隔符识别帧 |
| 波特率探测时帧校验 | 已实现 | 用于判断候选波特率是否可用 |
| 设备类型对应解析函数 | 已实现 | hand、bed、endi、carY 等存在不同转换函数 |
| 一维点阵转矩阵 | 已实现 | 按系统类型转换为对应尺寸 |
| 多系统矩阵尺寸配置 | 已实现 | 如 32x32、46x46、50x64、64x64 |
| WebSocket 实时推送 | 已实现 | 后端推送前端消费 |
| 采样率/HZ 处理 | 已实现或部分实现 | 口径需补齐 |
| 历史回放数据结构 | 已实现 | 与实时数据结构需保持一致 |

### 8.4 输入输出

| 类型 | 内容 | 来源/去向 | 说明 |
|---|---|---|---|
| 输入 | 串口原始字节流 | 设备串口 | 设备持续发送的原始数据 |
| 输入 | 设备类型 | 连接模块 | 决定使用哪套解析规则 |
| 输入 | 系统类型 | 前端/后端状态 | 决定矩阵尺寸和展示结构 |
| 输入 | 当前波特率 | 连接模块 | 影响数据帧接收速度 |
| 输出 | 原始帧 rawFrame | 数据处理模块内部 | 用于调试和追溯 |
| 输出 | 原始矩阵 rawData | 展示、统计、采集模块 | 设备原始方向和原始压力数据 |
| 输出 | 推送数据 payload | WebSocket | 前端实时展示使用 |
| 输出 | 采样率 HZ | 展示、历史、CSV | 用于显示、回放和导出 |
| 输出 | 异常状态 | 前端提示、日志 | 如坏帧、断帧、矩阵长度不匹配 |

### 8.5 主处理流程

| 步骤 | 处理动作 | 输入 | 输出 | 异常处理 |
|---|---|---|---|---|
| 1 | 接收串口原始字节 | 设备字节流 | 缓冲区数据 | 无数据时保持等待 |
| 2 | 查找帧分隔符 | 缓冲区数据 | 候选数据帧 | 找不到分隔符则继续累积或丢弃异常片段 |
| 3 | 校验帧完整性 | 候选数据帧 | 完整数据帧 | 长度不匹配则丢弃并记录 |
| 4 | 识别设备类型 | MAC 映射/设备类型 | 解析规则 | 类型未知则不进入展示和采集 |
| 5 | 解析压力点数组 | 完整数据帧 | 一维压力数组 | 解析失败则丢弃该帧 |
| 6 | 转换矩阵 | 一维压力数组 + 系统类型 | rawData 矩阵 | 长度和矩阵尺寸不匹配则报错 |
| 7 | 计算采样率 | 帧时间戳 | HZ | 异常波动时可平滑或标记 |
| 8 | 生成推送数据 | rawData + metadata | WebSocket payload | 推送失败时记录连接状态 |
| 9 | 分发下游模块 | payload | 展示、统计、采集 | 下游失败不应破坏原始数据 |

| `bigHand` | 4096 点阵转换规则 | 64 x 64 | 高密度矩阵 | 是 |

### 8.7 数据层业务逻辑

| 数据层 | 生成位置 | 业务含义 | 是否可被修改 | 下游用途 |
|---|---|---|---|---|
| rawFrame | 后端串口接收后 | 设备原始帧或原始字节片段 | 不应修改 | 调试、异常定位 |
| rawPointArray | 后端解析后 | 一维压力点数组 | 不应修改 | 矩阵转换 |
| rawData | 后端或前端矩阵转换后 | 原始压力矩阵 | 不应被展示工具污染 | 展示、统计、采集、导出 |
| displayData | 前端处理后 | 用于界面展示的矩阵 | 可受翻转、颜色、润滑影响 | 2D/3D 展示 |
| calculatedData | 统计模块处理后 | 用于指标计算的矩阵 | 受置零、框选等规则影响 | 平均值、最大值、面积等 |
| storedData | 采集模块处理后 | 入库保存的数据 | 口径必须明确 | 历史回放 |
| exportedData | 导出模块处理后 | CSV 输出数据 | 口径必须明确 | 科研分析、外部导入 |

### 8.8 异常处理业务逻辑

| 异常类型 | 判断条件 | 系统处理 | 用户提示 | 是否阻塞采集 |
|---|---|---|---|---|
| 空帧 | 收到空数据或无有效压力点 | 丢弃该帧 | 不一定提示，可记录日志 | 否 |
| 坏帧 | 帧头/帧尾/分隔符异常 | 丢弃该帧并记录 | 高频出现时提示设备数据异常 | 视频率而定 |
| 帧长度不匹配 | 解析长度不符合设备类型 | 丢弃该帧 | 提示数据长度异常 | 是 |
| 设备类型未知 | MAC 未命中且设备未返回类型 | 不解析为有效矩阵 | 提示设备类型未知 | 是 |
| 矩阵尺寸不匹配 | 点数无法映射到配置矩阵 | 拒绝进入展示/采集 | 提示设备配置异常 | 是 |
| 采样率异常 | HZ 长时间为 0 或明显跳变 | 记录并提示 | 视情况而定 |
| WebSocket 推送失败 | 前端连接断开 | 保留后端最新数据，等待重连 | 提示实时连接异常 | 不一定 |
| 连续坏帧 | 一段时间内坏帧超过阈值 | 标记设备数据异常 | 提示检查设备或连接 | 是 |

## 14. 全局数据规则

### 14.1 数据分层

| 数据层 | 定义 | 是否必须保留 | 影响因素 |
|---|---|---|---|
| 原始数据 | 设备原始方向、原始压力矩阵 | 必须保留 | 不应受可视化调节影响 |
| 置零后数据 | 扣除置零基线后的矩阵 | 根据置零状态生成 | 受置零影响 |
| 显示数据 | 用于界面展示的矩阵 | 必须可重建 | 受翻转、颜色、润滑影响 |
| 统计数据 | 用于计算指标的矩阵 | 口径必须明确 | 受置零、框选、可能受滤波影响 |
| 入库数据 | 采集保存到数据库的数据 | 必须可追溯 | 方向和置零口径需明确 |
| 导出数据 | CSV 导出的数据 | 必须标明口径 | 需和导入兼容 |

| 置零 | 不应影响 | 影响 | 影响 | 待确认 | 待确认 | P0 待确认 |
| 颜色调节 | 不影响 | 影响 | 不影响 | 不影响 | 不影响 | 清晰 |
| 高斯润滑 | 不影响 | 影响 | 待确认 | 不应影响 | 不应影响 | P0 待确认 |
| 噪点消除 | 不影响 | 影响 | 待确认 | 待确认 | 待确认 | P0 待确认 |
| 框选 | 不影响 | 影响局部显示 | 影响统计范围 | 不应丢全量 | 可导出派生数据 | P0 待确认 |

## 16. 发布准入

| 检查项 | 准入标准 | 是否阻塞 |
|---|---|---|
| 设备连接 | 真实设备一键连接成功率满足现场使用 | 是 |
| 授权校验 | 非法设备不能采集 | 是 |
| 类型校验 | 设备类型不匹配不能采集 | 是 |
| 实时展示 | 2D/3D 视图可实时反映压力变化 | 是 |
| 数据采集 | 连续采集 30 分钟无明显卡死或数据中断 | 是 |
| 历史数据 | 停止采集后历史记录不丢失 | 是 |
| 回放 | 历史数据可完整回放，拖动进度不崩溃 | 是 |
| CSV | 导出 CSV 可打开、可导入、字段可解释 | 是 |
| 置零 | 置零影响数据基线，不只是按钮状态变化 | 是 |
| 框选 | 框选不导致全量原始数据丢失 | 是 |
| 异常恢复 | 采集中强退后可恢复或保留数据 | 是 |
| 安装包 | Windows/macOS 打包后可启动，缓存路径正确 | 是 |
