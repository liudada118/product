# 触觉手套 / 力反馈手套 / 机器人遥操作手套竞品调研

## 1. 调研范围

本次调研对象是用于 XR 训练、机器人遥操作、机器人示教、动作捕捉和触觉反馈的手套类产品。

重点区分两类产品：

1. 输入型手套：核心价值是采集手部动作、手指姿态、手势和关节数据，用于机器人控制、动作捕捉、仿真训练和 AI 数据采集。
2. 反馈型手套：核心价值是给操作者触觉、振动、压力或力反馈，让操作者在虚拟环境或远程机器人操作中“感觉到”物体。

更细可以拆成三层：

| 类型 | 核心能力 | 典型竞品 |
|---|---|---|
| 输入型手套 | 手指追踪、手势识别、动作捕捉 | MANUS Quantum、Noitom Hi5 |
| 振动反馈手套 | LRA/ERM 振动、触觉提示、低成本反馈 | bHaptics TactGlove、Noitom Hi5 |
| 力反馈/真实触觉手套 | 阻力反馈、皮肤位移、压力反馈、遥操作 | HaptX、SenseGlove、MANUS Pro Haptic |

## 2. 市场概览

### 2.1 主要客户

1. 机器人公司：人形机器人、灵巧手、遥操作、模仿学习、具身智能数据采集。
2. 科研院所和高校：机器人控制、触觉交互、VR/AR、人机交互研究。
3. 工业培训客户：装配、维修、危险作业、航空航天、制造业训练。
4. XR 内容和仿真公司：VR 训练、游戏、医疗模拟、多人协作。
5. 动作捕捉和动画团队：手部高精度动捕、影视动画、虚拟人。

### 2.2 关键购买标准

1. 手指追踪精度。
2. 反馈类型：无反馈、振动反馈、力反馈、触觉压力反馈。
3. 延迟。
4. 自由度数量。
5. 是否支持 Unity、Unreal、ROS、Python、C++ SDK。
6. 是否可用于机器人遥操作。
7. 是否无线。
8. 是否易穿戴、易校准、可多人共用。
9. 价格和交付周期。
10. 是否能适配客户自研机器人或仿真系统。

## 4. 竞品一：HaptX Gloves G1

### 4.1 竞品定位

HaptX 是高端触觉手套代表，定位在企业级 VR 训练、机器人遥操作和 AI 训练。它不是普通消费级 VR 手套，而是“真实触觉 + 力反馈 + 高精度手部追踪”的系统级方案。

### 4.2 产品能力

| 能力 | 说明 |
|---|---|
| 触觉反馈 | 每只手 135 个微流体触觉反馈点 |
| 皮肤位移 | 通过微流体执行器物理推动皮肤 |
| 力反馈 | 每个手指最高约 8 lbf，阻止手指继续弯曲 |
| 手部追踪 | 每只手 36 DoF |
| 精度 | 官方标称 0.3 mm RMS 位置分辨率 |
| 电源/气源 | 需要 Airpack，支持桌面或背包配置 |
| SDK | 支持 Unity、Unreal、低层 C++ API、ROS 1/ROS 2 |
| 场景 | VR 训练、机器人控制、机器人 AI 训练 |

### 4.3 优势

1. 真实触觉能力最强，不只是振动。
2. 同时有触觉反馈、力反馈和手部追踪。
3. 明确支持机器人系统，包含 ROS 节点。
4. 有机器人遥操作案例，例如与 Shadow Robot 相关的触觉遥操作研究。
5. 技术壁垒高，专利和微流体技术形成护城河。

### 4.4 弱点和机会

1. 系统复杂，需要 Airpack，穿戴和部署门槛高。
2. 成本高，价格不透明，更偏大客户采购。
3. 不适合低成本批量部署。
4. 对国内客户来说，本地交付、二次开发、中文支持可能是机会。
5. 如果客户只需要输入采集或基础反馈，HaptX 可能过度配置。

### 4.5 对我们的启发

如果我们做高端机器人遥操作手套，HaptX 是天花板级对标对象。但不建议一开始正面硬打 HaptX。更现实的切入点是：

1. 做国产、轻量、低成本的机器人遥操作输入手套。
2. 先支持手指姿态采集和基础触觉反馈。
3. 逐步扩展到压力反馈或力反馈。
4. 优先做好 ROS / Python / Unity / Unreal SDK。

## 5. 竞品二：SenseGlove Nova 2

### 5.1 竞品定位

SenseGlove 是工业和科研场景中较成熟的力反馈手套公司。Nova 2 面向 VR 训练、机器人遥操作、研究和多人交互。相比 HaptX，它更轻、更实用、更接近可规模交付的专业产品。

### 5.2 产品能力

| 能力 | 说明 |
|---|---|
| 力反馈 | 拇指、食指、中指、无名指有力反馈，不含小指 |
| 单指反馈力 | 官方文档标称最高 20N |
| 掌心反馈 | Active Contact Feedback，掌心压力反馈 |
| 振动反馈 | 支持按钮点击、震动、碰撞等触觉效果 |
| 手指追踪 | 传感器手指追踪 + 视觉手部追踪结合 |
| 使用时长 | 官方资料显示约 3 小时仿真时间 |
| 耐久性 | 官方资料提到经过 100 万次使用循环测试 |
| 价格 | 官网显示 Nova 2 约 EUR 6,299 或 USD 5,999 起，不含税 |

### 5.3 优势

1. 力反馈、掌心反馈、振动反馈组合完整。
2. 无线、相对轻量，穿戴和部署比 HaptX 更容易。
3. 明确覆盖 XR 训练、机器人遥操作和研究。
4. 价格公开，采购路径比 HaptX 清晰。
5. 有工业客户背书，例如汽车、航空、研究机构等方向。

### 5.4 弱点和机会

1. 力反馈不覆盖小指。
2. 反馈是通过机械线缆和制动阻力实现，不是完整真实触觉。
3. 对国内客户仍是海外采购，价格较高。
4. SDK、本地化、定制和机器人适配可能存在服务机会。
5. 如果客户想要更多传感数据，例如指尖压力、接触面积、抓握力，SenseGlove 主要还是偏交互反馈，不一定直接提供完整传感手套方案。

### 5.5 对我们的启发

SenseGlove 是最适合做“专业可落地产品”对标的竞品。我们可以考虑做：

1. 输入型 + 基础振动反馈的低配版。
2. 输入型 + 指尖压力/触觉传感的机器人示教版。
3. 输入型 + 简单阻力反馈的专业版。

## 6. 竞品三：bHaptics TactGlove DK2

### 6.1 竞品定位

bHaptics TactGlove DK2 是低成本、轻量化、偏消费和开发者生态的触觉手套。它不是高精度力反馈手套，也不是机器人遥操作高端方案，而是通过振动反馈增强 VR/AR 手部交互。

### 6.2 产品能力

| 能力 | 说明 |
|---|---|
| 反馈类型 | LRA 振动反馈 |
| 反馈点 | 一副手套共 12 个触觉反馈点，每只手 6 个 LRA |
| 连接 | BLE 蓝牙 |
| 续航 | 约 3.5 小时 |
| 兼容 | Meta Quest、PC VR、PS VR 等生态 |
| 价格 | 官网价格 $269 |
| SDK/生态 | bHaptics 软件和内容生态，游戏兼容数量多 |

### 6.3 优势

1. 价格极低，容易购买。
2. 轻量、无线、穿戴简单。
3. 适合 VR 内容体验和开发者测试。
4. 有 bHaptics 现成内容生态。
5. 对消费级用户和入门开发者很友好。

### 6.4 弱点和机会

1. 没有力反馈。
2. 不做高精度手指追踪，更多依赖头显或外部手部追踪。
3. 触觉是振动级别，不能模拟真实接触压力。
4. 不适合作为严肃机器人遥操作主手套。
5. 对机器人和科研客户的数据价值有限。

### 6.5 对我们的启发

bHaptics 证明了一个低成本触觉手套市场存在，但它的方向不是机器人主控，而是“触觉反馈外设”。如果我们想做低价版本，可以参考：

1. 轻量穿戴。
2. 蓝牙连接。
3. 简单 SDK。
4. 低价格。
5. 消费/开发者优先。

但如果我们要做机器人遥操作，不能只做到 bHaptics 这个反馈水平。

## 7. 竞品四：MANUS Metagloves Pro Haptic / Quantum Metagloves

### 7.1 竞品定位

MANUS 是专业手部动作捕捉和手套产品公司。Quantum Metagloves 偏输入型高精度动捕，Metagloves Pro Haptic 则把高精度手部追踪和实时振动反馈结合，面向遥操作、XR 和具身 AI 数据采集。

### 7.2 产品能力

| 能力 | 说明 |
|---|---|
| 追踪技术 | EMF 或 Quantum 追踪，强调无漂移和无 occlusion |
| 自由度 | Metagloves Pro Haptic 官方标称 25 DoF |
| 反馈类型 | 实时振动触觉反馈 |
| 数据价值 | 适合机器人训练、动作科学、AI 研究 |
| 模块化 | 顶部电子模块可拆换，传感手套个人化 |
| 授权 | MANUS Core、SDK、Linux、XR 等授权方案 |

### 7.3 优势

1. 手部追踪数据质量强。
2. 适合动作捕捉、机器人训练和具身 AI 数据。
3. 软件、SDK 和专业动捕生态完整。
4. 模块化设计利于多人使用和卫生管理。
5. 作为输入型手套，是很强的专业参照。

### 7.4 弱点和机会

1. 触觉反馈主要是振动，不是 HaptX/SenseGlove 那类强力反馈。
2. 价格不透明，需要询价。
3. 对国内机器人客户，本地化、定制协议和成本仍有机会。
4. 如果客户需要指尖压力传感或接触数据，MANUS 更偏动作捕捉和反馈提示，不是触觉传感本体。

### 7.5 对我们的启发

如果我们的产品目标是“机器人示教 / 遥操作 / 数据采集手套”，MANUS 是很重要的对标对象。我们需要明确自己是：

1. 更便宜的 MANUS。
2. 加触觉传感的 MANUS。
3. 面向国产机器人生态的 MANUS。
4. 面向科研实验室的可定制手套。

## 8. 竞品五：Noitom Hi5 / Hi5 2.0

### 8.1 竞品定位

Noitom Hi5 是国内较典型的 VR 交互手套，核心能力是手部动作捕捉和 VR 交互，属于输入型手套，带基础振动反馈。

### 8.2 产品能力

| 能力 | 说明 |
|---|---|
| 传感器 | 每只手多个 9 轴 IMU |
| 动作捕捉 | 全手和手指姿态捕捉 |
| 输出帧率 | 英文资料显示 120fps 输出，国内 Hi5 2.0 页面提到 500fps 姿态计算 |
| 反馈 | 基础振动反馈 |
| SDK | Unity、Unreal 等开发支持 |
| 生态 | VR 头显、Pico、HTC Vive 等 |

### 8.3 优势

1. 国内品牌，交付和沟通更容易。
2. 动捕和 VR 交互经验较多。
3. 适合输入型和 VR 仿真场景。
4. 相比海外高端手套，可能更有价格和本地服务优势。

### 8.4 弱点和机会

1. 不是强触觉反馈或力反馈手套。
2. 不一定适合高端机器人遥操作。
3. IMU 方案可能存在漂移和磁环境影响，需要校准。
4. 如果我们有柔性压力传感或指尖触觉传感，可以和 Noitom 拉开差异。

### 8.5 对我们的启发

Noitom 更适合作为国内输入型手套参照。如果我们要从国内市场切入，可以考虑避开纯动捕，强调：

1. 指尖压力/触觉传感。
2. 抓握力数据。
3. 机器人遥操作协议。
4. ROS / Python SDK。
5. 低成本定制。

## 9. 输入型手套 vs 反馈型手套

### 9.1 输入型手套

目标是把人的手部动作变成数据。

核心功能：

1. 手指弯曲角度。
2. 手指外展/内收。
3. 手掌姿态。
4. 手势识别。
5. 关节角数据。
6. SDK 输出。
7. 机器人控制映射。

典型竞品：

1. MANUS Quantum。
2. Noitom Hi5。
3. MANUS Metagloves Pro Haptic 的输入部分。

适合客户：

1. 机器人示教。
2. 动作捕捉。
3. VR 手部交互。
4. 具身 AI 数据采集。

### 9.2 振动反馈手套

目标是通过振动提示用户发生了接触、碰撞或操作事件。

核心功能：

1. 指尖振动。
2. 手腕振动。
3. 事件触发反馈。
4. 波形控制。
5. 游戏或 VR 内容集成。

典型竞品：

1. bHaptics TactGlove。
2. Noitom Hi5。
3. MANUS Pro Haptic。

适合客户：

1. VR 内容。
2. 训练提示。
3. 低成本触觉反馈。
4. 开发者生态。

### 9.3 力反馈 / 真实触觉手套

目标是让用户感觉到物体大小、硬度、阻力、接触和压力。

核心功能：

1. 手指阻力反馈。
2. 掌心压力反馈。
3. 皮肤位移触觉。
4. 真实接触模拟。
5. 遥操作触觉闭环。

典型竞品：

1. HaptX Gloves G1。
2. SenseGlove Nova 2。

适合客户：

1. 机器人遥操作。
2. 高端 VR 训练。
3. 医疗/工业模拟。
4. 人形机器人 AI 训练。

## 10. 我们的差异化机会

### 10.1 机会一：国产机器人遥操作输入手套

切入点：

1. 不先做复杂力反馈。
2. 先做好手指姿态、手势、抓握动作和 SDK。
3. 重点支持 ROS、Python、Unity、Unreal。
4. 面向国产人形机器人、灵巧手、科研院所。

可对标：

1. MANUS。
2. Noitom。

差异化：

1. 更低价格。
2. 更强机器人协议适配。
3. 支持指尖压力/触觉传感扩展。
4. 中文本地化和快速定制。

### 10.2 机会二：带指尖压力传感的触觉采集手套

切入点：

1. 不只是采集手指角度，还采集指尖压力。
2. 用于抓握数据、示教数据、机器人学习。
3. 输出抓握力、接触点、触觉事件。

可对标：

1. MANUS 输入型手套。
2. HaptX 机器人遥操作方向。
3. SenseGlove 机器人训练方向。

差异化：

1. 触觉传感数据，而不是单纯触觉反馈。
2. 更适合具身智能数据采集。
3. 可和机器人皮肤/触觉传感器产品线联动。

### 10.3 机会三：低成本振动反馈开发者手套

切入点：

1. 低价。
2. 蓝牙。
3. 简单 SDK。
4. 适合教学、VR 课程、开发者实验。

可对标：

1. bHaptics TactGlove。
2. Noitom Hi5 的基础反馈。

风险：

1. 容易陷入消费电子价格战。
2. 对机器人业务价值不如输入型和触觉采集型。

## 11. 建议定位

不建议一开始定位为“HaptX 平替”。HaptX 是高端真实触觉系统，技术复杂、成本高，直接对标风险大。

更建议的首发定位：

**面向机器人遥操作和具身智能数据采集的国产触觉采集手套。**

核心卖点：

1. 手指姿态采集。
2. 指尖压力/触觉数据采集。
3. 抓握事件识别。
4. 低延迟 SDK。
5. 支持 ROS / Python / Unity / Unreal。
6. 中文软件和本地定制。

后续再做反馈分层：

1. 基础版：输入型手套。
2. Pro 版：输入 + 振动反馈。
3. Robotics 版：输入 + 指尖压力/触觉传感。
4. Haptic 版：输入 + 指尖压力 + 阻力/力反馈。

## 12. 12-18 个月风险

1. HaptX 和 SenseGlove 会继续向机器人 AI 训练和遥操作靠近。
2. MANUS 这类动捕厂商会加强 haptic feedback 和机器人数据采集定位。
3. bHaptics 可能继续降低触觉反馈硬件价格，影响低端市场。
4. 国内 Noitom 等厂商已经有 VR 手套和动捕积累，输入型手套不容易只靠低价胜出。
5. 如果我们没有明确触觉传感差异，容易被归类为普通动捕手套。
