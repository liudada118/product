# 视觉 + 触觉同步采集价值调研

文档日期：2026-05-12  
公司方向：纤维织物触觉传感器、触觉鞋垫、触觉全身衣、触觉手套  
目标客户：世界模型公司、机器人基础模型公司、人形机器人公司、具身智能实验室、AR/VR 与仿真训练团队  
使用 Skill：`competitor-analysis` + `value-proposition`

## 1. 核心结论

| 结论 | 说明 |
|---|---|
| 世界模型和机器人基础模型正在从“看世界”走向“进入物理世界” | Meta V-JEPA 2、Google Gemini Robotics、Figure Helix、Physical Intelligence π0 等都强调物理世界理解、机器人控制和真实交互。 |
| 第一人称视觉数据已经被大规模验证，但触觉数据仍是明显缺口 | Ego4D / Ego-Exo4D 已经证明第一人称视频、音频、眼动、IMU、第三人称同步视频的价值，但公开数据集和产品中很少有大规模同步触觉。 |
| 触觉是物理交互中视觉无法替代的模态 | 接触发生、压力大小、滑移、重量、软硬、摩擦、受力分布、人与人/机器人与人接触安全，很多信息不能只靠 RGB 视频可靠推断。 |
| 你们的独特机会是“人和机器人都能穿戴的触觉数据层” | 触觉鞋垫、全身衣、手套可以同时服务人体示教数据采集、机器人皮肤/末端触觉、人与机器人交互安全监测。 |
| 推荐定位不是“传感器公司”，而是“给物理世界模型补上触觉通道的数据采集基础设施” | 对大模型公司，价值不只是硬件，而是同步数据、标定、SDK、数据格式、可训练样本和任务 benchmark。 |

一句话定位：

> 为具身智能和世界模型提供视觉 + 触觉同步采集系统，让 AI 不只看到物理世界，还能学习接触、受力、滑移、材质和人机互动。

## 2. 市场背景

### 2.1 世界模型公司的趋势

当前 AI 从文本、图像、视频理解走向“物理世界理解”。几个公开方向很有代表性：

| 方向 | 代表项目 | 重点 |
|---|---|---|
| 第一人称视觉理解 | Meta Ego4D、Ego-Exo4D、Project Aria | 用人类第一视角采集日常活动、手物交互、技能活动和空间数据。 |
| 视频世界模型 | Meta V-JEPA 2 | 从视频中理解、预测和规划物理世界变化，并尝试用于机器人控制。 |
| 机器人 VLA/VLM | Google RT-2、Gemini Robotics | 把视觉、语言和动作连接起来，让机器人能在真实世界执行任务。 |
| 人形机器人 VLA | Figure Helix | 以视觉、语言和动作模型控制人形机器人上半身和手指。 |
| 通用机器人基础模型 | Physical Intelligence π0、Skild AI | 跨机器人形态、跨任务训练通用物理智能。 |
| 机器人触觉 | Meta Sparsh、Digit 360、ReSkin | 触觉被明确视为具身 AI、灵巧操作和人机交互的重要组成。 |

关键判断：行业已经认可“真实物理交互数据”是下一阶段 AI 的瓶颈，但目前主流数据仍偏视觉、语言、动作轨迹和机器人关节状态，触觉数据还没有像视觉数据一样规模化、标准化。

### 2.2 为什么第一人称视觉还不够

第一人称视觉可以回答：

1. 人看到了什么。
2. 手在哪里。
3. 物体是什么。
4. 动作大致是什么。
5. 环境结构是什么。

但视觉很难稳定回答：

1. 手到底有没有接触到物体。
2. 接触压力有多大。
3. 物体是否开始滑动。
4. 人是否用了过大的抓握力。
5. 软材料、布料、床品、衣物、人体接触时的受力状态。
6. 鞋底和地面之间的压力变化、重心转移和步态稳定性。
7. 机器人与人接触是否安全。
8. 两个机器人协作搬运时接触力是否合理。

这正是触觉传感器的价值入口。

## 3. 视觉 + 触觉同步数据的核心价值

### 3.1 对世界模型

| 能力 | 只有视觉时的问题 | 加入触觉后的价值 |
|---|---|---|
| 接触理解 | 视觉只能猜测是否接触，遮挡时更困难 | 触觉直接给出接触发生、接触位置和压力分布。 |
| 物理属性学习 | 材质、软硬、摩擦、重量经常不可见 | 触觉能提供形变、压力、滑移、接触时间等监督信号。 |
| 动作结果预测 | 视频看到动作结果，但不知道力学原因 | 触觉补充“为什么物体动了/没动/滑了/变形了”。 |
| 人类意图理解 | 视觉看到手势，但看不出力度和谨慎程度 | 抓握力、足底重心、身体接触能反映意图和控制策略。 |
| 失败案例学习 | 失败可能在视觉上发生得很晚 | 触觉可更早发现滑移、失衡、碰撞、过载。 |

### 3.2 对机器人学习

| 任务 | 触觉数据价值 |
|---|---|
| 抓取 | 判断接触、握持稳定性、滑移和夹持力。 |
| 布料/柔性物体操作 | 视觉难以恢复真实受力，触觉能提供拉扯、折叠、压迫、滑动信息。 |
| 行走和平衡 | 触觉鞋垫可提供足底压力、重心转移、步态周期和地面接触质量。 |
| 人机协作 | 全身衣/机器人皮肤可记录接触区域、接触压力和安全边界。 |
| 模仿学习 | 人穿戴采集动作和触觉，机器人侧用相同或相近触觉布局学习交互策略。 |
| 机器人自监督学习 | 机器人通过视觉预测触觉、通过触觉纠正视觉，形成跨模态自监督任务。 |

### 3.3 对数据资产

视觉 + 触觉同步采集形成的数据，不只是普通传感器记录，而是可训练的数据资产：

| 数据类型 | 可训练任务 |
|---|---|
| 第一人称视频 + 手套压力 | 预测抓取成功、滑移、抓握力度、物体软硬。 |
| 第一人称视频 + 鞋垫压力 | 预测步态阶段、重心、地面接触稳定性、跌倒风险。 |
| 第一人称视频 + 全身接触 | 预测人与环境/人与人/人与机器人接触事件。 |
| 机器人相机 + 机器人触觉皮肤 | 学习碰撞检测、安全接触、协作搬运、灵巧操作。 |
| 人体采集 + 机器人采集对齐 | 做人体示教到机器人执行的数据迁移。 |

## 4. 竞争格局

这里的“竞品”不是完全同类硬件公司，而是围绕“物理世界数据采集与具身 AI 训练”的关键玩家。

### 4.1 Meta Ego4D / Ego-Exo4D / Project Aria

| 项目 | 说明 |
|---|---|
| 定位 | 第一人称和多视角人类活动数据集，服务 egocentric AI、AR、空间智能和具身 AI。 |
| 强项 | 数据规模大，覆盖日常活动、技能活动、音频、眼动、IMU、多视角视频、3D 点云等。 |
| 弱项/空白 | 公开资料重点是视觉、音频、眼动、IMU 和空间数据，没有把全身触觉、手部压力、足底压力作为核心模态。 |
| 对我们的启发 | 可以对标其第一人称采集范式，但补上触觉通道，形成“Ego + Touch”的差异化数据集。 |

差异化机会：

1. 做第一人称视频 + 触觉手套 + 鞋垫 + 全身衣的同步数据样例。
2. 把触觉标注成接触事件、压力峰值、滑移、受力区域、身体接触安全等级。
3. 面向机器人训练而不是纯视频理解，定义触觉 benchmark。

### 4.2 Meta FAIR 触觉路线：Sparsh / Digit 360 / ReSkin

| 项目 | 说明 |
|---|---|
| 定位 | 给机器人和 AI 研究提供触觉编码器、指尖触觉传感器和触觉皮肤。 |
| 强项 | Meta 明确提出 touch perception 对具身 AI 关键；Digit 360、Sparsh、ReSkin 都是很强的触觉研究信号。 |
| 弱项/空白 | Digit 360 偏机器人指尖，ReSkin 偏贴片式触觉皮肤；不是面向人穿戴的大面积纤维织物触觉采集系统。 |
| 对我们的启发 | 行业头部玩家已经在教育市场“触觉是 AI 的关键模态”，你们可以顺势主张“大面积、可穿戴、可人机共用”的触觉数据层。 |

差异化机会：

1. 与指尖触觉不同，你们可覆盖手、脚、全身接触。
2. 与机器人专用皮肤不同，你们可以同时采集人体示教和机器人执行数据。
3. 与光学触觉不同，纤维织物形态更适合穿戴、柔性表面、大面积覆盖和长时间采集。

### 4.3 Google DeepMind RT-2 / Gemini Robotics

| 项目 | 说明 |
|---|---|
| 定位 | Vision-Language-Action 模型，把视觉、语言和机器人动作连接起来。 |
| 强项 | 强大的多模态基础模型、机器人控制、跨任务泛化、仿真和真实机器人能力。 |
| 弱项/空白 | 公开叙事主要围绕视觉、语言、动作和机器人示范，触觉不是其公开核心采集栈。 |
| 对我们的启发 | 可以把你们的数据定位为 VLA 模型的下一类训练输入：Vision-Language-Touch-Action。 |

差异化机会：

1. 提供可对接机器人学习的数据格式：`vision, touch, proprioception, action, language, reward`。
2. 用触觉数据补充 VLA 在接触物理上的短板。
3. 设计“视觉看不清但触觉能判断”的评测任务，例如滑移、夹持力、柔性物体折叠。

### 4.4 Figure Helix

| 项目 | 说明 |
|---|---|
| 定位 | 人形机器人 Vision-Language-Action 模型，控制上半身、手腕、手指等连续动作。 |
| 强项 | 人形机器人形态清晰，面向家庭和物流任务，展示了多机器人协作和泛化操作。 |
| 弱项/空白 | 公开技术重点仍是视觉、语言和动作控制；对全身触觉、接触安全、机器人皮肤数据的公开强调较少。 |
| 对我们的启发 | 人形机器人越接近人类任务，越需要触觉鞋垫、手套/手部触觉、躯干接触和人机接触安全数据。 |

差异化机会：

1. 给人形机器人提供“机器人穿戴版触觉衣/鞋垫/手套”，采集真实接触数据。
2. 给人类操作员提供同构采集设备，形成“人示教数据”和“机器人执行数据”的对齐。
3. 用两机器人协作搬运、递物、避碰演示触觉的必要性。

### 4.5 Physical Intelligence π0 / Skild AI 等通用机器人基础模型

| 项目 | 说明 |
|---|---|
| 定位 | 通用机器人基础模型，强调跨任务、跨机器人形态的数据和策略泛化。 |
| 强项 | 模型目标很接近“物理世界智能”，对高质量真实数据需求强。 |
| 弱项/空白 | 公开内容多强调机器人动作、视觉和跨形态控制；触觉采集标准化仍是产业空白。 |
| 对我们的启发 | 这类公司是潜在客户，不一定是竞品。它们需要多模态真实世界数据，触觉可能成为数据壁垒。 |

差异化机会：

1. 做“可量产的数据采集套件”，而不是单个实验室传感器。
2. 提供跨人、跨机器人、跨任务的一致触觉数据 schema。
3. 用纤维织物传感器覆盖机器人皮肤、人体穿戴和场景物体表面。

## 5. 未被充分满足的需求

| 未满足需求 | 当前替代方案 | 你们的机会 |
|---|---|---|
| 大面积、柔性、可穿戴触觉采集 | 少量指尖触觉、力传感器、动捕、IMU | 用织物触觉覆盖手、足、躯干和机器人表面。 |
| 人类第一人称视觉与身体触觉同步 | 头戴相机 + IMU + 动作标签 | 加入触觉鞋垫、手套、全身衣，做完整身体交互记录。 |
| 机器人触觉与人类示教数据对齐 | 机器人相机 + 关节数据 + 遥操作 | 人和机器人都穿戴相似触觉布局，降低模态迁移难度。 |
| 接触安全数据 | 碰撞检测、关节力矩估计 | 直接记录接触区域、压力峰值、持续时间和安全阈值。 |
| 柔性物体/人体交互数据 | 视觉 + 手工标注 | 触觉能提供布料、床品、人体接触和软体变形的真实监督。 |
| 可训练数据标准 | 各实验室自定义格式 | 提供统一数据格式、SDK、标定、时间同步和 benchmark。 |

## 6. 价值主张：6-Part JTBD

### 6.1 Who

目标客户是正在构建物理世界智能的团队：

1. 世界模型公司：需要视频之外的物理监督信号。
2. 机器人基础模型公司：需要真实、多模态、可规模化的数据。
3. 人形机器人公司：需要人类示教、人机协作和安全接触数据。
4. AR/VR/仿真训练公司：需要把虚拟交互与真实触觉信号绑定。
5. 高校和研究机构：需要可复现实验设备和数据集。

### 6.2 Why

客户的核心 Job to Be Done：

> 当他们训练能进入物理世界的 AI/机器人时，希望模型不仅知道“看到了什么”和“应该做什么动作”，还知道“是否接触、接触在哪里、用了多大力、是否滑动、是否安全”。

### 6.3 What Before

客户今天通常使用：

1. 第一人称相机、第三人称相机、深度相机。
2. IMU、动捕、机器人关节状态。
3. 机器人夹爪力矩、少量指尖传感器。
4. 人工标注接触事件。
5. 仿真环境中的接触力。

痛点：

1. 视觉无法可靠表达接触和受力。
2. 指尖触觉覆盖范围太小。
3. 人体穿戴数据和机器人执行数据很难对齐。
4. 数据采集链路碎片化，时间同步和标定成本高。
5. 缺少能直接训练模型的触觉数据格式和 benchmark。

### 6.4 How

你们的解决方案：

| 产品 | 数据价值 |
|---|---|
| 触觉手套 | 手部接触、抓握压力、滑移、物体软硬、操作策略。 |
| 触觉鞋垫 | 足底压力、步态、重心转移、地面接触、平衡控制。 |
| 触觉全身衣 | 躯干/四肢接触、人机接触安全、身体姿态与受力关系。 |
| 机器人穿戴触觉织物 | 机器人皮肤、末端执行器、协作接触、碰撞和安全数据。 |
| 同步采集 SDK | 视觉、触觉、IMU、动作、语言、事件标签统一时间轴。 |

### 6.5 What After

客户获得的新能力：

1. 建立视觉 + 触觉 + 动作的数据集。
2. 训练能理解接触物理的世界模型。
3. 让机器人学习更稳定、更安全的抓取和协作。
4. 把人体示教数据迁移到机器人执行数据。
5. 形成私有多模态数据壁垒。

### 6.6 Alternatives

| 替代方案 | 为什么不够 |
|---|---|
| 只用视频 | 无法直接测量接触力、滑移、压力分布。 |
| 只用 IMU/动捕 | 能描述运动，不能描述接触物理。 |
| 只用机器人关节力矩 | 间接、粗粒度，难定位具体接触区域。 |
| 只用指尖触觉 | 不能覆盖足底、全身、躯干、人机接触。 |
| 仿真接触力 | 和真实世界差距大，尤其是柔性材料、人体、布料、复杂摩擦。 |

价值主张语句：

> 对正在训练物理世界 AI 的模型团队，我们提供人和机器人都可穿戴的纤维织物触觉采集系统，把第一人称视觉、身体触觉和动作数据同步成可训练数据，让模型学习视觉看不到的接触、压力、滑移、材质和安全交互。

## 7. 推荐 Demo

### Demo 1：第一人称抓取，视觉看得到，触觉解释为什么成功

配置：

1. 头戴 RGB/深度相机。
2. 触觉手套。
3. 若可行，腕部 IMU 或手部姿态追踪。

任务：

1. 抓杯子、海绵、布料、工具、易碎物。
2. 有意制造滑移、过轻抓取、过重抓取。

展示指标：

1. 接触发生时间。
2. 各手指压力曲线。
3. 滑移前的压力变化。
4. 视觉帧 + 触觉热力图同步回放。

要表达的价值：

> 视频告诉模型“手碰到了杯子”，触觉告诉模型“是否真的抓稳、用了多大力、什么时候开始滑”。

### Demo 2：触觉鞋垫 + 第一人称视觉，展示步态和重心

配置：

1. 第一人称相机。
2. 触觉鞋垫。
3. 可选全身 IMU。

任务：

1. 正常走路、转身、上下台阶。
2. 搬重物走路。
3. 模拟失衡或急停。

展示指标：

1. 足底压力热力图。
2. 重心轨迹。
3. 步态周期。
4. 视觉画面与足底接触同步。

要表达的价值：

> 第一人称视觉看不到脚底真实接触，鞋垫触觉让模型学习行走、平衡和地面交互。

### Demo 3：人机接触安全，视觉无法判断压力是否安全

配置：

1. 人穿触觉全身衣。
2. 机器人或协作机械臂末端接触人体。
3. 外部相机 + 第一人称相机。

任务：

1. 机器人递物、扶人、轻触、靠近。
2. 记录安全接触和危险接触。

展示指标：

1. 接触位置。
2. 压力峰值。
3. 接触持续时间。
4. 安全阈值告警。

要表达的价值：

> 对机器人来说，“碰到人”不是二值事件，而是接触位置、压力、持续时间和人体接受度的连续问题。

### Demo 4：机器人穿戴触觉皮肤，学习机器人与物理世界接触

配置：

1. 机器人手、前臂或躯干贴合纤维织物触觉传感器。
2. 机器人相机。
3. 机器人关节状态。

任务：

1. 推门、扶墙、搬箱子、与人协作递物。
2. 两机器人协作搬运。

展示指标：

1. 机器人视觉。
2. 机器人触觉分布。
3. 关节动作。
4. 接触事件检测。

要表达的价值：

> 机器人需要的不只是“眼睛”，还需要可覆盖身体和末端的“皮肤”。

## 8. 推荐数据格式

建议你们把产品包装成“数据采集基础设施”，而不是只给客户一堆传感器。

### 8.1 最小数据包

| 字段 | 说明 |
|---|---|
| `session_id` | 一次采集任务 ID。 |
| `timestamp_ns` | 纳秒或毫秒级统一时间戳。 |
| `camera_frame_id` | 对应视觉帧。 |
| `ego_video_path` | 第一人称视频路径。 |
| `touch_device_id` | 手套、鞋垫、全身衣、机器人皮肤设备 ID。 |
| `touch_map` | 触觉矩阵或传感点数组。 |
| `touch_region` | 手指、掌心、足跟、前掌、胸、背、前臂等区域。 |
| `pressure_value` | 原始压力值或标定后压力值。 |
| `imu` | 可选姿态数据。 |
| `action_label` | 抓取、行走、推、拉、递物等任务标签。 |
| `contact_event` | 接触开始、稳定接触、滑移、脱离、过载。 |
| `language_instruction` | 可选自然语言任务指令。 |
| `robot_state` | 可选机器人关节、末端位姿、夹爪状态。 |

### 8.2 可训练任务标签

| 标签 | 用途 |
|---|---|
| `contact_start` / `contact_end` | 训练接触事件检测。 |
| `grasp_success` | 训练抓取成功预测。 |
| `slip` | 训练滑移检测和预防。 |
| `over_pressure` | 训练安全接触和过载检测。 |
| `material_type` | 训练材质/软硬识别。 |
| `gait_phase` | 训练步态和运动控制。 |
| `human_comfort_level` | 训练人机接触舒适度。 |

## 9. 产品化建议

### 9.1 对外产品包

| 产品包 | 目标客户 | 内容 |
|---|---|---|
| EgoTouch 数据采集套件 | 世界模型/多模态模型团队 | 第一人称相机同步、触觉手套、鞋垫、SDK、数据导出。 |
| RobotSkin 触觉感知套件 | 人形机器人/机械臂公司 | 机器人可穿戴触觉织物、触觉手套/末端贴片、接触检测 API。 |
| Human-Robot Interaction 安全套件 | 协作机器人/康复/服务机器人 | 全身衣、机器人触觉皮肤、安全阈值、接触热力图。 |
| Touch Dataset Studio | 研究机构/数据团队 | 回放、标注、同步、导出、benchmark 工具。 |

### 9.2 对外话术

不建议主打：

1. “我们有柔性压力传感器。”
2. “我们能采集压力数据。”
3. “我们做触觉鞋垫、手套、衣服。”

建议主打：

1. “给物理世界模型补上触觉数据。”
2. “让机器人学习接触，而不只是看见。”
3. “人类示教和机器人执行之间的同构触觉数据层。”
4. “视觉解释物体，触觉解释交互。”
5. “从第一人称视频到第一人称触觉。”

## 10. 12 个月路线建议

| 阶段 | 时间 | 目标 | 交付 |
|---|---|---|---|
| P0 | 0-2 个月 | 做出视觉 + 触觉同步 demo | 手套/鞋垫 + 第一人称相机同步回放。 |
| P1 | 2-4 个月 | 做出可训练样例数据集 | 10-20 个任务、统一 schema、标注工具。 |
| P2 | 4-6 个月 | 做机器人侧触觉 demo | 机器人穿戴触觉织物，接触事件检测。 |
| P3 | 6-9 个月 | 做多模态 benchmark | 视觉 vs 视觉+触觉在滑移/抓取/步态任务上的对比。 |
| P4 | 9-12 个月 | 试点客户合作 | 世界模型/机器人公司联合数据采集 PoC。 |

## 11. 对客户最有说服力的评测

| 评测 | 对照组 | 预期展示 |
|---|---|---|
| 滑移预测 | 只用视频 vs 视频+手套触觉 | 触觉提前发现滑移趋势。 |
| 抓取力度估计 | 只用视频 vs 视频+触觉 | 触觉显著提高力度估计可靠性。 |
| 步态阶段识别 | 只用第一人称视频 vs 视频+鞋垫 | 鞋垫提升步态和重心判断。 |
| 人机接触安全 | 只用视觉碰撞检测 vs 全身触觉 | 触觉能给出接触压力和区域。 |
| 柔性物体操作 | 只用视觉 vs 视频+手部/腕部触觉 | 触觉更好解释布料、海绵、软包形变。 |

## 12. 关键风险

| 风险 | 说明 | 应对 |
|---|---|---|
| 客户把你们当普通硬件供应商 | 只卖传感器会被压价 | 必须卖“同步采集 + 数据格式 + SDK + benchmark”。 |
| 数据同步不好 | 视觉和触觉错位会降低训练价值 | 时间戳、硬件同步、统一回放工具是 P0。 |
| 标定复杂 | 织物传感器不同穿戴状态可能影响数据一致性 | 建立快速标定流程和数据质量评分。 |
| 客户不知道怎么用触觉训练模型 | 触觉数据对 AI 团队仍较新 | 提供 baseline 模型和示例 notebook。 |
| 隐私和人体数据合规 | 第一人称视频 + 身体数据敏感 | 做匿名化、授权协议、局部脱敏和私有部署方案。 |

## 13. 推荐下一步

1. 先做一个 2 分钟演示视频：同屏展示第一人称画面、触觉手套热力图、鞋垫热力图、事件标签。
2. 选择 3 个最能体现触觉价值的任务：滑移抓取、负重行走、人机安全接触。
3. 定义 `EgoTouch` 数据 schema，并导出 10 条样例数据。
4. 做一个“只用视觉 vs 视觉+触觉”的小评测，哪怕先用简单模型，也要量化差异。
5. 准备面向世界模型公司的 pitch：你们不是替代相机，而是补齐物理接触监督信号。
