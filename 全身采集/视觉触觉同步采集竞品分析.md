# 视觉 + 触觉同步采集竞品分析

文档日期：2026-05-21  
公司方向：纤维织物触觉传感器、触觉鞋垫、触觉全身衣、触觉手套、机器人触觉皮肤  
分析对象：面向世界模型、具身智能、机器人基础模型、人形机器人、人机交互的数据采集与触觉感知产品  
使用 Skill：`competitor-analysis`

## 1. 市场定义

这个市场不是传统单一“压力传感器”市场，而是一个正在形成的新交叉市场：

> 为物理世界 AI、机器人基础模型和人机交互系统提供可规模化采集的视觉 + 触觉 + 动作 + 接触数据。

可拆成四个子市场：

| 子市场 | 典型产品 | 关键客户 |
|---|---|---|
| 第一人称多模态采集 | Project Aria、Ego4D / Ego-Exo4D 数据范式 | 世界模型公司、AR/AI 研究团队 |
| 可穿戴触觉 / 动捕设备 | TESLASUIT、MANUS、SenseGlove、bHaptics、StretchSense | XR、仿真训练、机器人遥操作、动作捕捉 |
| 机器人触觉传感 | Meta Digit 360、ReSkin、AnySkin、GelSight 类方案 | 机器人公司、具身智能实验室 |
| 压力分布 / 足底压力 / 接触测量 | Tekscan、XSENSOR、Moticon、medilogic | 医疗康复、运动科学、鞋服、机器人测试 |

你们的差异化位置在交叉点：

1. 人可以穿戴采集：手、足、全身。
2. 机器人可以穿戴感知：皮肤、手、足、躯干、末端执行器。
3. 数据可用于模型训练：视觉、触觉、动作、接触事件统一时间轴。
4. 织物形态天然适合大面积、柔性、身体覆盖和长时间采集。

## 3. 竞品一：Meta Project Aria / Ego4D / FAIR Tactile

### 3.1 竞品画像

| 项目 | 内容 |
|---|---|
| 公司/组织 | Meta Reality Labs、Meta FAIR、合作高校与研究机构 |
| 产品/项目 | Project Aria、Ego4D、Ego-Exo4D、Sparsh、Digit 360、ReSkin |
| 定位 | 第一人称多模态数据平台 + 触觉 AI 研究生态 |
| 客户/用户 | AI 研究者、AR/VR、机器人、机器感知、具身智能团队 |
| 商业模式 | 研究平台、数据集、开源工具、学术生态；不是传统硬件销售模式 |

### 3.2 核心优势

1. 第一人称数据生态领先。Ego4D 官方文档称其是大规模第一人称视频数据集，Ego-Exo4D 进一步提供第一人称和第三人称同步视频、音频、IMU、眼动、3D 信息和技能活动标注。
2. Project Aria 是成体系的研究工具，包含眼镜、数据管理工具、命令行工具、MPS 和 SDK。
3. Meta 同时布局触觉 AI，Sparsh 研究面向 vision-based tactile sensing 的通用触觉表征，Digit 360 是高精度指形触觉传感平台。
4. 具备定义 benchmark 和学术话语权的能力。

### 3.3 弱点和空白

1. 第一人称数据很强，但公开体系里触觉不是主模态，尤其缺少手、足、躯干、机器人皮肤的大面积触觉织物数据。
2. Digit 360、ReSkin 更偏机器人指尖/贴片触觉，不是人穿戴全身触觉系统。
3. Meta 更可能定义平台和标准，不太可能为每个机器人公司提供定制化织物触觉硬件。
4. 数据集覆盖技能活动，但“接触压力、滑移、抓握力度、人机接触安全”仍是视觉难以直接监督的空白。

### 3.4 对你们的威胁

1. 如果 Meta 把触觉数据标准做成事实标准，其他硬件需要适配它的数据格式。
2. 如果 Project Aria 类设备成为第一人称采集默认入口，触觉公司可能被迫成为外挂传感器。
3. Meta 的开源模型和 benchmark 会影响客户对触觉数据的理解方式。

### 3.5 差异化机会

1. 做 `Project Aria / 第一人称相机 + 触觉手套 + 触觉鞋垫 + 全身衣` 的同步采集方案。
2. 把自己定位成 Ego4D/Ego-Exo4D 的触觉补充层：`Ego + Touch`。
3. 主动定义触觉数据 schema：接触事件、压力峰值、滑移、受力区域、人体舒适度。
4. 做世界模型公司最容易理解的对比：只用视频 vs 视频 + 触觉。

## 4. 竞品二：TESLASUIT

### 4.1 竞品画像

| 项目 | 内容 |
|---|---|
| 公司 | TESLASUIT |
| 产品 | 全身 VR haptic suit |
| 定位 | 全身触觉反馈、动作捕捉、生理数据，用于 VR 训练、仿真、安全培训等 |
| 主要客户 | 企业培训、工业安全、汽车、体育、VR/仿真 |
| 商业模式 | 企业销售、项目制、演示和行业解决方案 |

### 4.2 核心优势

1. “全身触觉衣”心智强，官网明确强调 full body haptic feedback、motion capture、biometrics。
2. 与 VR/训练结合紧密，场景演示直观。
3. 覆盖全身，比单一手套或鞋垫更接近你们的“触觉全身衣”方向。
4. 企业级包装成熟，有较强展示效果。

### 4.3 弱点和空白

1. 产品叙事偏“反馈”和“沉浸式训练”，不是“触觉压力数据采集”和“机器人/世界模型训练数据”。
2. 触觉反馈不等于触觉采集。客户可能感受到震动/刺激，但 AI 模型需要结构化接触数据。
3. 与机器人穿戴、机器人皮肤、人机接触安全的数据闭环关联较弱。
4. 系统复杂度和价格门槛可能较高，不一定适合大规模数据采集团队部署。

### 4.4 对你们的威胁

1. 它已经占据“全身触觉衣”关键词。
2. 对 XR/仿真客户有更强品牌认知。
3. 如果客户只想要沉浸式触觉反馈，TESLASUIT 更容易被优先想到。

### 4.5 差异化机会

1. 明确区分：TESLASUIT 偏“给人反馈”，你们偏“采集人和机器人的真实触觉数据”。
2. 强调织物压力分布、接触矩阵和可训练数据，而不只是 haptic feedback。
3. 做机器人可穿戴版本，这是 TESLASUIT 叙事里不强的方向。
4. 用“视觉 + 触觉同步数据集”而不是“VR 体感服”切开新市场。

## 5. 竞品三：HaptX

### 5.1 竞品画像

| 项目 | 内容 |
|---|---|
| 公司 | HaptX |
| 产品 | HaptX Gloves G1 |
| 定位 | 高真实感触觉手套，面向 VR 培训、机器人遥操作、数字环境交互 |
| 主要客户 | 企业培训、机器人、仿真、研究机构 |
| 商业模式 | 企业销售，官网以联系销售为主 |

### 5.2 核心优势

1. 微流体触觉反馈和力反馈能力强，官网称每只手最高 40 lb 阻力反馈。
2. 面向机器人较明确，HaptX SDK 支持 ROS 1 / ROS 2，可用于遥操作、机器学习、计算机视觉。
3. 触觉体验更接近“真实接触”，在高端触觉手套中技术心智强。
4. 手部运动捕捉、触觉反馈、力反馈集成度高。

### 5.3 弱点和空白

1. 主要覆盖手，不覆盖足底、躯干、全身和机器人皮肤。
2. 设备复杂、体积和部署门槛较高，不适合轻量规模化采集。
3. 重点是让操作者“感受到虚拟物体/远端触觉”，不是大面积触觉数据采集标准。
4. 对世界模型公司的“海量第一人称视觉 + 触觉数据采集”叙事不如你们天然。

### 5.4 对你们的威胁

1. 在高端手部触觉和机器人遥操作上非常强。
2. 如果客户只看“触觉手套真实感”，HaptX 是强对标对象。
3. ROS 和机器人遥操作支持会让它进入机器人实验室采购清单。

### 5.5 差异化机会

1. 避免正面打“最真实触觉反馈”，改打“轻量、织物、大面积、可采集、可训练”。
2. 用触觉鞋垫和全身衣补足 HaptX 没覆盖的身体数据。
3. 提供机器人穿戴皮肤，而不只是人戴手套遥操作。
4. 做更低部署门槛的数据采集套件，服务模型团队批量采样。

## 6. 竞品四：SenseGlove

### 6.1 竞品画像

| 项目 | 内容 |
|---|---|
| 公司 | SenseGlove |
| 产品 | Nova / Nova 2 等力反馈触觉手套 |
| 定位 | XR 培训、工业仿真、研究用力反馈手套 |
| 主要客户 | 企业培训、VR/AR、研究机构、机器人遥操作 |
| 商业模式 | 企业/开发者硬件销售，配套 SDK 和文档 |

### 6.2 核心优势

1. Nova 2 具备手指力反馈、掌心 Active Contact Feedback、振动反馈和手部追踪。
2. 设备比 HaptX 更轻量，企业训练场景更容易部署。
3. 文档和 SDK 较完整，XR 开发者友好。
4. 对“抓取不同硬度/大小/密度的虚拟物体”表达清楚。

### 6.3 弱点和空白

1. 依然主要是手套，不覆盖全身和足底。
2. 偏反馈设备，不是面向大模型训练的触觉采集系统。
3. 触觉信息更多用于驱动体验，未形成“视觉+触觉同步数据资产”的商业叙事。
4. 对机器人皮肤、人机接触安全、机器人与机器人接触感知覆盖不足。

### 6.4 对你们的威胁

1. 在工业 XR 训练和力反馈手套方向已成熟。
2. 如果客户希望买现成手套做 VR 交互，SenseGlove 比你们更容易进入候选。
3. 其掌心反馈和力反馈容易被客户理解为“更完整触觉”。

### 6.5 差异化机会

1. 你们的核心不应只是“手套反馈”，而是“手套 + 鞋垫 + 全身衣 + 机器人皮肤”的触觉数据网络。
2. 强调真实压力采集、触觉矩阵、接触事件和模型训练数据。
3. 用第一人称视觉同步能力拉开差异。
4. 主推机器人和世界模型客户，而不是只做 XR 培训外设。

## 7. 竞品五：MANUS

### 7.1 竞品画像

| 项目 | 内容 |
|---|---|
| 公司 | MANUS |
| 产品 | Metagloves Pro、Metagloves Pro Haptic、Quantum Metagloves 等 |
| 定位 | 高精度手部数据手套，面向机器人、AI、VR、动捕 |
| 主要客户 | 机器人、AI、动捕、影视动画、XR |
| 商业模式 | 专业硬件销售和行业方案 |

### 7.2 核心优势

1. 官网强调被 2000+ robotics、AI、immersive tech 客户采用，品牌和客户背书强。
2. 手部追踪精度和动捕生态强，适合机器人示教和手部数据采集。
3. 产品线覆盖普通手套和带 haptic 的版本。
4. 对 AI、机器人、XR 的市场话术已经比较接近你们目标客户。

### 7.3 弱点和空白

1. 核心优势是手部姿态和动捕，不是全手压力分布和真实接触力采集。
2. Haptic 版本更多是振动反馈，不等于可训练的触觉压力矩阵。
3. 不覆盖鞋垫、躯干、全身接触和机器人皮肤。
4. 对第一人称视觉 + 触觉同步数据集的叙事较弱。

### 7.4 对你们的威胁

1. 如果客户的需求是“手部动作数据”，MANUS 可能比你们更成熟。
2. 机器人/AI 客户认知强，容易成为手部示教默认选择。
3. 与动捕、动画、机器人软件链路更成熟。

### 7.5 差异化机会

1. 与其竞争“手姿态精度”不如竞争“手部接触压力 + 视觉同步 + 机器人执行触觉”。
2. 可考虑与 MANUS 类手部动捕生态兼容，而不是完全替代。
3. 用触觉手套输出抓握力、接触区域、滑移事件，作为手部姿态数据的补充。

## 11. 推荐竞争定位

### 11.1 不建议定位

1. 不建议只说“我们是柔性压力传感器公司”。
2. 不建议只对标 HaptX / SenseGlove 做“触觉手套”。
3. 不建议只对标 TESLASUIT 做“触觉全身衣”。
4. 不建议只对标 XSENSOR / Tekscan 做“压力测量”。

这些定位都会把你们缩小到一个硬件品类里。

### 11.2 推荐定位

> 面向物理世界 AI 的视觉 + 触觉同步数据采集基础设施。

更销售化的表达：

> 我们为世界模型和机器人基础模型补上触觉通道，用可穿戴纤维织物传感器同步采集人和机器人在真实世界中的接触、压力、滑移、重心和安全交互数据。

### 11.3 核心差异化

| 差异化 | 说明 |
|---|---|
| 人机同构 | 人能穿，机器人也能穿；可做示教和执行数据对齐。 |
| 大面积覆盖 | 手、足、躯干、机器人表面，不限指尖。 |
| 数据导向 | 输出可训练数据，而不是只做反馈体验。 |
| 视觉同步 | 与第一人称相机、外部相机、机器人相机统一时间轴。 |
| 纤维织物形态 | 柔性、贴合、可大面积、适合穿戴和机器人表面。 |
| 接触事件标签 | 接触开始/结束、滑移、过压、重心、舒适度、安全阈值。 |

## 12. 目标客户打法

| 客户 | 需求 | 主打竞品替代 |
|---|---|---|
| 世界模型公司 | 需要视觉之外的物理监督信号 | 替代“只采视频”的数据路线 |
| 机器人基础模型公司 | 需要可训练的真实接触数据 | 补充 VLA 模型缺少的 Touch 模态 |
| 人形机器人公司 | 需要人类示教、足底接触、手部抓握、人机安全 | 替代单一动捕手套或单一机器人触觉 |
| XR/仿真训练 | 需要触觉反馈和动作数据 | 与 TESLASUIT/SenseGlove 差异化为“可采集可训练” |
| 医疗康复/运动科学 | 需要足底压力和全身接触 | 对标 Moticon/Tekscan，但扩展到全身与视觉同步 |

## 13. 12-18 个月竞争风险

| 风险 | 影响 | 建议 |
|---|---|---|
| Meta 或大模型公司定义触觉数据标准 | 你们被动适配别人的数据格式 | 现在就定义自己的 schema，并兼容 ROS/Gym/Parquet/HDF5 |
| 手套公司加入压力采集 | SenseGlove/MANUS 可能增强触觉采集 | 不只守手套，扩大到鞋垫、全身衣、机器人皮肤 |
| 压力测量公司转向 AI 数据 | XSENSOR/Tekscan/Moticon 有硬件和客户基础 | 抢先讲“视觉+触觉+模型训练” |
| 低价消费级设备拉低客户价格预期 | bHaptics 等让客户觉得触觉设备应很便宜 | 强调数据质量、标定、同步、SDK、benchmark |
| 机器人公司自研触觉皮肤 | 大客户可能内部做 | 先做 PoC，成为数据采集和标定供应商，而不是只卖硬件 |

## 14. 推荐下一步

1. 做一个“竞品矩阵图”：横轴是反馈体验 vs 数据采集，纵轴是局部触觉 vs 全身/机器人大面积触觉。你们应站在“数据采集 + 大面积触觉”象限。
2. 做 `EgoTouch` demo：第一人称视频 + 触觉手套 + 鞋垫 + 全身衣同步回放。
3. 做“视觉 vs 视觉+触觉”的量化评测：滑移预测、抓握力度估计、步态阶段识别、人机接触安全。
4. 准备三套销售话术：世界模型公司、人形机器人公司、XR/仿真训练公司。
5. 对接一个第一人称相机/眼镜生态，例如 Project Aria、GoPro、Quest、手机胸前相机，先证明你们不是相机替代品，而是触觉补充层。
