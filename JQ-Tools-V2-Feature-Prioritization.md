# JQ Tools V2.0 功能优先级评分

## 1. 评分模型

本次评分基于 JQ Tools V2.0 的产品目标：先完成可稳定连接、可实时采集、可视化、可回放、可导出的压力感知软件闭环。

评分维度：

| 维度 | 说明 | 分值 |
|---|---|---|
| 影响 | 对客户价值、销售演示、科研招标、产品闭环的影响 | 1 低 - 5 高 |
| 成本 | 研发、设计、测试、硬件联调成本 | 1 低 - 5 高 |
| 风险 | 技术不确定性、性能压力、需求不清晰、交付风险 | 1 低 - 5 高 |
| 战略匹配 | 是否支撑 V2.0 核心目标和国产替代定位 | 1 低 - 5 高 |

综合分计算：

`综合分 = 影响 * 0.4 + 战略匹配 * 0.3 + (6 - 成本) * 0.2 + (6 - 风险) * 0.1`

说明：

1. 综合分越高，越应该优先做。
2. 成本和风险分越高，表示越难，所以在综合分中反向计算。
3. 该评分用于产品排期，不替代研发工时评估。

### P0 结论

P0 中最应该优先完成的是：

1. 中文界面。
2. 开始/停止采集。
3. CSV 导出。
4. 路径选择对话框。
5. 一键连接、串口检测、MAC 读取、设备类型管理。
6. 2D 热力图和实时统计指标。

需要拆小的 P0：

1. 波特率自动探测：先做固定波特率配置，再做自动探测。
2. 本地离线授权或固件校验：先定授权文件/固件校验方案。
3. 异常恢复基础能力：V2.0 只做“未正常结束的采集记录可恢复”，不要一开始做复杂事务系统。
4. 历史回放：V2.0 先做播放、暂停、进度条、倍速，不做复杂剪辑和对比。

1. CSV 数据导入和外部数据加载。

## 5. Top 5 推荐优先项

### 1. 实时采集闭环

包含：

1. 一键连接。
2. 串口自动检测。
3. MAC 地址读取。
4. 设备类型管理。
5. 开始/停止采集。
6. 本地数据存储。

理由：

这是软件能否被使用的主路径。没有它，2D、3D、统计、回放都没有数据来源。

取舍：

自动波特率探测可以先做最小版本。先支持主波特率，失败后再尝试其他波特率。

### 2. 实时 2D 热力图和统计指标

包含：

1. 2D 热力图。
2. 坐垫/靠背分区显示。
3. 实时统计指标。
4. 画布翻转。
5. 滚轮缩放。
6. 视图重置。

理由：

这是用户最直接感知价值的界面，也是对标 XSENSOR 的基础能力。

取舍：

V2.0 应优先保证 2D 流畅和准确。3D 可延后，不应拖慢主流程。

### 3. 清零和数据可信度

包含：

1. 预压力清零。
2. 清零状态显示。
3. 清零值写入采集记录。
4. 取消置零逻辑。

理由：

清零影响数据可信度。如果只做按钮颜色，不做数据逻辑，客户会质疑测量结果。

取舍：

V2.0 先做全局置零。区域置零、按设备类型保存置零模板可后置。

### 4. 历史回放和 CSV 导出

包含：

1. 历史数据列表。
2. 回放播放/暂停/进度条/倍速。
3. CSV 导出。
4. 路径选择。
5. 导出成功提示。

理由：

研发和科研客户需要把数据带走、复盘和写报告。这是从展示工具升级为分析系统的关键。

取舍：

V2.0 先做单组回放和 CSV 导出。数据导入、双组对比、复杂报告后置。

### 5. 框选区域统计

包含：

1. 单矩形框选。
2. 边界校验。
3. 区域强调显示。
4. 框选区域统计。
5. 可按框选区域导出。

理由：

框选是研发和科研分析中的高价值能力，可以让用户分析局部压力，而不是只看全局。

取舍：

默认必须保存全量原始数据。框选只影响显示、统计和导出筛选，不能默认只采集框选数据。

## 7. 关键决策建议

### 决策 1: V2.0 不以 3D 为核心

3D 有展示价值，但成本和风险都高。V2.0 应以稳定采集、2D 热力图、统计、回放、导出为核心。这样更适合快速交付，也更适合科研招标的基础要求。

### 决策 2: 始终保存全量原始数据

框选不能改变默认采集范围。否则后续回放、复查、科研分析会丢失数据。框选应只影响显示、统计和导出筛选。

### 决策 3: 先 Windows，后 macOS

当前硬件连接涉及 CH340、串口、波特率、驱动和安装包。Windows 应作为 V2.0 主平台。macOS 等有明确客户需求后再排。

### 决策 4: 先定义数据格式，再做导入和对比

CSV 导入、双组对比、外部数据加载都依赖稳定的数据格式。V2.0 必须先定义采集文件和 CSV 版本号，否则 V2.2 会返工。

## 8. 需要研发预研的风险项

| 风险项 | 验证方式 | 建议时间 |
|---|---|---|
| 波特率自动探测稳定性 | 用 3-5 台设备测试识别耗时和成功率 | V2.0 开发前 |
| 2D 热力图帧率 | 用模拟矩阵测试不同点数下的 FPS | V2.0 开发前 |
| 本地缓存和异常恢复 | 模拟断电、强退、长时间采集 | V2.0 中前期 |
| CSV 文件体积 | 测试 10/30/60 分钟采集文件大小 | V2.0 中前期 |
| 清零数据逻辑 | 验证有载、无载、取消置零后的数据变化 | V2.0 中前期 |
| 3D 渲染性能 | Three.js 原型测试高密度点云 | V2.1 前 |
