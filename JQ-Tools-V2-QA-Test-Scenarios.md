# JQ Tools V2.0 P0 QA 测试用例与验收清单

## 1. 测试范围

本文基于 `JQ-Tools-V2-P0-User-Stories.md` 生成，用于 JQ Tools V2.0 P0 功能验收。

覆盖范围：

1. Windows 安装与中文界面。
2. 设备连接、串口检测、波特率尝试、MAC 读取。
3. 设备类型自动识别、设备参数页、离线授权/固件校验。
4. 预压力清零、取消清零、清零记录。
5. 实时采集、停止采集、采集状态保护。
6. 实时 2D 热力图、坐垫/靠背同时显示、数字矩阵、翻转、缩放、重置。
7. 实时统计、矩形框选、精确坐标输入、框选区域统计。
8. 历史数据列表、搜索、重命名、备注、删除、回放。
9. CSV 导出、CSV 格式版本、导出成功提示。
10. 本地缓存、异常恢复、清空缓存。
11. 实时性能和 60fps 目标验证。

不覆盖范围：

1. 3D 模型上传。
2. CSV 导入。
3. 双组数据对比。
4. macOS。
5. 多语言。
6. 复杂 PDF/Word 报告。
7. 多量尺管理。

## 2. 已确认产品口径

| 项目 | 产品口径 |
|---|---|
| 坐垫/靠背显示 | V2.0 需要支持汽车坐垫和汽车靠背同时显示 |
| 设备类型 | 由固件或设备信息自动识别 |
| 采样率 | 由设备返回，软件记录 |
| 预压力清零 | 用户点击后执行 |
| 最小压强 | 统计时排除 0 值 |
| CSV 压力矩阵 | 使用一维数组表 |
| 历史数据路径 | 默认保存在软件安装目录下的历史数据文件夹，设置中可修改 |
| 60fps | 目标指标，不作为硬性阻塞指标 |

待确认：

1. 受压面积压力阈值。
2. 异常缓存保留时长的具体数值。
3. 清零前是否需要提示用户保持无负载。

## 3. 测试环境

### 3.1 硬件环境

| 设备 | 用途 |
|---|---|
| 汽车坐垫传感器 | 坐垫压力采集和显示 |
| 汽车靠背传感器 | 靠背压力采集和显示 |
| 支持 CH340 的采集设备 | 串口连接测试 |
| 至少 3 台不同 MAC 的设备 | 授权、MAC 读取、设备识别测试 |
| 无授权设备或模拟无授权设备 | 授权失败测试 |
| 可产生稳定压力的砝码或工装 | 清零、统计、重复性测试 |

### 3.2 软件环境

| 环境 | 要求 |
|---|---|
| 操作系统 | Windows |
| 网络 | 支持断网测试 |
| 权限 | 普通用户权限和管理员权限各测一次 |
| 存储路径 | 默认路径和自定义路径各测一次 |
| 数据样本 | 10 秒、10 分钟、30 分钟采集数据 |

## 4. 测试用例

### TC-A1: Windows 安装包安装和卸载

**Test Objective:** 验证软件可以通过 Windows 安装包完成安装、启动和卸载。

**Starting Conditions:**
- Windows 测试机未安装 JQ Tools。
- 测试机可断网运行。

**User Role:** 交付/售后人员。

**Test Steps:**
1. 双击安装包。
2. 按默认路径完成安装。
3. 检查桌面或开始菜单是否出现 JQ Tools 启动入口。
4. 断开网络并启动软件。
5. 采集或准备一条历史数据。
6. 卸载软件。
7. 检查卸载流程是否提示是否删除用户数据。

**Expected Outcomes:**
- 安装成功。
- 软件可离线启动。
- 卸载不会默认删除历史数据。
- 安装失败或卸载失败时有明确中文提示。

### TC-A2: 中文界面文案检查

**Test Objective:** 验证主流程界面和错误提示为中文，且文案不溢出。

**Starting Conditions:**
- JQ Tools 已安装。
- 显示缩放分别设置为 100% 和 125%。

**User Role:** 国内用户。

**Test Steps:**
1. 启动软件。
2. 查看主界面按钮和状态文案。
3. 进入设备连接、采集、回放、导出页面。
4. 触发一次连接失败和导出失败。
5. 检查按钮、弹窗、表格、状态栏文案显示。

**Expected Outcomes:**
- 主流程文案为中文。
- 不出现关键操作文案中英文混杂。
- 文案不超出容器边界。
- 错误提示能被普通用户理解。

### TC-B1: 一键连接成功

**Test Objective:** 验证用户点击“一键连接”后，软件能自动连接合法设备。

**Starting Conditions:**
- 插入 1 台合法 CH340 设备。
- 设备固件可响应握手协议。
- 软件未连接设备。

**User Role:** 测试工程师。

**Test Steps:**
1. 点击“一键连接”。
2. 观察按钮状态。
3. 等待连接完成。
4. 查看设备连接状态、MAC 地址、设备类型、波特率。

**Expected Outcomes:**
- 扫描期间按钮显示连接中或不可重复点击状态。
- 软件自动连接设备。
- 状态显示“已连接”。
- 显示 MAC 地址、设备类型、串口和实际波特率。
- 连接日志记录本次连接信息。

### TC-B2: 未插入设备时连接失败

**Test Objective:** 验证未插入设备时，软件给出明确失败提示。

**Starting Conditions:**
- 未插入任何采集设备。
- 软件未连接设备。

**User Role:** 测试工程师。

**Test Steps:**
1. 点击“一键连接”。
2. 等待扫描完成。
3. 查看界面提示。
4. 点击重试入口。

**Expected Outcomes:**
- 软件不会卡死。
- 显示“未检测到设备”或同等明确提示。
- 提供重试入口。
- 不允许进入采集状态。

### TC-B3: 多串口环境只连接协议正确设备

**Test Objective:** 验证多串口环境中，软件只连接 JQ Tools 支持的设备。

**Starting Conditions:**
- 同时插入一个目标采集设备和一个非目标串口设备。
- 两个设备都能被系统识别为串口。

**User Role:** 测试工程师。

**Test Steps:**
1. 点击“一键连接”。
2. 查看连接日志。
3. 查看最终连接设备信息。

**Expected Outcomes:**
- 软件逐个检测可用串口。
- 只连接协议响应正确的设备。
- 不错误连接非目标串口设备。
- 连接日志记录尝试过的串口和结果。

### TC-B4: 波特率自动尝试

**Test Objective:** 验证软件能尝试 921600、1000000、3000000 并记录成功波特率。

**Starting Conditions:**
- 准备支持不同波特率的设备或模拟设备。
- 软件未连接设备。

**User Role:** 测试工程师。

**Test Steps:**
1. 插入设备。
2. 点击“一键连接”。
3. 检查连接日志中的波特率尝试顺序。
4. 查看连接成功后的实际波特率字段。
5. 使用无法响应三种波特率的设备重复测试。

**Expected Outcomes:**
- 软件按预设顺序尝试波特率。
- 某一波特率成功后停止继续尝试。
- 实际波特率被记录。
- 全部失败时提示“波特率探测失败”。
- 探测过程有超时限制。

### TC-B5: MAC 地址读取和不可编辑

**Test Objective:** 验证软件能读取 MAC 地址，并禁止普通用户修改。

**Starting Conditions:**
- 插入合法设备。
- 设备可返回 MAC 地址。

**User Role:** 交付/售后人员。

**Test Steps:**
1. 连接设备。
2. 查看设备信息区。
3. 尝试编辑 MAC 地址。
4. 开始并停止一次短采集。
5. 查看历史记录或 CSV 中的 MAC 字段。

**Expected Outcomes:**
- MAC 地址显示正确。
- 普通用户不能手动修改 MAC 地址。
- 采集记录包含 MAC 地址。
- MAC 读取失败时有明确错误提示。

### TC-B6: 设备类型自动识别

**Test Objective:** 验证软件能根据设备信息自动识别设备类型。

**Starting Conditions:**
- 准备汽车坐垫、汽车靠背、工学椅坐垫等不同设备或模拟固件信息。
- 软件未连接设备。

**User Role:** 测试工程师。

**Test Steps:**
1. 插入汽车坐垫设备并连接。
2. 查看设备类型。
3. 断开后插入汽车靠背设备并连接。
4. 查看设备类型。
5. 使用无法识别类型的设备测试。

**Expected Outcomes:**
- 软件自动识别设备类型。
- 设备类型变化后视图模板同步变化。
- 无法识别时提示用户，不允许直接采集。
- 设备类型写入采集记录和 CSV。

### TC-B7: 坐垫和靠背同时显示

**Test Objective:** 验证汽车坐垫和汽车靠背同时连接或配套输入时，界面能同时显示两个区域。

**Starting Conditions:**
- 准备汽车坐垫和汽车靠背设备，或一个能同时返回坐垫/靠背数据的设备。
- 两个区域都有可变化压力数据。

**User Role:** 汽车座椅测试工程师。

**Test Steps:**
1. 连接设备。
2. 查看主界面是否出现坐垫和靠背区域。
3. 分别对坐垫和靠背施加压力。
4. 观察两个区域热力图变化。
5. 开始并停止采集。

**Expected Outcomes:**
- 坐垫和靠背区域同时显示。
- 两个区域名称清晰。
- 坐垫压力变化只影响坐垫区域。
- 靠背压力变化只影响靠背区域。
- 采集记录保存两个区域数据。

### TC-B8: 本地离线授权/固件校验通过

**Test Objective:** 验证合法设备在断网环境下可以通过授权或固件校验。

- 校验通过后允许采集。

### TC-E1: 实时 2D 热力图显示

**Test Objective:** 验证压力变化能实时反映到 2D 热力图。

**Starting Conditions:**
- 合法设备已连接。

- 热力图显示不阻塞采集。

### TC-E4: 鼠标滚轮缩放边界

**Test Objective:** 验证缩放范围限制在 10% 到 1000%。

**Starting Conditions:**
- 热力图显示。
- 鼠标可滚轮操作。

**User Role:** 测试工程师。

**Test Steps:**

2. 继续向下滚动。
3. 连续向上滚动直到最大缩放。

## 6. 验收清单

### 6.1 版本准入

| 检查项 | 通过标准 |
|---|---|
| 安装包 | Windows 上可安装、启动、卸载 |
| 离线可用 | 断网情况下可启动并连接授权设备 |
| 中文界面 | 主流程无关键英文残留 |
| 主流程闭环 | 连接、采集、停止、回放、导出完整可用 |
| 数据完整性 | 采集记录包含设备、时间、类型、清零、采样率 |
| CSV | 有格式版本，字段固定，压力矩阵为一维数组表 |
| 异常恢复 | 强退后可恢复未完成采集 |
| 性能 | 记录 FPS 和长时间采集表现 |

### 6.2 阻塞级缺陷

出现以下问题，不建议通过 V2.0 验收：

1. 合法设备无法稳定连接。
2. MAC 地址读取错误或缺失。
3. 授权失败设备仍可采集。
4. 采集数据无法保存。
5. 停止采集后历史记录丢失。
6. CSV 导出数据缺失或格式无法解析。
7. 清零逻辑只改变按钮状态，不改变数据基线。
8. 框选导致原始全量数据丢失。
9. 异常退出后缓存损坏且无法恢复。
10. 热力图无法实时反映压力变化。

### 6.3 非阻塞但必须记录

以下问题可不阻塞 V2.0，但必须记录并排期：

1. 60fps 未完全达到。
2. 高密度数字矩阵显示拥挤。
3. 长时间采集文件体积过大。
4. 缩放到 1000% 时操作变慢。
5. 历史搜索性能不足。
6. CSV 文件过大导致 Excel 打开慢。
7. 未补齐受压面积阈值配置。
8. 未补齐校准和误差报告。
