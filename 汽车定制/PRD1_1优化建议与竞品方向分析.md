# PRD1.1 优化建议与竞品方向分析

文档日期：2026-05-19  
分析对象：`D:\pm-skill\汽车定制\PRD1_1.md`  
产品名称：压力感知软件 JQ Tools  
使用技能：`create-prd`、`competitor-analysis`、`brainstorm-ideas-existing`  
分析目标：检查当前 PRD 可优化点，补充竞品对标，明确后续产品方向。

## 1. 总体判断

当前 `PRD1_1.md` 已经不是一个普通 PRD，更接近“实现对齐版产品规格 + 数据口径文档 + 验收规则合集”。它的优势是细、可落研发、异常处理比较完整；短板是战略层、竞品层、版本优先级和业务价值表达还不够清楚。

| 维度 | 当前状态 | 判断 |
|---|---|---|
| 研发可执行性 | 连接、数据处理、采集、回放、CSV、框选、对比都有业务规则 | 较强 |
| 数据可信度 | rawFrame/rawData/displayData/calculatedData/exportedData 已拆分 | 较强 |
| 异常处理 | 连接锁、坏帧、强退恢复、导入校验已有定义 | 较强 |
| PRD 结构 | 文档太长，PRD、实现说明、补丁说明混在一起 | 需要优化 |
| 竞品差异 | 提到对标 XSENSOR，但没有系统对比表和差异化路线 | 需要补 |
| 业务指标 | 缺少成功指标、客户价值指标、发布后衡量方式 | 需要补 |
| 报告交付 | 只把 PDF/Word 报告放到 V2.2，当前闭环偏弱 | 需要提前规划 |
| 舒适性分析 | 当前偏压力数据工具，离“汽车座椅分析系统”还有一步 | 需要加强 |

结论：产品方向是对的，但需要从“压力数据采集和可视化工具”升级为“汽车座椅压力分析与交付系统”。

### 4.4 JQ Tools 当前能打的点

| 差异点 | 为什么能打 |
|---|---|
| 纤维压力传感器绑定 | 如果硬件确实有成本、柔性、厚度或可定制优势，这是核心差异 |
| 国产替代和本地交付 | 对展厅、门店、国内研发客户更友好 |
| 离线使用 | 对内网、展厅、招标和科研客户有价值 |
| 连接和设备授权闭环 | PRD 对连接、MAC、授权、重连、异常恢复定义很细 |
| 框选独立分析 | 框选不耦合采集，实时/回放/对比都能使用，这是一个好方向 |
| 数据分层口径 | rawData、displayData、calculatedData、exportedData 已经有工程基础 |
| 座椅坐垫/靠背独立方向 | 适合真实座椅安装和传感器摆放偏差 |

### 4.5 JQ Tools 还弱的点

| 弱项 | 影响 |
|---|---|
| 校准和精度体系不够 | 对科研、研发和招标客户会被质疑数据可信度 |
| 报告能力偏后 | 数据对比后无法形成客户交付物，价值闭环弱 |
| 舒适性/座椅工程指标不够 | 目前只有压力统计，缺少“座椅分析系统”的专业感 |
| 竞品对比页没有写入 PRD | 产品团队容易只做功能，不做差异化 |
| 数据质量可视化不足 | 现场问题排查依赖日志，不够产品化 |
| 外部分析接口不够 | 科研客户可能需要 MATLAB/Python/Excel 友好数据 |
| 事件标记/阶段分析缺失 | 不能很好分析坐姿调整、靠背角度变化、工况变化 |

## 5. 后续该做的产品方向

### 5.1 V2.0 必须先收口的方向

| 方向 | 目标 | 原因 |
|---|---|---|
| 连接稳定性 | 一键连接、重连、断开、异常恢复不死锁 | 入口不稳，后面都不可用 |
| 数据可信度 | rawData、方向、置零、噪点、CSV 字段可追溯 | 对科研和研发客户是底线 |
| 框选闭环 | 实时、回放、对比都能框选，且不影响采集 | 这是当前最像“产品亮点”的能力 |
| A/B 数据对比 | 三图、指标差值、变化率、同记录时间点对比 | 对标竞品的基础分析能力 |
| 发布验收 | 把 P0 清单变成可执行测试用例和负责人 | 防止 PRD 写完但发布不可控 |

### 5.2 V2.1 建议做成差异化的方向

| 方向 | 具体能力 | 价值 |
|---|---|---|
| 标准分析报告 | reportData、报告预览、PDF 导出、CSV 附件 | 从“看数据”变成“交付结果” |
| 座椅工程指标 | 左右均衡、前后均衡、重心迁移、峰值持续时间、压力集中指数 | 更像汽车座椅分析系统 |
| 时间段对比 | 同一记录内 A 时间段 vs B 时间段 | 适合坐姿调整前后稳定阶段对比 |
| 框选模板增强 | 常用区域模板、坐垫/靠背区域模板、模板导入导出 | 提高区域分析效率 |
| 数据质量面板 | 帧率、坏帧率、丢帧、设备状态、采样率波动 | 提高现场排查能力 |
| 外部分析导出 | Python/MATLAB/Excel 友好结构 | 对科研客户更有吸引力 |

### 5.3 V2.2 可以做护城河的方向

| 方向 | 具体能力 | 价值 |
|---|---|---|
| 校准与精度体系 | 传感器校准、漂移检测、重复性测试、校准报告 | 解决竞品最强的“可信度”壁垒 |
| 舒适性分析模型 | 坐垫/靠背分区评分、压力风险提示、支撑不足提示 | 从工具升级为分析系统 |
| 视频/事件同步 | 标记“坐姿调整”“靠背角度变化”“上车/下车”等事件 | 适合汽车座椅研发实验 |
| 客户报告模板 | 研发模板、科研模板、展厅模板、品牌化模板 | 提高交付和销售价值 |
| 多传感器同步 | 坐垫+靠背+脚部或其他区域同步采集 | 提升系统能力上限 |

## 7. 建议补充的成功指标

| 指标 | 建议目标 |
|---|---|
| 一键连接成功率 | 合法设备首次连接成功率 >= 95% |
| 连接失败恢复 | 连接失败后 1 次重连可恢复，不需要重启软件 |
| 实时显示延迟 | 正常采集下前端显示延迟 <= 200ms，具体需研发实测确认 |
| 长采集稳定性 | 连续采集 30 分钟无明显 UI 卡死、无数据写入中断 |
| 坏帧容错 | 单个坏帧不影响后续正常帧；连续坏帧有明确提示 |
| CSV 可用性 | 导出 CSV 可重新导入、回放、参与对比 |
| 数据对比准确性 | A/B 差值、变化率、A=0 处理符合 PRD |
| 框选可信度 | 框选不影响全量采集；回放/对比可重新框选 |
| 报告可交付性 | V2.1 PDF 报告包含图、表、摘要、数据口径和附件 |
