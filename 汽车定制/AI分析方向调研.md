# AI 分析方向调研

文档日期：2026-05-19  
产品名称：压力感知软件 JQ Tools  
关联 PRD：`D:\pm-skill\汽车定制\PRD1_1.md`  
使用技能：`brainstorm-ideas-existing`、`competitor-analysis`  
调研目标：明确下一步 AI 分析应该往哪个方向做，以及 AI 应该分析哪些数据。

## 1. 结论先行

JQ Tools 下一步做 AI，不建议一开始就做“舒适度自动评分”或“坐姿诊断结论”。原因是这类能力需要大量标注数据、真实用户主观反馈、座椅工程标准和校准体系，短期容易做成不可信的黑盒。

推荐先做三层：

| 优先级 | AI 方向 | 为什么先做 |
|---|---|---|
| 第一优先级 | AI 数据质量诊断 | 不依赖大量人工标签，能直接提升现场稳定性和数据可信度 |
| 第二优先级 | AI 对比分析摘要和报告 | 基于现有统计、框选、A/B 差值生成解释，最快形成客户交付价值 |
| 第三优先级 | 座椅压力模式识别 | 从压力集中、左右不均、前后不均、重心迁移等工程指标切入，逐步走向座椅分析系统 |

中长期再做：

| 阶段 | AI 方向 |
|---|---|
| V2.1 | 数据质量评分、AI 对比摘要、报告文案生成 |
| V2.2 | 坐姿/工况识别、压力模式聚类、相似测试检索 |
| V3.0 | 舒适性预测、座椅设计建议、自动优化建议 |

## 3. AI 应该分析哪些数据

### 3.1 当前 PRD 已具备的数据

| 数据类型 | 字段/来源 | AI 可分析内容 |
|---|---|---|
| 设备原始帧 | rawFrame、rawPointArray、received_at | 帧异常、断帧、通信质量、采样率稳定性 |
| 业务原始矩阵 | rawData.matrix、width、height、matrix_key、direction | 压力分布模式、峰值区域、坐标变化、区域形态 |
| 后端处理矩阵 | backendProcessedData | 噪点消除前后差异、数据质量变化 |
| 显示数据 | displayData | 用户实际看到的压力图，适合报告截图和图像解释 |
| 统计数据 | calculatedData | 平均、最大、最小、总和、面积、点数、重心、框选指标 |
| 采集历史 | storedData.frames、metadata | 时间序列、长期趋势、动态变化、异常段定位 |
| 导出数据 | CSV 字段 | 外部训练、复核、批量分析 |
| 框选数据 | selection 坐标、名称、颜色、区域统计 | 局部区域 AI 分析，例如坐垫左前/右后/靠背腰部 |
| 数据对比结果 | A/B/差值图、变化率 | AI 摘要、差异解释、报告结论 |
| 数据质量字段 | bad_frame_count、frame_interval_ms、sampleRateHz | 数据质量评分、异常诊断 |

### 3.2 AI 还需要补充采集的数据

| 数据 | 为什么需要 | 建议优先级 |
|---|---|---|
| 测试对象信息 | 例如座椅型号、坐垫/靠背、材料、软硬度、角度 | P0/P1 |
| 被测人员信息 | 身高、体重、坐姿类型，注意隐私和授权 | P1 |
| 工况标签 | 静坐、左倾、右倾、前倾、后靠、上下车、调整坐姿 | P1 |
| 事件标记 | 例如“调整靠背角度”“更换坐垫材料”“用户移动” | P1 |
| 主观评价 | 舒适度评分、压迫感评分、左右支撑感评分 | P2，做舒适性预测前必须有 |
| 视频或图片 | 用于校验坐姿和事件，但不是必须 | P2 |
| 校准信息 | 传感器校准时间、压力范围、漂移情况 | P1/P2 |
| 设计参数 | 座椅结构、泡棉硬度、支撑件位置 | V3.0，用于设计建议 |

## 4. 推荐 AI 方向

### 4.1 方向一：AI 数据质量诊断

| 项目 | 内容 |
|---|---|
| 目标 | 自动判断本次采集数据是否可信 |
| 输入数据 | rawFrame、rawData、sampleRateHz、frame_interval_ms、bad_frame_count、matrix size、metadata |
| 输出结果 | 数据质量评分、异常原因、建议处理方式 |
| 示例结论 | “本次数据质量较低：第 120-145 帧存在连续坏帧；建议重新连接后采集。” |
| 技术方式 | 规则 + 异常检测模型；先不需要大量标签 |
| 版本建议 | V2.1 P0 |

可识别的问题：

| 问题 | 判断依据 |
|---|---|
| 采样率异常 | sampleRateHz 为 0、跳变过大、帧间隔不稳定 |
| 坏帧过多 | bad_frame_count、坏帧率、连续坏帧 |
| 矩阵异常 | 长度不匹配、行列错位、异常全 0 |
| 传感器疑似失效 | 某一行/列长期为 0、某些点长期饱和 |
| 数据噪声高 | 相邻帧无真实动作但数值剧烈抖动 |
| 方向口径风险 | A/B 数据方向、置零、噪点消除状态不一致 |

为什么最值得先做：它直接服务交付、售后、QA 和研发，也能为后续 AI 模型建立可信数据基础。

### 4.2 方向二：AI 对比分析摘要

| 项目 | 内容 |
|---|---|
| 目标 | 把 A/B 对比、同记录时间点对比、框选对比转成可读结论 |
| 输入数据 | A/B 指标、差值图、变化率、框选指标、重心变化、面积变化 |
| 输出结果 | 结构化摘要、风险提示、报告文案 |
| 示例结论 | “B 相比 A，坐垫左后区域压力总和上升 18.2%，压力重心向左后移动，说明该区域承压增加。” |
| 技术方式 | 规则引擎 + LLM 文案生成；先不做黑盒判断 |
| 版本建议 | V2.1 P0/P1 |

推荐输出字段：

| 输出 | 说明 |
|---|---|
| 最大变化区域 | 哪个框选/区域变化最大 |
| 压力集中变化 | 最大压强、压力总和、受压面积共同判断 |
| 重心迁移 | 重心方向和距离 |
| 左右/前后均衡变化 | 是否更偏左、偏右、偏前、偏后 |
| 框选局部结论 | 每个框选区域的差异摘要 |
| 数据口径提醒 | 采样率、置零、方向、噪点消除是否一致 |

注意：这里的 AI 应该说“数据变化”，不要直接说“舒适/不舒适”“设计合格/不合格”。

### 4.3 方向三：座椅压力模式识别

| 项目 | 内容 |
|---|---|
| 目标 | 从压力图中识别典型座椅压力模式 |
| 输入数据 | rawData 矩阵、框选区域、时间序列、重心轨迹 |
| 输出结果 | 压力模式标签、区域说明、变化趋势 |
| 示例标签 | 左右不均、前后不均、局部高压、接触面积不足、重心后移 |
| 技术方式 | 先用工程规则，后续用聚类/分类模型 |
| 版本建议 | V2.1 P1 / V2.2 |

可先做的模式：

| 模式 | 判断数据 |
|---|---|
| 局部压力集中 | 最大值、局部 pressure_sum、局部面积小但压力高 |
| 左右不均衡 | 左半区压力总和 vs 右半区压力总和 |
| 前后不均衡 | 前区压力总和 vs 后区压力总和 |
| 重心偏移 | center_of_pressure 坐标 |
| 支撑不足 | 接触面积小、有效点数少、局部悬空 |
| 动态不稳定 | 重心轨迹波动大、压力总和曲线抖动大 |

### 4.4 方向四：坐姿/工况识别

| 项目 | 内容 |
|---|---|
| 目标 | 自动识别用户坐姿或测试阶段 |
| 输入数据 | 连续压力帧、重心轨迹、接触面积变化、事件标记 |
| 输出结果 | 静坐、左倾、右倾、前倾、后靠、上下车、坐姿调整 |
| 技术方式 | 有监督分类，需要标注数据 |
| 版本建议 | V2.2 |

这类能力有吸引力，但前置条件比较高：

| 前置条件 | 说明 |
|---|---|
| 标注数据 | 每段数据需要知道真实坐姿或工况 |
| 多人数据 | 不同身高体重、不同坐姿习惯 |
| 多座椅数据 | 不同座椅结构、材料、硬度 |
| 验证集 | 需要验证模型不能只记住某个测试人 |

### 4.5 方向五：舒适性预测

| 项目 | 内容 |
|---|---|
| 目标 | 根据压力数据预测用户舒适度或座椅支撑质量 |
| 输入数据 | 压力矩阵、统计指标、分区指标、用户主观评分、座椅参数 |
| 输出结果 | 舒适度趋势、压迫风险、支撑不足提示 |
| 技术方式 | 监督学习，需要大量主观评分和工程验证 |
| 版本建议 | V3.0 |

为什么不建议先做：

| 原因 | 说明 |
|---|---|
| 标签难 | 没有主观舒适度评分，模型无法学 |
| 个体差异大 | 身高、体重、体型、习惯都会影响舒适评价 |
| 风险高 | 容易输出无法解释的“舒适/不舒适” |
| 竞品强 | 高端竞品有长期数据和校准体系 |

建议先做“舒适性相关指标”，不要先做“舒适性最终评分”。

## 5. AI 数据特征设计

### 5.1 单帧特征

| 特征 | 来源 | 用途 |
|---|---|---|
| 平均压强 | calculatedData | 整体承压水平 |
| 最大压强 | calculatedData | 局部高压判断 |
| 最大值坐标 | calculatedData / CSV | 压力集中位置 |
| 非零最小压强 | calculatedData | 接触稳定性 |
| 压力总和 | calculatedData | 总承载变化 |
| 受压面积 | calculatedData | 接触面积 |
| 有效点数 | calculatedData | 接触范围 |
| 压力重心 | calculatedData | 坐姿偏移 |
| 左/右压力比 | 分区计算 | 左右均衡 |
| 前/后压力比 | 分区计算 | 前后均衡 |
| 高压点占比 | rawData / 阈值 | 压力集中 |

### 5.2 时间序列特征

| 特征 | 用途 |
|---|---|
| 压力总和变化率 | 判断上下车、移动、坐姿调整 |

| 重心轨迹长度 | 判断坐姿稳定性 |
| 重心移动方向 | 判断左倾、右倾、前倾、后靠 |
| 最大压强持续时间 | 判断持续压迫区域 |
| 高压区域持续时间 | 判断风险区域 |
| 帧间差分 | 判断动作变化或噪声 |
| 稳定段识别 | 用于时间段对比和报告 |

### 5.3 框选区域特征

| 特征 | 用途 |

| selection 占全量比例 | 局部区域贡献度 |

### 5.4 数据质量特征

| 特征 | 用途 |

## 6. AI 产品能力分层

| 层级 | 能力 | 是否需要模型 | 价值 |
|---|---|---|---|
| L1 规则智能 | 数据质量、阈值判断、对比摘要 | 不一定 | 快速上线，可信 |
| L2 统计学习 | 异常检测、模式聚类、稳定段识别 | 需要少量模型 | 提升分析深度 |
| L3 有监督分类 | 坐姿识别、工况识别 | 需要标注 | 形成差异化 |
| L4 预测模型 | 舒适度预测、设计建议 | 需要大量数据 | 长期护城河 |
| L5 生成式报告 | 自动生成客户报告、解释图表 | 可用 LLM | 提升交付效率 |

建议从 L1 + L5 的“可解释分析 + 报告”开始，不要从 L4 开始。

## 7. 推荐版本路线

### 7.1 V2.1：AI 分析最小可行版本

| 功能 | 输入 | 输出 |
|---|---|---|
| 数据质量评分 | 帧率、坏帧、矩阵、异常点 | 优/中/差 + 原因 |
| AI 对比摘要 | A/B 指标、差值、框选 | 3-5 条数据变化结论 |
| 框选区域解释 | selection 指标 | 每个区域一句解释 |
| 稳定段提示 | 时间序列指标 | 建议用于对比的稳定时间段 |
| 报告文案生成 | reportData | 报告摘要和数据口径说明 |

V2.1 的 AI 不需要训练大模型，核心是“规则 + 统计 + 可编辑文案”。

### 7.2 V2.2：AI 模式识别

| 功能 | 输入 | 输出 |
|---|---|---|
| 压力模式聚类 | 历史 rawData 和统计特征 | 常见压力模式分类 |
| 异常样本检索 | 新采集数据 + 历史库 | 相似历史案例 |
| 坐姿/工况识别 | 标注过的时间段 | 左倾、右倾、前倾、后靠等 |
| 事件自动检测 | 压力总和、面积、重心突变 | 坐姿调整、上下车、异常移动 |

### 7.3 V3.0：舒适性和设计建议

| 功能 | 输入 | 输出 |
|---|---|---|
| 舒适度预测 | 压力特征 + 主观评分 | 舒适度趋势或评分 |
| 座椅设计建议 | 压力模式 + 座椅参数 | 支撑区域优化建议 |
| 多人群模型 | 多用户、多座椅、多工况 | 人群级分析 |
| 自动优化对比 | 多方案 A/B/C | 推荐更优方案 |

## 8. 不建议优先做的 AI

| 不建议先做 | 原因 |
|---|---|
| 自动舒适度评分 | 没有主观评分和工程标准，可信度不足 |
| 医疗/健康风险诊断 | 风险高，合规和责任边界复杂 |
| 完全黑盒神经网络判断 | 难解释，客户和研发不容易接受 |
| 直接给设计修改建议 | 需要座椅结构参数和大量验证 |
| 云端大数据 AI | 当前产品强调本地离线，云端会增加交付阻力 |

|---|---|---|---|
| DEC-AI-01 | AI 是否进入 V2.1 | 进入，但只做数据质量和对比摘要 | 控制研发风险 |
| DEC-AI-02 | 是否做舒适度评分 | 暂不做，先做舒适性相关指标 | 避免不可信结论 |
| DEC-AI-03 | AI 是否允许输出判断性结论 | 不允许，只输出数据变化和风险提示 | 降低责任风险 |
| DEC-AI-04 | 是否采集主观评分 | 开始采集，但不立即用于模型 | 为 V3.0 做准备 |
| DEC-AI-05 | 是否采集被测人信息 | 采匿名 ID、身高体重需授权 | 涉及隐私 |
| DEC-AI-06 | AI 报告文案是否可编辑 | 可编辑并保存 | 适合客户交付 |
| DEC-AI-07 | AI 是否本地运行 | V2.1 本地规则和统计；后续再评估模型 | 符合离线交付 |
