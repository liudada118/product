## 1. 概要

AI 分析功能用于把 JQ Tools 已采集到的压力矩阵、统计指标、框选区域、数据对比结果和采集元数据，转化为更容易理解的分析结论、数据质量提示和报告文案。

本功能第一阶段不做黑盒“舒适度评分”，也不输出“合格/不合格”等强判断结论。V2.1 优先做可解释、可追溯、可编辑的辅助分析能力，帮助研发、展厅演示、科研测试和客户交付更快理解压力数据。

## 3. 背景

JQ Tools 当前已具备设备连接、实时压力展示、2D 数字、3D 点图、框选、量尺、统计图表、数据采集、历史回放、CSV 导入导出和数据对比能力。

现阶段的问题是：软件已经能展示和记录大量压力数据，但用户仍需要人工判断“这组数据说明什么”“A/B 哪个区域变化最大”“本次采集是否可信”“报告应该怎么写”。如果只停留在图像和表格展示，产品更像数据工具；如果能给出可解释的分析摘要，产品才能向“汽车座椅压力分析系统”升级。

竞品方向也说明，高端压力映射软件通常不只做热力图展示，还会强调数据分析、测试结果对比、异常识别、报告和工作流优化。JQ Tools 的 AI 功能应先围绕已有数据闭环做增强，而不是一开始就做大模型或复杂预测。

## 8. AI 分析总流程

### 8.1 分析链路

| 步骤 | 输入 | 处理 | 输出 |
|---|---|---|---|
| 1. 数据读取 | rawData、backendProcessedData、calculatedData、selection、metadata | 校验数据完整性和口径 | analysisInput |
| 2. 数据质量诊断 | 帧率、坏帧、矩阵、异常点、设备信息 | 规则判断和异常统计 | qualityResult |
| 3. 特征计算 | 全量矩阵、框选区域、时间序列 | 计算压力、面积、重心、均衡、变化率 | featureSet |
| 4. 对比分析 | A/B 指标、时间点指标、差值图、变化率 | 找最大变化、方向变化、局部变化 | compareInsight |
| 5. 结论生成 | qualityResult、featureSet、compareInsight | 模板化文案或 LLM 生成 | aiSummary |
| 6. 人工确认 | aiSummary | 用户编辑、删除、保存 | finalReportText |
| 7. 存档追溯 | 指标、帧范围、区域 ID、结论 | 写入历史记录或报告 | analysisRecord |

### 8.2 AI 分析原则

| 原则 | 说明 |
|---|---|
| 可解释 | 每条结论必须能看到来源指标 |
| 可追溯 | 结论必须关联帧范围、时间点、框选区域或对比记录 |
| 可编辑 | 报告文案生成后用户可修改 |
| 可关闭 | 用户可关闭 AI 辅助分析 |
| 不污染原始数据 | AI 分析不修改 rawData，不修改采集文件 |
| 不替代人工判断 | AI 只提示数据变化和风险，不做最终业务判定 |

## 10. AI 功能详细需求

### 10.1 AI 数据质量诊断

#### 10.1.1 功能说明

系统在实时采集、采集结束、历史回放、数据导入和数据对比前，对数据进行质量诊断，输出数据质量等级、异常原因和建议动作。

#### 10.1.2 输入

| 输入 | 说明 |
|---|---|
| rawFrame | 原始帧和帧长度 |
| rawData.matrix | 业务原始矩阵 |
| sampleRateHz | 后端计算采样率 |
| frame_interval_ms | 帧间隔 |
| bad_frame_count | 坏帧数量 |
| consecutive_bad_frames | 连续坏帧数量 |
| zero_matrix_duration | 全 0 或近似全 0 持续时长 |
| saturated_point_ratio | 饱和点比例 |
| dead_point_ratio | 长期无变化点比例 |
| metadata | 设备类型、方向、置零、噪点消除、采集时间 |

#### 10.1.3 输出

| 输出 | 示例 |
|---|---|
| quality_level | 优 / 中 / 差 |
| quality_score | 0-100 |
| issue_list | 采样率不稳定、连续坏帧、疑似行列异常 |
| affected_frame_range | 第 120-145 帧 |
| recommendation | 建议重新连接设备后重新采集 |
| can_compare | true / false |
| can_report | true / false |

#### 10.1.4 规则

| 规则 | 判断 | 结果 |
|---|---|---|
| 坏帧率过高 | bad_frame_count / total_frames 超过阈值 | 质量降级 |
| 连续坏帧 | consecutive_bad_frames 超过阈值 | 标记异常段 |
| 采样率跳变 | frame_interval_ms 波动过大 | 提示采样不稳定 |
| 全 0 异常 | 非空测试中矩阵长期全 0 | 提示设备或接触异常 |
| 饱和点过多 | saturated_point_ratio 超过阈值 | 提示可能超量程 |
| 死点过多 | dead_point_ratio 超过阈值 | 提示传感点异常 |
| 口径缺失 | 缺少 direction、zero_state、device_type | 不建议用于对比报告 |

#### 10.1.5 界面展示

| 位置 | 展示 |
|---|---|
| 实时页顶部 | 数据质量状态：优 / 中 / 差 |
| 采集结束弹窗 | 本次采集质量摘要 |
| 历史详情页 | 质量评分、异常帧范围、建议 |
| 数据对比页 | 如果 A/B 质量差异大，进入对比前提示 |
| 报告页 | 自动写入数据质量说明 |

### 10.2 AI 对比分析摘要

#### 10.2.1 功能说明

系统基于数据对比结果，自动生成 A/B 对比摘要。对比对象包括两条历史记录、同一记录不同时间点、同一记录不同稳定段、框选区域对比。

#### 10.2.2 支持的对比类型

| 类型 | 说明 | V2.1 是否支持 |
|---|---|---|
| 历史 A/B 对比 | 两条历史记录对比 | 支持 |
| 同记录时间点对比 | 同一记录不同帧或时间点对比 | 支持 |
| 稳定段对比 | 同一记录或不同记录的稳定时间段对比 | P1 |
| 框选区域对比 | 同一个框选作用于 A/B 和差值图 | 支持 |
| 导入 CSV 对比 | 外部 CSV 与历史数据对比 | P1 |

#### 10.2.3 输出内容

| 内容 | 说明 |
|---|---|
| 总体变化摘要 | B 相比 A 的压力总和、面积、重心变化 |
| 最大变化区域 | 变化最大的框选区域或系统分区 |
| 局部高压变化 | 最大压强、最大值坐标、高压点比例变化 |
| 均衡性变化 | 左右、前后压力分布是否更均衡 |
| 重心迁移 | 重心移动方向和距离 |
| 数据口径提醒 | 方向、置零、噪点消除、采样率是否一致 |

#### 10.2.4 文案规则

| 情况 | 文案规则 |
|---|---|
| 指标变化小 | 使用“变化不明显” |
| 指标变化中等 | 使用“有所上升/下降” |
| 指标变化大 | 使用“明显上升/下降” |
| A 为 0 | 变化率显示 N/A，只显示绝对差值 |
| 质量较差 | 结论前加“受数据质量影响，本分析仅供参考” |
| 口径不一致 | 明确提示“不建议直接下结论” |

#### 10.2.5 示例输出

| 类型 | 示例 |
|---|---|
| 总体摘要 | B 相比 A，压力总和上升 12.6%，受压面积增加 8.4%，整体承压范围扩大。 |
| 区域摘要 | 坐垫左后区域压力总和上升 18.2%，是本次对比中变化最大的区域。 |
| 重心摘要 | 压力重心向左后移动，说明承压重心发生偏移。 |
| 口径提醒 | A/B 的噪点消除状态不一致，建议统一处理口径后再生成正式报告。 |

### 10.3 框选区域 AI 解释

#### 10.3.1 功能说明

框选是 JQ Tools 的核心分析能力。AI 分析需要支持实时框选、回放框选和数据对比框选，对每个框选区域输出局部解释。

#### 10.3.2 输入

| 输入 | 说明 |
|---|---|
| selection_id | 框选区域 ID |
| selection_name | 框选名称 |
| row_range / column_range | 框选坐标 |
| selection stats | 平均、最大、总和、面积、重心、有效点数 |
| selection time series | 框选区域随时间变化 |
| compare selection stats | A/B/差值/变化率 |

#### 10.3.3 输出

| 输出 | 说明 |
|---|---|
| 区域压力水平 | 当前区域压力偏高、偏低或变化不明显 |
| 区域变化趋势 | 上升、下降、波动、稳定 |
| 区域贡献度 | 该区域占整体压力总和比例 |
| 区域风险提示 | 局部高压、接触不足、波动较大 |
| 对比结论 | A/B 下该区域变化 |

#### 10.3.4 业务规则

| 规则 | 说明 |
|---|---|
| 框选不影响采集 | AI 只能基于框选派生数据分析，不改变采集全量矩阵 |
| 框选可用于回放 | 历史回放中创建新框选后，可重新生成 AI 区域解释 |
| 框选可用于对比 | 数据对比中，一个框选区域同时作用于 A/B 和差值图 |
| 多框选独立解释 | 最多 4 个框选区域分别生成解释，不默认合并 |
| 框选越界不分析 | 框选区域超出有效矩阵时不生成 AI 结论 |

### 10.4 稳定段提示

#### 10.4.1 功能说明

系统根据压力总和、受压面积、压力重心和最大压强的变化情况，识别相对稳定的时间段，帮助用户选择更适合对比和报告的片段。

#### 10.4.2 判断依据

| 指标 | 说明 |
|---|---|
| 压力总和波动 | 波动越小越稳定 |
| 受压面积波动 | 面积变化越小越稳定 |
| 重心轨迹长度 | 重心移动越小越稳定 |
| 最大压强波动 | 峰值变化越小越稳定 |
| 坏帧率 | 坏帧越少越适合分析 |

#### 10.4.3 输出

| 输出 | 说明 |
|---|---|
| stable_segment_list | 推荐稳定段列表 |
| start_frame / end_frame | 起止帧 |
| start_time / end_time | 起止时间 |
| stability_score | 稳定分数 |
| reason | 推荐原因 |

### 10.5 AI 报告文案生成

#### 10.5.1 功能说明

系统基于数据质量、统计指标、对比结果和框选解释，生成报告摘要。报告文案必须可编辑、可保存、可复制。

#### 10.5.2 报告结构

| 模块 | 内容 |
|---|---|
| 测试概况 | 测试对象、设备类型、采集时间、采样率 |
| 数据质量 | 质量等级、异常说明、是否建议用于报告 |
| 关键指标 | 平均压强、最大压强、压力总和、受压面积、重心 |
| 对比结论 | A/B 或时间点对比摘要 |
| 框选区域分析 | 每个框选区域的局部结论 |

| 注意事项 | 数据不足、质量较差、口径不一致等提示 |

#### 10.5.3 生成方式

## 13. 技术实现建议

### 13.1 V2.1 推荐架构

| 层级 | 责任 |
|---|---|
| 数据准备层 | 从历史、回放、对比、框选中组织统一 analysisInput |
| 特征计算层 | 计算全量、框选、时间序列和对比特征 |
| 规则分析层 | 基于阈值和规则生成结构化 insight |
| 文案生成层 | 用模板生成自然语言摘要 |
| 存储层 | 保存 analysisResult、用户编辑后的报告文案 |
| UI 层 | 展示 AI 面板、状态、追溯和编辑 |

### 13.2 V2.1 技术路线

| 能力 | 推荐实现 |
|---|---|
| 数据质量评分 | 规则引擎 + 阈值配置 |
| 对比摘要 | 指标差值排序 + 文案模板 |
| 框选解释 | selection 指标计算 + 局部规则 |
| 稳定段提示 | 滑动窗口 + 波动率计算 |
| 报告文案 | 本地模板生成，后续可接 LLM |
| 追溯 | insight 绑定 metric_refs、frame_range、selection_id |

### 13.3 不建议的技术路线

| 不建议 | 原因 |
|---|---|
| V2.1 直接训练舒适度模型 | 缺少标签和验证数据 |
| V2.1 强依赖云端大模型 | 离线交付和数据安全风险 |
| 只让 LLM 看 CSV 自由总结 | 不可控、不可追溯、容易编造 |
| AI 结果直接覆盖业务指标 | 会破坏数据可信度 |

## 14. 数据安全和边界

| 规则 | 说明 |
|---|---|
| 默认本地分析 | V2.1 默认在本地完成规则和统计分析 |
| 不上传原始压力数据 | 未经授权不得上传客户数据 |
| 被测人信息匿名化 | subject_id 不直接使用姓名、手机号等敏感信息 |
| 报告可编辑 | AI 文案进入正式交付前必须允许人工确认 |
| 明确辅助性质 | 界面说明 AI 分析仅作为辅助，不作为医学或最终工程判定 |

## 15. 验收标准

### 15.1 P0 验收

| 编号 | 验收项 | 标准 |
|---|---|---|
| AI-AC-01 | 数据质量评分 | 采集结束后能生成质量等级、评分、异常原因 |
| AI-AC-02 | 异常帧追溯 | 坏帧、连续坏帧、采样异常能定位到帧范围 |
| AI-AC-03 | 对比摘要 | A/B 对比可生成 3-5 条结构化摘要 |
| AI-AC-04 | 框选解释 | 有框选时每个框选区域可生成局部解释 |
| AI-AC-05 | 口径提醒 | A/B 方向、置零、噪点消除、采样率不一致时必须提示 |
| AI-AC-06 | 不修改原始数据 | AI 分析不修改 rawData、storedData、CSV 原始字段 |
| AI-AC-07 | 结论可追溯 | 每条 AI 结论可查看对应指标、帧范围或区域 |
| AI-AC-08 | 高风险结论拦截 | 不输出医学诊断、绝对舒适评分、合格判定 |

### 15.2 P1 验收

| 编号 | 验收项 | 标准 |
|---|---|---|
| AI-AC-09 | 稳定段提示 | 历史回放中能提示至少一个稳定段或说明无法识别 |
| AI-AC-10 | 报告文案 | 可生成、编辑、保存、复制报告摘要 |
| AI-AC-11 | 低质量提示 | 数据质量差时报告中自动附带风险说明 |
| AI-AC-12 | 重新生成 | 用户修改框选或对比对象后可重新生成 AI 分析 |
