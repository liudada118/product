# PRD: JQ Tools 汽车座椅压力感知系统

## 1. Summary

本 PRD 定义 JQ Tools 汽车座椅压力感知系统的产品目标、用户场景、核心功能、技术要求和版本规划。该系统由 JQ Tools 软件和适配的纤维压力传感器组成，用于实时采集、显示、分析和导出汽车座椅压力数据。

产品目标是对标 XSENSOR、Tekscan、novel pliance 等国外压力成像系统，在汽车座椅研发、展厅演示、门店体验和科研招标场景中提供国产替代方案，并在本地化、成本、交付、定制和实时展示体验上形成优势。

## 3. Background

### 3.1 Initiative Context

汽车座椅研发、座椅材料验证、展厅展示和科研测试场景中，客户需要看到人体坐在座椅上时的实时压力分布。传统展示方式只能依靠主观体验或静态说明，无法直观证明座椅舒适性、支撑性和材料差异。

JQ Tools 压力感知系统希望补齐这套能力：从传感器端感知压力，到软件端实时显示压力图像，再到数据采集、回放、分析和报告导出。

### 3.2 Why Now

当前出现了三个机会：

1. 展厅和门店需要更有科技感的实时展示工具，帮助客户理解座椅舒适性。
2. 中科研、高校和实验室客户在招标中对压力图像显示、数据采集、数据导出和科研分析提出更高要求。
3. 国外品牌如 XSENSOR、Tekscan、novel pliance、Tactilus、PPS 在价格、交付、本地化和定制响应上存在国产替代空间。

### 3.3 Competitive Context

直接竞品包括：

| Competitor | Key Strength | Gap JQ Tools Can Attack |
|---|---|---|
| XSENSOR | 座椅压力成像标杆，软件体验强 | 价格高、本地定制弱、国内交付成本高 |
| Tekscan | 产品线广，工程和科研信任度强 | 系统复杂，座椅展示场景不够轻量 |
| novel pliance | 科研属性强，适合招标和实验室 | 展示体验弱，本地化和服务存在机会 |
| Tactilus | 工业压力映射和定制能力强 | 软件体验偏工程，不聚焦座椅舒适性 |
| PPS | 柔性触觉传感和软件集成强 | 国内座椅场景品牌认知较弱 |

JQ Tools 的机会是做成一套面向中国汽车座椅场景的国产压力成像平台，而不是只做单一传感器或单一显示软件。

## 4. Objective

### 4.1 Product Objective

建立一套可商用、可演示、可交付、可参与招标的汽车座椅压力感知系统。系统需要支持实时压力图像显示、数据采集、数据分析、回放和报告导出，配合纤维压力传感器体现技术差异。

### 4.2 Business Objective

1. 支持纤维压力传感器从单一硬件产品升级为完整解决方案。
2. 提升产品科技属性，让销售和展厅场景更容易演示价值。
3. 在科研招标和汽车座椅研发场景中替代 XSENSOR 等国外品牌。
4. 建立 JQ Tools 软件平台能力，为后续更多传感器和场景扩展打基础。

### 4.3 Customer Objective

客户可以用更直观、更科学的方式回答这些问题：

1. 这个座椅坐上去哪里压力大？
2. 左右压力是否平衡？
3. 不同座椅、材料、坐姿之间有什么差异？
4. 数据是否能保存、回放、导出并用于报告？
5. 设备是否能满足科研测试或招标参数要求？

### 4.4 Key Results

| Objective | Key Result |
|---|---|
| 完成可演示版本 | V1 支持传感器连接、实时热力图、基础指标、录制和导出 |
| 支持销售试点 | 完成不少于 3 个展厅/门店/客户现场试点演示 |
| 支持科研招标 | 形成一套标准参数表、测试报告模板和校准说明 |
| 提升数据可信度 | 完成压力范围、重复性、漂移和采样稳定性测试 |
| 对标竞品 | 输出与 XSENSOR、Tekscan、pliance 的功能和参数对比表 |

## 5. Market Segment(s)

### 5.1 Segment A: 汽车座椅研发团队

他们的任务是验证座椅舒适性、支撑性、材料差异和结构改进效果。他们需要客观数据，而不是只依赖试坐感受。

核心需求：

1. 采集座面和靠背压力分布。
2. 对比不同座椅、材料、坐姿和测试人员。
3. 导出数据给研发、质量和供应商团队。
4. 生成可复用的测试报告。

主要约束：

1. 数据必须稳定可信。
2. 传感器不能明显影响坐感。
3. 软件要能长期记录和回放。

### 5.2 Segment B: 展厅、门店和体验中心

他们的任务是让用户直观看到座椅的支撑效果和舒适性差异。展示必须简单、好看、实时。

核心需求：

1. 大屏实时显示压力热力图。
2. 用简单指标解释坐姿和座椅支撑。
3. 支持不同座椅或坐姿的快速对比。
4. 一键生成体验结果。

主要约束：

1. 操作必须简单。
2. 界面要有科技感。
3. 不能暴露复杂工程参数。

### 5.3 Segment C: 科研院所、高校和实验室

他们的任务是进行压力测试、人体工程学研究、材料研究或设备招标验收。

核心需求：

1. 明确的传感器参数。
2. 数据采集、导出和复现实验。
3. 校准说明和误差说明。
4. 支持科研报告和二次分析。

主要约束：

1. 招标文件会对传感点数、采样率、压力范围、软件功能提出硬要求。
2. 客户会对标 XSENSOR、Tekscan、pliance 等国外品牌。
3. 数据格式需要开放，便于后续分析。

## 6. Value Proposition(s)

### 6.1 Core Value Proposition

JQ Tools 压力感知系统是一套面向汽车座椅场景的国产实时压力成像平台。它把纤维压力传感器、实时压力图像、数据采集、分析和报告整合在一起，帮助客户更直观、更科学地理解座椅压力分布。

### 6.2 Customer Gains

客户可以获得：

1. 实时可视化的压力热力图。
2. 可保存、可回放、可导出的测试数据。
3. 更适合中国客户的中文软件、中文报告和本地交付。
4. 相比国外品牌更灵活的定制和更低的采购门槛。
5. 能用于研发、展示和科研的同一套系统。

### 6.3 Customer Pains Avoided

客户可以避免：

1. 只靠主观试坐判断座椅舒适性。
2. 国外设备采购周期长、价格高、维护不方便。
3. 软件不中文化、报告不适合国内交付。
4. 数据展示好看但无法用于分析，或数据分析强但不适合展示。

### 6.4 Differentiation Against Competitors

| Dimension | Foreign Competitors | JQ Tools Opportunity |
|---|---|---|
| 传感器 | 成熟压力垫或电容阵列 | 纤维压力传感器，强调柔性贴合和国产原创 |
| 软件 | 工程或科研导向 | 同时支持演示模式和科研模式 |
| 本地化 | 中文、本地招标、本地服务有限 | 中文界面、中文报告、快速响应 |
| 成本 | 采购和维护成本高 | 做国产替代，降低采购门槛 |
| 定制 | 响应慢，费用高 | 可按座椅、展厅和招标需求定制 |

## 7. Solution

### 7.1 UX / User Flows

#### Flow 1: 展厅实时演示

1. 工作人员打开 JQ Tools。
2. 选择“演示模式”。
3. 系统自动连接传感器。
4. 用户坐上座椅。
5. 大屏显示实时压力热力图。
6. 系统显示左右平衡、最大压力、接触面积和坐姿变化。
7. 工作人员点击“生成体验报告”。

#### Flow 2: 研发测试采集

1. 工程师创建测试项目。
2. 选择座椅型号、传感器型号和测试人员信息。
3. 执行校准或零点设置。
4. 开始采集压力数据。
5. 标记测试片段，如正常坐姿、前倾、后靠、左右倾斜。
6. 停止采集并回放数据。
7. 导出 CSV/Excel/图片/报告。

#### Flow 3: 科研实验

1. 研究人员创建实验。
2. 设置采样率、采集时长、压力单位和导出格式。
3. 连接座面和靠背传感器。
4. 记录实验过程。
5. 对比多个样本或多个座椅。
6. 导出原始数据和分析结果。

### 7.2 Key Features

#### F1. 设备连接与状态管理

系统需要识别适配传感器和采集设备，显示连接状态、设备编号、传感器规格、采样状态和异常提示。

Acceptance Criteria:

1. 用户能看到传感器是否在线。
2. 设备断开时软件给出明确提示。
3. 重新连接后可恢复采集。
4. 支持查看传感器基本信息。

#### F2. 实时压力热力图

系统需要把传感器数据转成实时压力图像，支持座面和靠背显示。

Acceptance Criteria:

1. 支持 2D 热力图显示。
2. 支持颜色标尺和压力单位。
3. 支持最大值、最小值、平均值显示。
4. 支持座面和靠背单独显示或组合显示。
5. 图像刷新应满足实时演示体验。

#### F3. 基础压力指标

系统需要计算常用指标，帮助用户理解压力分布。

Required Metrics:

1. 最大压力。
2. 平均压力。
3. 总压力或总载荷。
4. 接触面积。
5. 左右压力占比。
6. 前后压力占比。
7. 压力中心位置。
8. 压力中心轨迹。

#### F4. 采集、录制与回放

系统需要支持完整测试过程记录，便于研发和科研复盘。

Acceptance Criteria:

1. 支持开始、暂停、停止采集。
2. 支持保存完整压力数据。
3. 支持时间轴回放。
4. 支持片段标记。
5. 支持导出某一时间点的压力图像。

#### F5. 数据导出

系统需要支持客户后续分析和报告使用。

Export Formats:

1. CSV。
2. Excel。
3. PNG/JPG 压力图。
4. PDF 或 Word 报告。
5. 可选：MATLAB/Python 友好格式。

Acceptance Criteria:

1. 原始数据可导出。
2. 统计指标可导出。
3. 图片和报告中包含测试信息、时间和设备信息。

#### F6. 报告生成

系统需要提供标准报告模板，服务销售演示、研发测试和科研招标。

Report Content:

1. 项目信息。
2. 座椅信息。
3. 测试人员或样本信息。
4. 传感器信息。
5. 压力图。
6. 核心指标。
7. 对比结果。
8. 结论备注。

#### F7. 演示模式

演示模式面向展厅和门店。它需要少配置、强视觉、易解释。

Acceptance Criteria:

1. 一键进入全屏展示。
2. 界面突出热力图和少量关键指标。
3. 支持 A/B 对比。
4. 支持体验报告生成。
5. 支持隐藏复杂工程参数。

#### F8. 科研/工程模式

科研/工程模式面向研发和实验室。它需要更多参数和数据能力。

Acceptance Criteria:

1. 支持采样参数设置。
2. 支持校准和零点设置。
3. 支持多次测试对比。
4. 支持原始数据导出。
5. 支持传感器参数和测试条件记录。

#### F9. 竞品对标参数页

为了支持销售和招标，系统和材料中需要维护一张对标表。

Parameters:

1. 传感器类型。
2. 传感点数。
3. 采样率。
4. 压力范围。
5. 传感器厚度。
6. 支持座面/靠背。
7. 导出格式。
8. 报告能力。
9. 是否支持中文。
10. 交付周期和售后方式。

### 7.3 Technology

#### Hardware

1. 纤维压力传感器。
2. 座面传感器。
3. 靠背传感器。
4. 采集设备。
5. 数据连接线或无线模块。

#### Software

1. JQ Tools 桌面端软件。
2. 实时数据采集模块。
3. 压力矩阵转换模块。
4. 热力图渲染模块。
5. 指标计算模块。
6. 数据存储和回放模块。
7. 报告导出模块。

#### Data

每次测试建议保存：

1. 测试项目 ID。
2. 设备 ID。
3. 传感器规格。
4. 时间戳。
5. 原始压力矩阵。
6. 处理后的压力矩阵。
7. 统计指标。
8. 用户标记和备注。

### 7.4 Assumptions

1. 客户愿意用国产系统替代 XSENSOR 等国外品牌，只要数据可信、软件好用、价格合理。
2. 纤维压力传感器在贴合性、成本或可定制性上具备真实优势。
3. 展厅和门店客户更重视实时视觉效果和易用性。
4. 科研客户更重视参数、校准、导出和报告。
5. 第一版不需要完全覆盖国外竞品的全部高级功能，但必须覆盖核心演示和基础科研采集。

### 7.5 Non-Goals

V1 不做以下内容：

1. 不做完整云平台。
2. 不做多人账号权限系统。
3. 不做自动医学诊断。
4. 不做复杂 AI 坐姿识别，除非已有稳定算法。
5. 不支持所有第三方传感器，只支持当前适配传感器。

2. 基础压力指标。
3. 数据保存和导出。
