## 2. 产品背景

### 2.1 产品定位

JQ Tools V2.0 是一套用于压力感知硬件的桌面软件。

软件连接适配传感器后，可以实时显示压力数据，并支持 2D/3D 可视化、区域框选、量尺、数据统计、数据采集、历史回放、CSV 导入导出和数据对比。

当前优先场景是汽车座椅分析系统，尤其是汽车坐垫和汽车靠背的压力实时反馈、压力图像展示、数据采集和科研分析。

### 2.2 业务价值

JQ Tools V2.0 需要支撑三类核心价值：

1. 展厅和门店演示  
   实时展示用户坐上座椅后的压力分布，让压力变化可视化。

2. 汽车座椅研发测试  
   采集坐垫和靠背压力数据，分析平均压强、最大压强、受压面积、重心轨迹等指标。

3. 科研和招标交付  
   提供可追溯、可导出、可回放的数据能力，满足更高阶的数据采集和压力图像分析需求。

### 2.3 竞品对标方向

主要对标方向：

1. XSENSOR
2. Tekscan
3. 其他压力分布测试系统

JQ Tools 的核心差异点：

1. 适配自有纤维压力传感器。
2. 软件和硬件可一起定制。
3. 本地化部署，不依赖公网。
4. 成本更可控。
5. 更适合汽车座椅、展厅演示和国内科研交付场景。

## 3. 产品目标

### 3.1 V2.0 总目标

完成一套可交付的压力感知桌面软件，让用户可以完成从设备连接、实时展示、工具分析、数据采集、历史回放到 CSV 导出的完整闭环。

### 3.2 业务目标

1. 支撑汽车座椅和靠背压力感知系统的展示、测试和交付。
2. 支撑门店、展厅、研发和科研客户的实时压力图像展示。
3. 提供可追溯、可导出、可回放的数据能力。
4. 为后续数据对比、报告生成、标定分析和高级算法打基础。

### 3.3 成功标准

设备连接：合法设备可一键连接，非法设备不能采集。  
实时展示：2D/3D 视图可实时反映压力变化。  
数据采集：可开始、停止、命名、备注并保存历史记录。  
数据回放：历史数据可播放、暂停、拖动进度、倍速回放。  
数据导出：CSV 可导出，并可被软件重新导入校验。  
数据可信：原始数据、显示数据、统计数据、导出数据口径明确。  
稳定性：采集中异常退出后，可恢复或保留已采集数据。  
兼容性：Windows/macOS 可打包运行，串口缓存写入 userData。

## 7. 模块一：打开软件后的连接流程

### 7.1 模块目标

用户打开软件后，可以完成设备配置、设备连接、设备识别、授权校验和设备参数读取。连接成功后，软件进入实时压力数据展示状态。

### 7.2 当前已实现

已实现：

1. 首次启动检查本地设备配置。
2. 没有配置时进入 `/macConfig`。
3. 支持 `MAC地址:类型` 批量录入。
4. 支持一键连接。
5. 支持重新连接和断开连接。
6. 支持串口扫描。
7. 支持 CH340 设备识别。
8. 支持波特率自动探测，范围包括 921600、1000000、3000000。
9. 支持 MAC 地址读取。
10. 支持设备在线状态展示。
11. 支持系统类型切换。
12. 支持设备管理页。
13. 打包后串口缓存写入 Electron userData，避免写入 app.asar。

### 7.3 V2.0 需求

CON-01 软件启动  
优先级：P0  
Windows/macOS 安装包启动后进入中文界面。

CON-02 设备配置检查  
优先级：P0  
首次启动无配置时进入 `/macConfig`。

CON-03 一键连接  
优先级：P0  
用户点击后自动完成串口扫描、波特率探测、MAC 读取。

CON-04 设备类型识别  
优先级：P0  
当前版本以 MAC 映射为主，后续可升级为固件返回。

CON-05 固件/授权校验  
优先级：P0  
未授权设备或类型不匹配设备不得采集。

CON-06 设备参数展示  
优先级：P0  
展示 MAC、设备类型、串口号、波特率、采样率。

CON-07 连接异常提示  
优先级：P0  
连接失败、MAC 读取失败、授权失败、设备断开时给出中文提示。

### 7.4 待补齐项

【P0】【待确认】设备类型识别口径  
需要明确当前 V2.0 是以 MAC 本地映射为主，还是由固件直接返回设备类型。当前实现更接近 MAC 本地映射。

【P0】【待补齐】授权失败是否拦截采集  
需要明确 `/startCol` 在授权失败、MAC 未配置、设备类型不匹配时是否直接拒绝采集。

【P1】【待确认】MAC 配置分隔符  
当前 `MAC地址:类型` 容易和 MAC 地址自身的冒号冲突。建议改为 `MAC=类型`，或明确按最后一个冒号解析。

【P0】【待补齐】设备参数字段  
需要明确固件版本、点阵数量、压力范围是否读取、是否展示、是否入库、是否导出。

### 7.5 验收标准

1. 无设备配置时，启动后进入 MAC 配置页。
2. 合法设备可一键连接成功。
3. 非法设备、类型不匹配设备不能开始采集。
4. 连接成功后能看到 MAC、设备类型、串口、波特率、采样率。
5. 设备断开后界面有明确状态变化和提示。

1. 置零后，压力图和统计指标按置零后口径变化。

5. 框选采集不丢失全量原始矩阵。
6. 翻转只改变显示方向，或明确保存显示方向时可被还原。
7. 量尺在缩放和翻转后仍按真实点距计算。

## 13. 全局数据规则

### 13.1 数据分层

为避免后续数据口径混乱，V2.0 必须明确以下数据层：

rawData  
设备原始方向、原始压力矩阵。必须保留。

zeroedData  
扣除置零基线后的矩阵。根据置零状态生成。

displayData  
用于界面展示的矩阵，可受翻转、颜色、润滑影响。必须可重建。

calculatedData  
用于统计指标的矩阵。口径必须明确。

storedData  
采集入库数据。必须可追溯。

exportedData  
CSV 导出数据。必须标明口径。

### 13.2 原始数据原则

1. 框选不得导致全量原始矩阵丢失。
2. 可视化调节不得污染原始数据。
3. 翻转如参与落库，必须保存 `dataDirection`，并保证可还原。
4. 置零如影响统计或导出，必须记录置零状态和置零时间。
5. CSV 导出必须让后续导入、回放和对比有足够字段。

### 13.3 采集和展示关系

展示操作可以影响用户看到的图像，但不应破坏采集数据。

需要明确：

1. 翻转是否只影响展示。
2. 润滑是否只影响展示。
3. 噪点消除是否只影响展示。
4. 颜色调节是否只影响展示。
5. 置零是否影响统计和导出。
6. 框选是否只影响局部统计。

## 14. 非功能需求

### 14.1 性能要求

1. 实时展示低延迟。
2. 目标 60fps，但不作为硬阻塞指标。
3. 采集期间视图操作不应导致明显卡死。
4. 30 分钟采集后软件不应明显变慢。
5. 大文件导出时软件不应无响应。

### 14.2 可靠性要求

1. 采集中异常退出后，应保留已采集数据。
2. SQLite 数据库损坏时，应保留备份。
3. 本地缓存应可清理。
4. 正式历史数据不应被误删。

### 14.3 安全性要求

1. 未授权设备不得采集。
2. 设备类型不匹配时不得采集。
3. MAC 配置需要有格式校验。
4. 导入 CSV 需要结构校验。

### 14.4 兼容性要求

1. 支持 Windows。
2. 支持 macOS。
3. 支持多种传感器硬件。
4. 打包后文件写入必须使用 userData、data 或用户选择目录，不能写入 app.asar。

## 17. 发布准入

### 17.1 必须通过

真实设备一键连接成功率满足现场使用。

采集  
连续采集 30 分钟无明显卡死或数据中断。

回放

CSV  
导出 CSV 可打开、可导入、字段可解释。

异常

性能  
2D/3D 基本操作不卡死。

安装  
Windows/macOS 打包后可启动，缓存路径正确。

### 17.2 阻塞发布的问题

出现以下任一问题，不建议发布 V2.0：

1. 合法设备无法稳定连接。
2. 非法设备仍可采集。
3. 设备类型不匹配仍可采集。
4. 采集数据无法保存。
5. 停止采集后历史记录丢失。
6. CSV 导出数据缺失或格式无法解析。
7. 置零只改变按钮状态，不改变数据基线。
8. 框选导致全量原始数据丢失。
9. 异常退出后缓存损坏且无法恢复。
10. 实时压力图无法反映压力变化。
